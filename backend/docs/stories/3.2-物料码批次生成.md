## 1. V1.3.7 → V1.3.8 升级动因

V1.3.7 物料码 `WL-XXXX`（如 `WL-A001`）只标识料号，不带批次。仓管扫码只能定位物料，无法定位具体批次（哪次到货、哪批质检）。

**V1.3.8 方案**：物料码 = 物料号 + 批次号 复合，扫码一次同时获得两层信息。

### 1.1 复合规则

```
V1.3.7: WL-A001                    (物料粒度)
V1.3.8: WL-A001-BATCH-20260613-0001 (物料 + 批次)
```

| 部分 | 含义 | 来源 |
|------|------|------|
| WL- | 前缀（保留） | V1.3.7 1.11 |
| A001 | material_no | crm_material.material_no |
| BATCH- | 批次前缀（新增） | V1.3.8 3.2 |
| YYYYMMDD-流水 | 批次号 | crm_batch.batch_no |

## 2. Acceptance Criteria

### AC-3.2.1 物料码自动生成（到货时）

**Given** Story 3.1 到货接口 `POST /api/v1/incoming/batch-create` 完成
**When** 系统为每条到货物料写入 `crm_batch` 表
**Then** 同步调用 `MaterialBarcodeService.generate(materialId, batchId)` 生成复合物料码
**And** 写入新表 `crm_material_barcode_batch`
**And** 格式：`WL-{material_no}-BATCH-{YYYYMMDD}-{seq:4}`
**And** DB 唯一索引 `uniq_barcode_no` 保证不重复
**And** 旧物料码 `WL-{material_no}` 标记 `is_active=0` 但保留历史

### AC-3.2.2 扫码解析（APP 端）

**Given** 仓管 APP 扫码 `WL-A001-BATCH-20260613-0001`
**When** 调用 `GET /api/v1/material-barcode/parse?barcode=WL-A001-BATCH-20260613-0001`
**Then** 解析返回 `{material_id, material_no, batch_id, batch_no, arrived_at, quality_status}`
**And** HTTP 200，P95 < 50ms（Redis 缓存）
**And** 物料码格式错误 HTTP 400

## 3. 数据库迁移（V50）

### V50__material_barcode_batch.sql

```sql
CREATE TABLE crm_material_barcode_batch (
  id BIGINT NOT NULL AUTO_INCREMENT,
  material_id BIGINT NOT NULL,
  batch_id BIGINT NOT NULL,
  barcode_no VARCHAR(50) NOT NULL COMMENT 'WL-{material_no}-BATCH-{YYYYMMDD}-{seq}',
  is_active TINYINT(1) DEFAULT 1,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (id),
  UNIQUE KEY uniq_barcode_no (barcode_no),
  KEY idx_material (material_id),
  KEY idx_batch (batch_id),
  KEY idx_active (is_active)
) ENGINE=InnoDB COMMENT='物料码批次映射表 V1.3.8';
```

## 5. DocNoGenerator 扩展

复用 V1.3.7 1.5 DocNoGenerator，新增模板 `BATCH{yyyyMMdd}{seq:4}`：

```java
String batchNo = DocNoGenerator.next("BATCH{yyyyMMdd}{seq:4}");
// 输出: BATCH-20260613-0001
```

复合物料码拼接：

```java
String materialBarcode = "WL-" + materialNo + "-" + batchNo;
// 输出: WL-A001-BATCH-20260613-0001
```

并发安全：DocNoGenerator 已用 Redis INCR 保证（V1.3.7 1.5 已实装）。
