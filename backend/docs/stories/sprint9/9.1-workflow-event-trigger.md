## 2. 触发点

### 2.1 4.1 NoOrderPurchaseService.createNoOrderPurchase

PO 创建 AFTER_COMMIT 触发 `EVENT_CREATED` 事件：
```java
workflowEventService.recordEvent(
    "PO_APPROVAL", po.getId(), po.getPoNo(),
    WorkflowEventService.EVENT_CREATED,
    po.getApprovalRoute(),  // 初始路由（如 PROCUREMENT_MANAGER）
    null, null, "创建无订单 PO", null, null);
```

### 2.2 4.2 ProcurementApprovalRouter.previewRoute

预览路由时触发 `EVENT_PREVIEWED`：
```java
workflowEventService.recordEvent(
    "PO_APPROVAL", bizId, bizNo,
    "PREVIEWED",
    route.get(0),  // PROCUREMENT_MANAGER / GM+PROCUREMENT_MANAGER
    null, null, comment, matchedNodeIndex, matchedThreshold);
```

### 2.3 1.32 RfqService.awardRfq

中标后触发 `EVENT_AWARDED`：
```java
workflowEventService.recordEvent(
    "QUOTE_APPROVAL", rfqId, rfqNo,
    "AWARDED",
    "PURCHASER", null, null,
    "RFQ 中标，PO 号：" + poNo, null, null);
```

## 4. 风险

| 风险 | 等级 | 缓解 |
|------|------|------|
| recordEvent 抛异常影响主流程 | 🟢 低 | @Transactional + 异常吞掉（审计日志不影响业务） |
| 重复触发（previewRoute + 实际审批） | 🟢 低 | recordEvent 不做去重（按时间序记录即可） |
| AFTER_COMMIT 钩子失效 | 🟢 低 | 用 TransactionSynchronizationManager（与 Sprint 7 4.1 一致） |
