## Epic Context

- **WHY（来自 PRD）**：客户原话"业务员不能改单价、经理才能看毛利"—— 报价是销售流程的"第一个有金额的单据"，既要支持业务员快速报价（自审 < 5万），也要支持多级审批（5-20万 部门经理 / > 20万 总经理），还要支持黑名单客户直接驳回。
- **业务红线**（V1.3.7）：
  ① 报价单号 `BJ+年月日+4位流水`（如 `BJ20260610-0001`），由 `DocNoGenerator` 实时生成（Story 1.3 已实装 `biz.doc-no.quote` 参数）
  ② 金额阈值：`< 5万` 业务员自审 / `5-20万` 部门经理 / `> 20万` 总经理（客户为黑名单直接驳回 40902 `CUSTOMER_BLACKLIST`）
  ③ 审批通过后一键转订单（订单号 `XS+年月日+4位流水`），图纸/明细/数量/单价自动带出，可调整后下单
  ④ 报价单导出 PDF（带公司 logo + 报价明细 + 审批签字栏）+ Excel
- **解耦约束**：报价服务是 `erp_business` schema 业务模块（与 `erp_platform` 工作流中台解耦），通过 **Feign 客户端** + **Redis Stream 事件**消费工作流能力

## Acceptance Criteria

> 渲染源：PRD `docs/prd/epic-2-客户与销售.yaml#2.2` + `docs/prd/2-需求.md#FR-2-2`（FR-2-2-1/2/3/4）

### AC-2.2.1 — 报价单 CRUD + 字段必填校验（FR-2-2-1）

**GIVEN** 业务员已登录（具备 `quote:create/quote:read/quote:update` 权限），Story 1.1 已初始化 `salesperson` 角色 + Story 1.2 已就绪工作流引擎
**WHEN** 业务员执行下列操作之一：
- (a) `POST /api/v1/quotes` 创建报价单（请求体含 `customerId` + `deliveryDate` + `currency=CNY` + `items[]` 图号/材质/尺寸/数量/单价/金额/是否FA/是否新件 + `comment`）
- (b) `GET /api/v1/quotes/{id}` 查询报价详情（含审批状态 + 历史修改记录）
- (c) `PUT /api/v1/quotes/{id}` 修改报价（仅 `DRAFT` 状态可改；`SUBMITTED` 状态抛 40903 `QUOTE_NOT_EDITABLE`）
- (d) `GET /api/v1/quotes?page=0&size=20&status=&customerId=&owner=&dateFrom=&dateTo=` 列表查询（按权限过滤：业务员见自己 / 经理见本部门 / 总经理见全部）

**THEN** 8 端点全部通过；报价单号 `BJ{yyyyMMdd}-{seq:4}` 实时生成；字段校验（`customerId` 必填 + `items` ≥ 1 + 数量 > 0 + 单价 ≥ 0 + 交期 ≥ 今天 + 是否FA/是否新件 是 boolean）；CRUD 100% 留痕（`@AuditLog` AFTER_COMMIT）

**AND** 金额自动计算：`amount = sum(item.quantity * item.unitPrice)`，保存到 `quote.total_amount` 字段；金额阈值由 Story 1.3 `sys_global_threshold` 双轨（Nacos 优先 + DB 回退）

### AC-2.2.2 — 多级审批 + 黑名单直接驳回（FR-2-2-2）

**GIVEN** 业务员已创建报价（`DRAFT` 状态），Story 1.2 4 工作流模板 + `QuoteApprovalRouter` 已就绪
**WHEN** 业务员或系统执行：
- (a) `POST /api/v1/quotes/{id}/submit` 提交审批（status `DRAFT → SUBMITTED`，触发 Story 1.2 4 通道推送 `stream:notify` + `OrSignRequired=false` + `WorkflowConfig.refreshScope`）
- (b) 客户为黑名单 → 直接驳回 40902 `CUSTOMER_BLACKLIST`，不进入审批流
- (c) 金额 `< 5万` → 业务员自审（`salesperson` 角色）
- (d) 金额 `5-20万` → 部门经理审批（`dept_manager` 角色，可能 OR 会签多人）
- (e) 金额 `> 20万` → 总经理审批（`gm` 角色）+ 财务总监双签（V1.3.7 `PAYMENT_FLOW` 模板）
- (f) `POST /api/v1/quotes/{id}/approve` 审批通过（status `SUBMITTED → APPROVED`，触发 `stream:notify` 推送给业务员）
- (g) `POST /api/v1/quotes/{id}/reject` 驳回（请求体含 `reason`，status `SUBMITTED → REJECTED`，通知业务员 + `@AuditLog` 写 `quote.reject_reason`）

**THEN** 4 阈值路由 100% 准确（与 Story 1.1 `QuoteApprovalRouter` 兼容）；`skipOnLeaveRule` 跳过请假候选人；OR 会签候选 `candidates: List<Long>` 完整列表返回前端；`@AuditLog` 100% 留痕

**AND** Story 1.2 4 工作流模板复用：`QUOTE_FLOW` 3 节点（salesperson/dept_manager/gm）；V1.3.7 P1 修补 ① OR 会签 + ② 跳过请假 100% 兼容

### AC-2.2.3 — PDF + Excel 导出 + 报价转订单（FR-2-2-3/4）

**GIVEN** 报价已 `APPROVED`，Story 1.3 `app.cache-ttl` 系统参数已就绪，PDF 模板已配置
**WHEN** 业务员执行：
- (a) `GET /api/v1/quotes/export/{id}?format=pdf` 导出 PDF（带公司 logo + 报价明细 + 审批签字栏 + 审批时间戳）
- (b) `GET /api/v1/quotes/export/{id}?format=excel` 导出 Excel（多 Sheet：报价基本信息 + 报价明细 + 审批记录）
- (c) `POST /api/v1/quotes/{id}/convert-to-order` 转订单（图纸/明细/数量/单价自动带出，可调整后下单；生成订单号 `XS{yyyyMMdd}-{seq:4}`，status `APPROVED → CONVERTED`，触发 `OrderService.createOrder`）
- (d) PDF 导出后 1h 内可重复下载（`app.cache-ttl` 控制 PDF 文件缓存 + `crm_audit_log` 写 `pdf.download` 留痕）

**THEN** PDF 用 `iText 7.2.5` 或 `OpenPDF 1.3.34` 生成（V1.3.7 部署阶段选型）；Excel 用 `Apache POI 5.2.5` 生成；转订单字段 100% 自动带出；`@AuditLog` 写 `quote.convert_to_order`

**AND** 5 条 UI 红线（V1.3.7 §8）覆盖：① 不显示金额（业务自审 < 5万） ② 二次密码确认（> 20万） ③ 黑名单高亮 ④ 转订单预览 ⑤ 导出审计

## 关键反馈（dev 实施时须注意的）

1. **T0.1 实体字段对齐**：`crm_quote.quote_no` 唯一索引（`UNIQUE KEY uniq_quote_no`）+ `crm_quote_item (quote_id, sort)` 复合索引
2. **T1.4 DocNoGenerator 复用**：Story 1.3 `biz.doc-no.quote` 已实装（`QUOTE-{yyyyMMdd}-{seq:4}`），但 PRD FR-2-2-1 是 `BJ+年月日+4位流水`（无 `-`），需调整 `biz.doc-no.quote` 模板为 `BJ{yyyyMMdd}{seq:4}`
3. **T1.5 金额只读**：写操作不接收 `totalAmount` 字段，由 `items` 自动计算；否则抛 40001 `QUOTE_AMOUNT_READONLY`
4. **T2.5 状态机守卫**：DRAFT 可改可提交；SUBMITTED 可撤回（重置为 DRAFT）；SUBMITTED 可审批/驳回；APPROVED 可转订单；REJECTED 终态
5. **T3.7 PDF 缓存**：`quote_pdf_cache` 表（id + quote_id + file_path + expire_at），1h 过期由定时任务清理
6. **V1.3.7 5 条 UI 红线**：① 业务自审 < 5万 列表不显示金额（仅显示图号/数量） ② > 20万 二次密码确认（admin 密码） ③ 黑名单客户高亮红色 ④ 转订单预览对话框 ⑤ 导出审计（谁/何时/导出哪个文件）
