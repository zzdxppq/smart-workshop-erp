## 1. V1.3.7 → V1.3.8 升级动因（PM 范蠡反馈 #3）

PM 反馈：CNC 加工厂存在**无订单采购**场景：
- 紧急补料（生产中途物料损坏）
- 客户加单（口头/微信下单，订单系统未录入）
- 库存置换（与同行互换物料）
- 其他（车间临时需求）

V1.3.7 1.32 RFQ 流程强制 `source_order_id`，无法支持无订单采购。

**V1.3.8 方案**：扩展 1.32 RFQ 流程，新增 `source_type=NO_ORDER` 模式 + `purchase_reason` 4 选 1 枚举字段。

### 1.1 PM 建议 vs V1.3.7 现状

| PM 建议 | V1.3.7 现状 | V1.3.8 缺口 |
|---------|-------------|-------------|
| MRP 自动触发 PO | 1.17 + 1.32 已闭环 | 无 |
| 采购主动创建（无订单）| 1.32 RFQ 强制 order_id | **新增 NO_ORDER 模式** |
| 采购理由 4 选 1 | 1.32 无字段 | **新增 purchase_reason 枚举** |
| 总经理汇总报表 | 1.45 已有 | 由 4.3 扩展过滤 |

## 2. Acceptance Criteria

### AC-4.1.1 无订单采购创建

**Given** 采购员 PURCHASER 选择「无订单采购」入口
**When** 调用 `POST /api/v1/purchase/no-order` 传入：
```json
{
  "purchase_reason": "URGENT_REPLENISH",
  "items": [{"material_id": 5001, "quantity": 50, "estimated_price": 80.0}],
  "supplier_id": 2001,
  "remark": "生产中途 M1 损坏 5 件，紧急补料"
}
```
**Then** 创建 PO `PO-20260613-0008`，`source_type=NO_ORDER`
**And** `purchase_reason=URGENT_REPLENISH` 必填
**And** 走 4.2 审批路由（金额 > 1 万 → PROCUREMENT_MANAGER 审批）
**And** HTTP 201，返回 PO 详情

### AC-4.1.2 采购理由枚举

**Given** 系统枚举 4 类采购理由
**When** 调用 `GET /api/v1/purchase/reasons`
**Then** 返回：
```json
{
  "code": 0,
  "data": [
    {"code": "URGENT_REPLENISH", "name": "紧急补料", "color": "red"},
    {"code": "CUSTOMER_ADD",     "name": "客户加单", "color": "orange"},
    {"code": "STOCK_SWAP",       "name": "库存置换", "color": "blue"},
    {"code": "OTHER",            "name": "其他",     "color": "gray"}
  ]
}
```
**And** 字典存 `sys_dict` 表（V1.3.7 1.3 体系）

### AC-4.1.3 看板联动

**Given** V1.3.7 1.45 多维度看板已有「采购维度」过滤
**When** 总经理在 1.45 看板勾选 `purchase_reason=URGENT_REPLENISH`
**Then** 看板返回所有无订单采购 PO
**And** 4.3 总经理汇总报表复用此过滤条件

```sql
INSERT INTO sys_dict (dict_type, dict_code, dict_name, sort_order, is_active, created_at) VALUES
  ('PURCHASE_REASON', 'URGENT_REPLENISH', '紧急补料', 1, 1, NOW()),

  ('PURCHASE_REASON', 'OTHER',            '其他',     4, 1, NOW());

-- PO 表扩展
ALTER TABLE crm_purchase_order
  ADD COLUMN source_type ENUM('FROM_ORDER','FROM_MRP','NO_ORDER') NOT NULL DEFAULT 'FROM_ORDER' AFTER po_type,
  ADD COLUMN purchase_reason VARCHAR(30) NULL AFTER source_type,
  ADD KEY idx_source_type (source_type),
  ADD KEY idx_reason (purchase_reason);

COMMENT ON COLUMN crm_purchase_order.source_type IS '订单来源：订单/MRP/无订单';
COMMENT ON COLUMN crm_purchase_order.purchase_reason IS '采购理由（仅 NO_ORDER 必填）';
```

## 4. 端点契约

### 4.1 `POST /api/v1/purchase/no-order`（核心新建）

**Request**: 详见 AC-4.1.1

**Response 201**:
```json
{
  "code": 0,
  "data": {
    "po_id": 8008,
    "po_no": "PO-20260613-0008",
    "source_type": "NO_ORDER",
    "purchase_reason": "URGENT_REPLENISH",
    "approval_route": "PROCUREMENT_MANAGER",
    "estimated_total": 4000.00
  }
}
```

**Response 400**: 缺 `purchase_reason` · `{"code": 40002, "msg": "purchase_reason required for NO_ORDER"}`

### 4.2 `GET /api/v1/purchase/reasons`（字典查询）

详见 AC-4.1.2。

### 4.3 `GET /api/v1/purchase/board`（看板聚合扩展 · 1.45 复用）

新增 query 参数 `purchase_reason=URGENT_REPLENISH` 与 `source_type=NO_ORDER` 过滤。
