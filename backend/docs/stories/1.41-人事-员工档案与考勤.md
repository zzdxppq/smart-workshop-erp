- GET  /api/v1/hr/employees/{id}（档案详情）
- PUT  /api/v1/hr/employees/{id}（档案更新）

## Acceptance Criteria（3）

- AC-10.1.1：员工档案 CRUD（增/查/改/分页），含工号/姓名/部门/岗位/入职日期/在职状态等字段
- AC-10.1.2：考勤打卡（含上班/下班/午休进出），自动写入考勤流水；时间冲突检测见 P1
- AC-10.1.3：跳过请假状态检测（复用 1.2 SkipOnLeaveRule 规则引擎）

## P1 修补（3）

- 员工工号唯一（DB 唯一索引 + 服务层校验）
- 考勤打卡时间冲突检测（同员工同方向 5 分钟内重复打卡 → 拒绝 409）
- 跳过请假状态（复用 1.2 SkipOnLeaveRule；请假中打卡不计入有效出勤）
