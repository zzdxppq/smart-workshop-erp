## Epic Context

- **WHY（来自 PRD）**：CNC 加工厂的"图纸"是核心资产。一张图错了，整批报废。图纸管理是 Epic 3 的入口 Story，必须先把"图号 + 版本"管理好，下游工艺库 / BOM / 工程转化 才有数据基础。
- **业务红线**（V1.3.7）：
  ① 图纸号 `DWG{yyyyMMdd}{seq:4}`（如 `DWG20260612-0001`），由 Story 1.5 `DocNoGenerator` 复用（`biz.doc-no.drawing` 模板）
  ② 图纸版本：每次发布新版本号 `V1.0` → `V1.1` → `V2.0`（major.minor 二级版本号，V1.3.7 §附录-d 术语表）
  ③ 图纸状态机：`DRAFT → PUBLISHED → OBSOLETE → ARCHIVED`（4 状态机，与订单 7 状态机独立）
  ④ 发布审批：FA 件图纸（`is_fa=1`）需总工 + 部门经理双签（V1.3.7 §附录-a 金额阈值速查表，类比订单 > 20万 双签）
  ⑤ 图纸查询：支持图号/物料/规格/客户 多维过滤 + 全文搜索（V1.3.7 红线 4：业务唯一性）
- **解耦约束**：图纸服务是 `erp_business` schema 业务模块（与 `erp_platform` 工作流中台解耦），通过 **Feign 客户端** + **Redis Stream 事件**消费工作流能力；下游 Story 3.2（工程转化）/ 3.3（BOM）/ 3.4（工艺）都依赖 1.7 的图号/版本数据

## Acceptance Criteria

> 渲染源：PRD `docs/prd/epic-3-图纸与物料.yaml#3.1` + `docs/prd/2-需求.md#FR-3-1`（FR-3-1-1/2/3/4）
> 关联前置：Story 1.1 `sys_user/sys_role` · Story 1.2 4 工作流模板 · Story 1.3 `sys_dict` + `sys_change_log` · Story 1.4 APP 5 类码 · Story 1.5 `DocNoGenerator` · Story 1.6 `OrderItem` 图号/材质/尺寸

### AC-3.1.1 — 图纸 CRUD + 字段必填校验（FR-3-1-1）

**GIVEN** 工程师已登录（具备 `eng:create/eng:read/eng:update` 权限），Story 1.5 `DocNoGenerator` 已实装
**WHEN** 工程师执行下列操作之一：
- (a) `POST /api/v1/drawings` 创建图纸（请求体含 `drawingNo` 可选自动生成 + `category` + `customerId` 可选 + `material` + `spec` + `dimensions` + `isFa` 1/0 + `isNew` 1/0 + `version` 默认 `V1.0` + `comment`）
- (b) `GET /api/v1/drawings/{id}` 查询图纸详情（含状态机 + 历史版本 + 发布审批签字）
- (c) `PUT /api/v1/drawings/{id}` 修改图纸（仅 `DRAFT` 状态可改；`PUBLISHED` 及之后状态抛 40903 `DRAWING_NOT_EDITABLE`）
- (d) `DELETE /api/v1/drawings/{id}` 删除图纸（仅 `DRAFT` 状态可删；已 `PUBLISHED` 需先 `ARCHIVED`）

**THEN** 5 端点全部通过；图纸号 `DWG{yyyyMMdd}{seq:4}` 实时生成（继承 1.5 `DocNoGenerator` 扩展 `biz.doc-no.drawing`）；字段校验（`category` 必填 + `material` 必填 + `spec` 必填 + `dimensions` 必填 + `isFa/isNew` 是 0/1）；CRUD 100% 留痕（`@AuditLog` AFTER_COMMIT）

**AND** 权限过滤：工程师见自己（`owner_user_id`）/ 部门经理见本部门（`dept_id`）/ 总工见全部 + `@PreAuthorize("hasAuthority('eng:read')")`

### AC-3.1.2 — 版本管理 + 发布审批（FR-3-1-2）

**GIVEN** 图纸已创建（`DRAFT` 状态），Story 1.2 4 工作流模板已就绪
**WHEN** 工程师或审批人执行：
- (a) `POST /api/v1/drawings/{id}/versions` 新增版本（请求体含 `version` 默认 `V{major}.{minor}` + `changeNote` + `attachment`）
- (b) `POST /api/v1/drawings/{id}/publish` 发布图纸（status `DRAFT → PUBLISHED`，触发 Story 1.2 4 通道推送 `stream:notify` + `OrSignRequired=false`）
- (c) `is_fa=1`（FA 件）→ 总工 + 部门经理双签（V1.3.7 `FA_DRAWING_FLOW` 模板，类比订单 > 20万 双签）
- (d) `is_fa=0`（非 FA 件）→ 总工单人审批
- (e) `POST /api/v1/drawing-versions/{id}/obsolete` 废弃旧版本（status `PUBLISHED → OBSOLETE`，仅在新增版本后允许）
- (f) `POST /api/v1/drawings/{id}/archive` 归档（status `OBSOLETE → ARCHIVED`，所有字段只读）

**THEN** 4 状态机严格守卫：`DRAFT → PUBLISHED → OBSOLETE → ARCHIVED`；`PUBLISHED → DRAFT` 是唯一可回退路径（驳回）；非法转换抛 40904 `DRAWING_STATE_INVALID`；OR 会签候选人列表完整；`@AuditLog` 100% 留痕

**AND** 状态机守卫：版本号自动递增（V1.0 → V1.1 → V1.2 → V2.0），每次发布写入 `eng_drawing_version` 快照表

### AC-3.1.3 — 图纸查询（多维过滤 + 全文搜索）（FR-3-1-3）

**GIVEN** 图纸已就位（含 DRAFT/PUBLISHED/OBSOLETE/ARCHIVED 4 状态）
**WHEN** 工程师或系统执行：
- (a) `GET /api/v1/drawings?page=0&size=20&keyword=&version=&category=&customerId=&isFa=&status=` 列表查询（6 维过滤）
- (b) `GET /api/v1/drawings/{id}/versions` 版本列表
- (c) 全文搜索：图号 / 物料 / 规格 / 客户名 4 字段 `LIKE %keyword%` 模糊匹配

**THEN** 列表查询支持 6 维过滤 + 权限过滤；分页 20/50/100 三档；性能 P95<200ms（V1.3.7 §10 性能基线）

**AND** 全文搜索：`keyword` 命中 4 字段中任一即返回（OR 语义），按相关度排序（图号完全匹配 > 物料 > 规格 > 客户名）
