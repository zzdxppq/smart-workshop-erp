## 1. PM 反馈来源

Sprint 10 PM 决策 #2（2026-06-13）· 采纳方案 A · 详见 `docs/qa/evidence/sprint10-pm-decision-2-inspection-dto.md`

### 1.1 Sprint 10.5 Option A 现状（临时方案）

V1.3.8 Sprint 10.5 dev log §6.3 表明 InspectionCreate.vue 当前用 Option A **本地 interface**：

```typescript
// web-impl/src/views/quality/InspectionCreate.vue （当前 Option A）
interface InspectionFormLocal {
  materialCode: string
  qualityStatus: 'PASS' | 'FAIL' | 'PENDING'
  inspectItems: Array<{ name: string; value: string; result: 'OK' | 'NG' }>
  remark?: string
}

const form = ref<InspectionFormLocal>({ ... })  // 本地 interface · 不享受 codegen
```

**问题**：
1. 后端字段拼写错误（如 `materialCode` vs `material_code`）前端无法 typecheck 拦截
2. 后端 nullable 字段前端不知情（如 `actualInspectionDate` 可空）
3. 与 Sprint 10.1 codegen 价值脱节（类型基础设施有了但消费页用本地 interface）

### 1.2 PM 决策 #2 选项 A（采纳）

- 在 `backend/spec/openapi.yaml` 新增 `InspectionDTO` / `InspectionCreateRequest` / `InspectionResponse` schema + 1 端点契约
- web-impl `InspectionCreate.vue` 替换本地 interface 为 `import type { InspectionCreateRequest } from '@/api/generated/models/InspectionCreateRequest'`
- 二次 regen（依赖 Sprint 13.5 enum drift 对齐后 · regen baseline 稳定）

### 1.3 Sprint 13 立项占位

`core-config.yaml` `assigned_stories` 第 447-461 行已有 `id: "11.x"` 占位（V1.3.9-S1 backlog）· Sprint 13 立项后**正式 sharded 为 `id: "13.1"`**。

## 3. 端点契约（OpenAPI YAML 新增）

```yaml
# backend/spec/openapi.yaml (新增 · Sprint 13.1)
paths:
  /api/v1/quality/inspections:
    post:
      tags: [E7-Quality]
      summary: 创建品质检验单（V1.3.9 Sprint 13.1 · 补 InspectionDTO schema）
      operationId: createInspectionV1389
      security:
        - bearerAuth: []
      requestBody:
        required: true
        content:
          application/json:
            schema:
              $ref: '#/components/schemas/InspectionCreateRequest'
      responses:
        '201':
          description: 检验单创建成功
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/InspectionResponse'
        '400':
          description: 字段校验失败（code: 40001 INVALID_FIELD · 字段名 details.field）
          content:
            application/json:
              schema:
                $ref: '#/components/schemas/ErrorResponse'
        '403':
          description: 无 QC 权限（40301 QC_REQUIRED）
        '409':
          description: 物料码已被检验（40901 INSPECTION_DUPLICATE）

components:
  schemas:
    InspectionDTO:
      type: object
      description: 品质检验单完整 DTO（V1.3.7 1.28-1.30 + V1.3.9 Sprint 13.1 补齐）
      properties:
        id: { type: integer, format: int64 }
        inspectionNo: { type: string, description: '检验单号 · INSP-yyyyMMdd-NNNN' }
        materialCode: { type: string, description: '物料码 · WL-... 前缀' }
        materialName: { type: string, nullable: true, description: '物料名 · 关联 mtl_material.name' }
        batchNo: { type: string, nullable: true, description: '批次号 · 关联 wms_inbound.batch_no' }
        inspectionType:
          type: string
          enum: [INCOMING, IN_PROCESS, OUTGOING, FA, CMM]
          description: '检验类型 · V1.3.7 1.28 5 类'
        qualityStatus:
          type: string
          enum: [PASS, FAIL, PENDING, REWORK]
          description: '质检结果 · 与 Sprint 1.4 MaterialBarcodeParseResponse.qualityStatus 一致'
        inspectItems:
          type: array
          description: '检验项明细'
          items:
            $ref: '#/components/schemas/InspectionItemDTO'
        inspectorUserId: { type: integer, format: int64, description: '检验人 userId' }
        inspectorName: { type: string, nullable: true, description: '检验人姓名 · 关联 sys_user.full_name' }
        inspectionDate: { type: string, format: date-time, description: '实际检验日期' }
        actualInspectionDate: { type: string, format: date-time, nullable: true, description: '实际检验完成时间 · nullable（进行中为空）' }
        remark: { type: string, maxLength: 500, nullable: true }
        tenantId: { type: integer, format: int64, description: '多租户隔离' }
        createdAt: { type: string, format: date-time }
        updatedAt: { type: string, format: date-time }

    InspectionCreateRequest:
      type: object
      required: [materialCode, inspectionType, qualityStatus, inspectItems]
      properties:
        materialCode: { type: string, description: '物料码 · 必填' }
        inspectionType:
          type: string
          enum: [INCOMING, IN_PROCESS, OUTGOING, FA, CMM]
        qualityStatus:
          type: string
          enum: [PASS, FAIL, PENDING, REWORK]
        inspectItems:
          type: array
          minItems: 1
          items:
            $ref: '#/components/schemas/InspectionItemDTO'
        remark: { type: string, maxLength: 500, nullable: true }

    InspectionItemDTO:
      type: object
      properties:
        itemName: { type: string, maxLength: 50, description: '检验项名称（如"外观"/"尺寸"/"硬度"）' }
        standard: { type: string, maxLength: 200, nullable: true, description: '检验标准' }
        measuredValue: { type: string, maxLength: 100, nullable: true, description: '实测值' }
        result: { type: string, enum: [OK, NG, NA], description: '单项结果' }
        sortOrder: { type: integer, default: 0 }

    InspectionResponse:
      type: object
      properties:
        inspectionId: { type: integer, format: int64 }
        inspectionNo: { type: string }
        status: { type: string, enum: [CREATED, REJECTED] }
        createdAt: { type: string, format: date-time }
```

> **字段对齐 V1.3.7 既有端点**：
> - `inspectionType` 5 类（INCOMING/IN_PROCESS/OUTGOING/FA/CMM）来自 V1.3.7 1.28 来料/过程/成品 + 1.29 FA 首件 + 1.30 CMM 三次元
> - `qualityStatus` 4 值（PASS/FAIL/PENDING/REWORK）与 Sprint 1.4 `MaterialBarcodeParseResponse.qualityStatus` 一致
> - 字段顺序按 V1.3.7 既有 DDL 顺序（id → 业务字段 → 审计字段）

### AC-13.1.2 — `POST /api/v1/quality/inspections` 端点契约

**GIVEN** `InspectionCreateRequest` schema 已 ship
**WHEN** QC 角色调用 `POST /api/v1/quality/inspections`：
- Body `{ materialCode: "WL-260614-001", inspectionType: "INCOMING", qualityStatus: "PASS", inspectItems: [{ itemName: "外观", result: "OK" }] }`
- → 201 Created + `{ inspectionId: 12345, inspectionNo: "INSP-20260614-0001", status: "CREATED" }`
- Body 缺 `inspectItems` → 400 INVALID_FIELD details.field="inspectItems"
- Body `materialCode` 不存在 → 400 INVALID_FIELD details.field="materialCode"
- 非 QC 角色调用 → 403 QC_REQUIRED
- 物料码已被检验 → 409 INSPECTION_DUPLICATE
**THEN** 端点契约与 Sprint 10.1 codegen 一致 · typecheck:ci 退出码 0
