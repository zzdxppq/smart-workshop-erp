## 4. web-impl 协同（10.5 4 typecheck 错误修复）

### 4.1 useOutsourceStateMachine.ts 字面量修复

```typescript
// web-impl/src/composables/useOutsourceStateMachine.ts (改造)
import type { OutsourceState } from '@/api/generated/models/OutsourceState'

// 改动前（Sprint 10.5 临时方案 · 字符串字面量不与 codegen union 一致）：
const stateSteps: OutsourceState[] = ['DRAFT', 'SUBMITTED', 'ACCEPTED', 'IN_PROGRESS', 'COMPLETED', 'REWORK', 'CLOSED']

// 改动后（用 as const 字面量 · 与 codegen union 100% 一致）：
const stateSteps = ['DRAFT', 'SUBMITTED', 'ACCEPTED', 'IN_PROGRESS', 'COMPLETED', 'REWORK', 'CLOSED'] as const
type StateStep = typeof stateSteps[number]

// 校验 stateStep 是否合法
function isValidState(state: string): state is StateStep {
  return stateSteps.includes(state as StateStep)
}
```

### 4.2 OutsourceStateMachine.vue 4 typecheck 错误修复

| # | 错误位置 | 错误 | 修复 |
|---|---------|------|------|
| 1 | `currentOrder` 类型 | 用 `OutsourceStateHistory` 但访问 `supplierName` / `workorderNo` | 改为 `OutsourceOrder & {supplierName?: string; workorderNo?: string}` |
| 2 | `stateSteps` 联合类型不匹配 | `OutsourceState` union 8 值 vs stateSteps 7 值 | 用 `as const` 字面量（4.1 节）|
| 3 | `useOutsourceStateMachine().transition()` 入参 | `OutsourceState = 'DRAFT' \| ...` 与 generated union 不匹配 | 用 `StateStep` 字面量类型 |
| 4 | 5 处 `as any` 边界 | typecheck:ci 隐藏潜在字段拼写错误 | 全部替换为具体类型或 `as unknown as TypeName` |
