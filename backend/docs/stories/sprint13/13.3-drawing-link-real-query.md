## 4. 真实查询实现

### 4.1 5 类 link JOIN 真实查询

```java
// backend/src/main/java/.../core/authz/DrawingAuthz.java (改造)

@Component
public class DrawingAuthz {
    private final JdbcTemplate jdbc;
    private final RedisTemplate<String, List<Long>> redis;
    private static final Duration CACHE_TTL = Duration.ofMinutes(5);

    // 1. SALES → 订单 IDs
    @Cacheable(value = "drawingLink:SALES", key = "#userId")
    public List<Long> findSalesOrderIds(Long userId) {
        // 真实查询：crm_order.owner_user_id = userId AND status NOT IN ('DRAFT','CANCELLED')
        return jdbc.queryForList(
            "SELECT id FROM crm_order WHERE owner_user_id = ? AND status NOT IN ('DRAFT','CANCELLED')",
            Long.class, userId
        );
    }

    // 2. PURCHASER → PO IDs
    @Cacheable(value = "drawingLink:PURCHASER", key = "#userId")
    public List<Long> findPurchaserPoIds(Long userId) {
        return jdbc.queryForList(
            "SELECT id FROM purchase_order WHERE created_by = ? AND status NOT IN ('DRAFT','CANCELLED')",
            Long.class, userId
        );
    }

    // 3. WAREHOUSE → 入库单 IDs
    @Cacheable(value = "drawingLink:WAREHOUSE", key = "#userId")
    public List<Long> findWarehouseInboundIds(Long userId) {
        return jdbc.queryForList(
            "SELECT id FROM wms_inbound WHERE operator_user_id = ? AND status NOT IN ('DRAFT','CANCELLED')",
            Long.class, userId
        );
    }

    // 4. QC → 质检单 IDs
    @Cacheable(value = "drawingLink:QC", key = "#userId")
    public List<Long> findQcInspectionIds(Long userId) {
        return jdbc.queryForList(
            "SELECT id FROM qc_inspection WHERE inspector_user_id = ?",
            Long.class, userId
        );
    }

    // 5. OPERATOR → 当前工序 IDs（IN_PROGRESS）
    @Cacheable(value = "drawingLink:OPERATOR", key = "#userId")
    public List<Long> findOperatorProcessIds(Long userId) {
        return jdbc.queryForList(
            "SELECT id FROM crm_workorder_process WHERE operator_user_id = ? AND status = 'IN_PROGRESS'",
            Long.class, userId
        );
    }

    // 缓存失效（crm_drawing_link 写入时 @CacheEvict）
    @CacheEvict(value = {"drawingLink:SALES", "drawingLink:PURCHASER", 
                         "drawingLink:WAREHOUSE", "drawingLink:QC", 
                         "drawingLink:OPERATOR"}, allEntries = true)
    public void refreshAllLinkCaches() {
        // crm_drawing_link 写入后调用
    }
}
```

## 5. Acceptance Criteria

### AC-13.3.1 — 5 类 link 真实查询

**GIVEN** V54 `crm_drawing_link` 已 ship + crm_order/purchase_order/wms_inbound/qc_inspection/crm_workorder_process 5 表均含 owner_user_id / operator_user_id 字段
**WHEN** dev agent 替换 DrawingAuthz.findXxxLinkIds 5 个方法为真实 SQL 查询
**THEN**：
- SALES userId=100 → `findSalesOrderIds(100)` 返回 `[100,101]`（crm_order.owner_user_id=100 的非 DRAFT/CANCELLED 订单）
- PURCHASER userId=200 → `findPurchaserPoIds(200)` 返回 `[200,201]`
- WAREHOUSE userId=300 → `findWarehouseInboundIds(300)` 返回 `[300]`
- QC userId=400 → `findQcInspectionIds(400)` 返回 `[400,401]`
- OPERATOR userId=500 → `findOperatorProcessIds(500)` 返回 `[500]`（crm_workorder_process.operator_user_id=500 AND status='IN_PROGRESS'）
- 5 类查询均返回真实业务数据（非 username.hashCode() % N 占位）

### AC-13.3.2 — Redis 7 缓存（5min TTL + @CacheEvict 一致性）

**GIVEN** Redis 7 客户机房就位 + `@Cacheable` 范式已 ship
**WHEN** dev agent 添加 `@Cacheable(value="drawingLink:{ROLE}", key="#userId")` + `@CacheEvict allEntries=true`
**THEN**：
- 同一 userId 连续 2 次查询 → 第二次命中缓存（Redis `EXISTS drawingLink:SALES::100` = 1）
- 缓存 TTL = 5 分钟（`TTL drawingLink:SALES::100` = 300）
- crm_drawing_link 写入（INSERT）→ `@CacheEvict` 触发 → 下次查询重新加载 DB
- 缓存命中率 ≥ 80%（生产环境 · 5min 内同 userId 多次查询）

### AC-13.3.3 — 灰度 feature flag 集成

**GIVEN** Sprint 12.1 灰度 4 阶段 feature flag 已 ship
**WHEN** admin 在 sys_dict 开启 `draw.acl.gray.SALES = true`
**THEN**：
- SALES 角色下次请求触发 `findSalesOrderIds` 真实查询（不再用占位）
- 缓存命中则走 Redis · 未命中则走 DB
- 灰度关闭（`false`）→ 占位逻辑重新生效（Sprint 12.1 阶段 1 行为）

### AC-13.3.4 — 性能优化（5 类查询 < 100ms P99）

**GIVEN** V58 索引迁移已 ship + Redis 缓存 5min TTL
**WHEN** 1000 并发 SALES 用户同时查询 findSalesOrderIds
**THEN**：
- P99 响应时间 < 100ms（含 DB 查询 + Redis 缓存）
- DB QPS < 100（缓存命中 80%+）
- `EXPLAIN SELECT id FROM crm_order WHERE owner_user_id = ?` 使用 `idx_user_owner` 索引（type=ref）

### AC-13.3.5 — 管理员对账端点

**GIVEN** `GET /api/v1/drawings/admin/link-stats?linkType=ORDER` 端点契约已 ship
**WHEN** admin 调用该端点
**THEN**：
- 200 + `{ totalUsers: 50, usersWithRealQuery: 50, usersWithPlaceholder: 0, coverageRate: 1.0, byLinkType: { ORDER: 200, PO: 150, ... } }`
- 覆盖率 = 1.0（所有用户均用真实查询）
- 非 admin 角色调用 → 403 ADMIN_REQUIRED
