## 4. Acceptance Criteria

### AC-12.1.1 — 权限矩阵元数据定义（基础）

**GIVEN** Story 1.1 `sys_role` 已 ship（10 角色枚举）· Story 1.3 sys_global_threshold 4 金额阈值已 ship（独立 ACL 范式）
**WHEN** 系统执行 `GET /api/v1/drawings/permissions/matrix`（admin 角色调用）
**THEN** 返回 7 角色 × 5 操作矩阵（ENGINEER 全 5 ✅ · PROD_PLANNER preview+print ✅ · SALES preview+print+scope=ORDER · PURCHASER preview+print+scope=PO · WAREHOUSE preview+print+scope=INBOUND · QC preview+print+scope=INSPECTION · OPERATOR preview+scope=PROCESS · FINANCE 全 5 ❌ scope=NONE）
**AND** 返回 `updatedAt` + `version=V1.3.9.0` · 与 `sys_global_threshold` 是两条独立权限线（在返回体的 `scope` 字段注释明确"图纸 ACL 非金额 ACL"）

### AC-12.1.2 — 角色 ACL 拦截（draw:preview / draw:print 等 5 类）

**GIVEN** 7 角色已登录 · `DrawAclService.checkPermission(role, drawingId, operation)` 实装
**WHEN** 任意角色调用 `GET /api/v1/drawings/{id}/preview`：
- FINANCE → 40320 `FINANCE 角色无图纸权限`
- SALES 关联订单 → 200 PDF 流
- SALES 不关联订单 → 40321 `该图纸未关联您的订单`
- OPERATOR 工序 P03 关联 → 200 PDF 流
- OPERATOR 工序 P03 不关联 → 40322 `当前工序未关联该图纸`
- ENGINEER → 200 全量 PDF 流
**THEN** 5 类权限（draw:preview / draw:print / draw:download / draw:upload / draw:delete）按矩阵生效 · 拒绝 100% 记录审计日志（`@AuditLog` DENY 切面 · 新增）

### AC-12.1.3 — 关联过滤（5 类：订单/PO/入库/质检/工序）

**GIVEN** 新增 `crm_drawing_link` 关联表（V54 · drawing_id + link_type + link_id 五元组）：
- `link_type=ORDER` → crm_order.id
- `link_type=PO` → purchase_order.id
- `link_type=INBOUND` → wms_inbound.id
- `link_type=INSPECTION` → qc_inspection.id
- `link_type=PROCESS` → work_order_process.id

**WHEN** SALES 用户调用 `GET /api/v1/drawings/{id}/preview`（id=789）：
- 系统查 `crm_drawing_link` WHERE drawing_id=789 AND link_type='ORDER' → 返回 order_ids=[100,101]
- 用户所属 order_ids 包含 100 → 200 PDF 流
- 用户所属 order_ids 不含 100 → 40321
**THEN** 5 类关联过滤生效 · `DrawLinkService.findLinkedDrawings(role, linkType, linkId)` 反向查询也支持（如 SALES 查"我的订单有哪些图纸"）

### AC-12.1.4 — 多仓协同（backend + web-impl + android-impl）

**GIVEN** backend `DrawAclService` 已 ship · web-impl `DrawingViewer.vue` + android-impl `DrawPermissionInterceptor` 待实装
**WHEN** 三仓集成测试：
- **backend**：`POST /api/v1/test/draw-acl-fixtures` 测试夹具（admin）创建 7 角色用户 + 5 类关联数据 · `@WithMockUser` 各角色调用 preview 端点
- **web-impl**：`<DrawingViewer>` 组件 + Element Plus `<el-image-preview>` 包 `drawApi.preview(id)` · 40321/40322 时弹"该图纸未关联您的{订单/PO/...}" · 不暴露 PDF 路径
- **android-impl**：`MaterialBarcodeScan` + `NoOrderPurchase` Fragment 共用 `DrawPermissionInterceptor.handleResponse(code)` · 40320/40321/40322 各自 Toast 文案
**THEN** 三仓权限行为一致 · 灰度发布策略：admin 先开 FINANCE 拒权限 → 再开 SALES 仅关联订单 → 最后开 OPERATOR 仅工序关联（每角色观察 1 天）
