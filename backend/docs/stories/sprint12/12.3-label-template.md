## 1. PM 反馈来源

PM 范蠡 6/14 客户反馈 · `docs/prd-feedback-v1.3.9.md#反馈-3标签模板4种`

### 1.1 4 种标签

| 标签类型 | 前缀 | 色条颜色 | 复用模板 |
|----------|------|----------|----------|
| 工单码 | `GD-` | 蓝色 #1E40AF | 独立 |
| 流转码 | `LZ-` | 绿色 #16A34A | 独立 |
| 设备码 | `SB-` | 灰色 #6B7280 | 复用 GD- 模板（仅色条色不同）|
| 委外单码 | `WW-` | 橙色 #EA580C | 独立 |

**统一规格**：50mm × 30mm

### 1.2 三区布局

```
┌────────────────────────────┐
│ ▌▌▌ GD │  昆山佰泰胜精密加工  │  ← 顶行：左色条 + 右厂名（可配置）
├────────────────────────────┤
│                            │
│       [QR Code 二维码]      │  ← 中央：QR 二维码（纯文本内容）
│                            │
├────────────────────────────┤
│  GD-260614-001              │  ← 下方：明文文字（最多 6 行）
│  工单：WO20260614001         │
│  工序：P03                  │
│  数量：50                   │
│  日期：2026-06-14           │
└────────────────────────────┘
```

### 1.3 二维码内容（纯文本 · APP 扫码壳按前缀自动路由）

- `LZ-260613-001-P03` → 流转码
- `GD-260614-001` → 工单码
- `WW-260614-001` → 委外单码
- `SB-260614-001` → 设备码
- `WL-...` → 物料码（V1.3.8 已有 · 保持）

### 1.4 厂名可配置

admin 在系统设置（复用 1.3 `sys_dict`）配置 · 默认"昆山佰泰胜精密加工"

## 4. Acceptance Criteria

### AC-12.3.1 — 4 模板元数据（GD/LZ/SB/WL/WW）

**GIVEN** V56 label_template 表已 seed 5 行（GD/LZ/SB/WW/WL · WL 为 V1.3.8 既有物料码）
**WHEN** 任意已登录用户调用 `GET /api/v1/labels/templates`
**THEN** 返回 5 模板数组：
- `code=GD` `name=工单码` `prefix=GD-` `colorBarHex=#1E40AF` `reuseBaseTemplate=null`
- `code=LZ` `name=流转码` `prefix=LZ-` `colorBarHex=#16A34A` `reuseBaseTemplate=null`
- `code=SB` `name=设备码` `prefix=SB-` `colorBarHex=#6B7280` `reuseBaseTemplate=GD`
- `code=WW` `name=委外单码` `prefix=WW-` `colorBarHex=#EA580C` `reuseBaseTemplate=null`
- `code=WL` `name=物料码` `prefix=WL-` `colorBarHex=#000000` `reuseBaseTemplate=null`（V1.3.8 既有 · 本 Story 兼容）
**AND** 顶层 `companyName` 字段读取 `sys_dict` `type=COMPANY_NAME` · 默认"昆山佰泰胜精密加工"

### AC-12.3.2 — 渲染 PNG/PDF 预览（三区布局）

**GIVEN** 调用方传入 templateCode + qrContent + lines（≤6 行）
**WHEN** 调用 `POST /api/v1/labels/preview` {templateCode=GD, qrContent="GD-260614-001", lines=["GD-260614-001","工单：WO20260614001","工序：P03","数量：50","日期：2026-06-14"], format=PNG}
**THEN** 返回 50mm×30mm PNG 二进制流：
- 顶行：左侧蓝色 #1E40AF 色条（宽 8mm）+ 右侧厂名"昆山佰泰胜精密加工"
- 中央：QR 二维码（300×300 像素 · ZXing 渲染 · 内容 "GD-260614-001"）
- 下方：5 行明文 · 字号 8pt · 黑色
**AND** format=PDF 时 · 返回 PDF 流（OpenPDF · 单标签 · 50mm×30mm 页面）
**AND** templateCode=SB 时 · 复用 GD 模板 · 仅 colorBarHex=#6B7280 不同 · 其余完全一致

### AC-12.3.3 — QR 内容路由（按前缀 · 与 1.4 扫码壳一致）

**GIVEN** 5 类 QR 内容纯文本 · APP 扫码壳按前缀分发
**WHEN** 生成预览时 QR 内容严格保持传入字符串（如 `LZ-260613-001-P03`）：
- 扫码壳扫描后 · 按前缀解析：`LZ-` → 流转码 → 路由到 1.16 扫码开工报工过站接口
- `GD-` → 工单码 → 路由到工单详情
- `WW-` → 委外单码 → 路由到 1.18 委外下单详情
- `SB-` → 设备码 → 路由到设备台账（V1.3.7 1.41 HR 模块 · V1.3.9 设备管理扩展）
- `WL-` → 物料码 → 路由到 1.50 入库扫码
**THEN** 12.3 渲染的 QR 与扫码壳路由 1:1 一致 · 不做编码/解码 · 仅纯文本

### AC-12.3.4 — 多仓预览（web-impl + android-impl）

**GIVEN** backend `LabelPreviewService` 已 ship · 返回 PNG/PDF 二进制流
**WHEN** 三仓预览组件实装：
- **web-impl**：`<LabelPreview>` Vue 组件 · `props: { templateCode, qrContent, lines }` · 内嵌 `<el-image :src="previewUrl" />` · previewUrl 来自 `labelsApi.preview(...)`
- **android-impl**：`LabelPreviewFragment` · `MaterialBarcodeScan` 扫到 GD-/LZ- 等前缀后跳转至此 Fragment · 显示 PNG（Glide 加载）
- **backend**：`/preview` 端点响应 `Content-Disposition: inline; filename="GD-260614-001.png"`
**THEN** 三仓 UI 显示一致（色条 + 厂名 + QR + 明文 5-6 行）· web-impl 预览加载 < 500ms · android-impl Fragment 启动 < 1s

## 7. 风险

| 风险 | 等级 | 缓解 |
|------|------|------|
| QR 二维码内容长度限制（QR v3-M 最多 200 字符）· 明文 6 行若超长需切换 v4 | 🟡 中 | 校验：qrContent.length ≤ 200 · lines[i].length ≤ 50 · 超长抛 422 |
| 色条复用 · SB- 复用 GD- 模板仅色条色不同 | 🟢 低 | 代码层抽 `BaseLabelTemplate.render(BarColor)` · V56 已设计 reuse_base_template 字段 |
| APP 扫码壳新增 SB-/WW- 两类需配合发布 | 🟢 低 | 1.4 已设计为可扩展（按前缀分发）· 仅新增 2 类枚举 |

| OpenPDF 中文渲染（厂名含中文）· 默认字体不支持中文 | 🟢 低 | V1.3.7 1.5/1.6 已 ship 中文字体（NotoSansCJK）· 直接复用 |
