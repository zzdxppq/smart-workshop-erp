## 1. PM 反馈来源

PM 范蠡 6/14 客户反馈 · `docs/prd-feedback-v1.3.9.md#反馈-4双模式打印`

### 1.1 模式一：标签打印机（ZPL/TSPL 直连）

1. 系统将标签数据填入 ZPL 模板（`LabelRenderService.renderZpl(BarColor, qrText, lines)` → ZPL II 字符串）
2. 后端通过 `Socket(ip, port=9100)` 连接打印机
3. 直接发送 ZPL 指令（`OutputStream.write(zpl.getBytes(UTF_8))`）
4. 打印机在 50mm×30mm 标签纸上输出
5. 失败 → 弹错误提示"请检查打印机连接"（错误码 40950 `PRINT_CONNECTION_FAILED`）

### 1.2 模式二：普通打印机（A4 排版 PDF）

1. 后端 `LabelRenderService.renderPdfA4(templates, count)` → OpenPDF · 一页排 30 标签（3 列 × 10 行）
2. 前端 `window.print()` · 用户选普通激光/喷墨 + A4 纸
3. 打印后人工裁剪

### 1.3 打印留痕表（sys_print_log · 与 8.3 sys_workflow_event 同 sys_ 命名空间）

字段：
- `id` · `operator_user_id` · `printed_at`
- `code_type[GD/LZ/SB/WW/WL]` · `code_value`
- `count`（份数 · 1-100）
- `printer_ip` · `printer_name`
- `mode[ZPL_DIRECT/PDF_BROWSER]`
- `status[SUCCESS/FAILED/PENDING]`
- `error_msg` · `remark`

### 1.4 历史/补打

- 打印管理页面查询历史（多维过滤 + 分页）
- 选记录一键补打（同模式或换模式）

## 4. Acceptance Criteria

### AC-12.4.1 — 模式一 ZPL/TSPL 直连

**GIVEN** sys_printer 已有 LABEL 类型 + protocol=ZPL/TSPL + status=ONLINE
**WHEN** 调用 `POST /api/v1/print/labels/zpl` {templateCode=GD, qrContent="GD-260614-001", lines=[...], printerId=5, count=1}
**THEN** 后端：
1. `LabelRenderService.renderZpl(colorBarHex=#1E40AF, qrText="GD-260614-001", lines=[...])` → ZPL II 字符串
2. `Socket("192.168.1.100", 9100)` 连接 · 2 秒 connect 超时
3. `OutputStream.write(zpl.getBytes(UTF_8))` 发送
4. 关闭 Socket · 写 sys_print_log status=SUCCESS · 返回 printLogId + bytesSent + latencyMs

**AND** 失败场景：
- IP 不通 → 502 错误码 40951 `请检查 IP/端口/网线` · sys_print_log status=FAILED · error_msg 记录
- 协议不匹配（NORMAL 类型调此端点）→ 409 错误码 40952 `NORMAL 类型打印机不支持 ZPL`
- count > 100 → 422

### AC-12.4.2 — 模式二 A4 PDF 浏览器

**GIVEN** 用户调用 `POST /api/v1/print/labels/pdf-a4` {items: [30 个标签项]}
**WHEN** 后端 `LabelRenderService.renderPdfA4(items)`：
1. 30 标签按 3 列 × 10 行排版（A4 纸 210mm×297mm · 标签 50mm×30mm · 边距 5mm）
2. OpenPDF 生成 PDF · 单页 · 返回 application/pdf 流
3. 同步写 sys_print_log · mode=PDF_BROWSER · printer_name="普通浏览器" · status=SUCCESS
4. 响应头 `X-Print-Log-Id` 返回 sys_print_log.id
**THEN** 前端：
- web-impl：`window.print()` 触发浏览器打印对话框 · 用户选激光/喷墨 + A4 纸 · 打印后人工裁剪
- android-impl：Android PrintManager API 触发 PDF 打印 · 选系统打印机
**AND** items 超过 30 → 422 `单次最多 30 标签/页（A4 排版）`

### AC-12.4.3 — 打印留痕（sys_print_log）

**GIVEN** V57 sys_print_log 表已建 · 12.1/12.4 共表
**WHEN** 任意打印动作（模式一 / 模式二 / 12.1 图纸打印凭据 / 补打）执行：
**THEN** sys_print_log 写入完整字段：
- id（雪花算法 · 与 8.3 sys_workflow_event 同生成器）
- operator_user_id（当前登录用户）
- printed_at（NOW()）
- code_type（GD/LZ/SB/WW/WL · 或 DRAWING）
- code_value（如 "GD-260614-001"）
- count（1-100）
- printer_ip（如 "192.168.1.100" · 模式二为 NULL）
- printer_name（如 "Zebra-1" · 模式二为 "普通浏览器"）
- mode（ZPL_DIRECT / PDF_BROWSER）
- status（SUCCESS / FAILED / PENDING）
- error_msg（FAILED 时记录）
- reference_log_id（补打时指向原始日志）

**AND** `@AuditLog` AFTER_COMMIT 切面 100% 记录 · 模块名 `SYS_PRINT_LOG`

### AC-12.4.4 — 历史与补打

**GIVEN** sys_print_log 已有 N 条记录 · admin/操作员 角色访问 `/admin/print-logs`
**WHEN** 查询与补打：
- `GET /api/v1/print/logs?codeType=GD&dateFrom=2026-06-01&dateTo=2026-06-14` → 分页列表
- 列表列：时间 / 操作人 / 类型 / 编码 / 份数 / 打印机 / 模式 / 状态 / 操作（补打）
- 点击"补打" → 弹选择：同模式 / 换模式 → 调 `/logs/{id}/replay`
- 补打新写 sys_print_log · `reference_log_id` 指向原始 · status=SUCCESS

**AND** 原始 status=FAILED 时 · 补打按钮 disabled + tooltip "FAILED 状态不可补打 · 请重新创建打印任务"
**AND** 补打统计：reference_log_id 计数 ≥ 1 → 标记"补打"红色 tag

### AC-12.4.5 — 多仓打印入口（web-impl + android-impl）

**GIVEN** 12.2 打印机配置已 ship · 12.3 标签模板已 ship · 12.1 图纸权限已 ship
**WHEN** 三仓打印入口实装：
- **web-impl**：`<PrintButton>` Vue 组件（V1.3.8 10.1/10.5 已 ship · 扩展）
  - props: { templateCode, qrContent, lines, drawingId? }
  - 点击 → 调 12.2 `/printers/available?type=LABEL` → 1 台直打 / 多台弹 `<el-dialog>` 选 / 未配弹"管理员先配置"
  - 选中后调 12.4 `/print/labels/zpl` 或 `/print/labels/pdf-a4`
- **android-impl**：`PrintDialogFragment`（V1.3.8 已 ship 8 角色 · 扩展）
  - 触发：扫码后 / 列表行"打印"按钮
  - 同款 1/多/未 逻辑 · 用 `<AlertDialog>` 弹选
  - 调 12.4 端点 · 模式一成功后 Toast "已发送到 Zebra-1"
- **图纸打印入口**（12.1 关联）：web-impl `<DrawingViewer>` 工具栏"打印"按钮
  - 前置 ACL · `draw:print` 权限 + 关联过滤
  - 调 12.1 `/drawings/{id}/print-ticket` 创建凭据 · 调 12.4 `/print/logs/{id}/replay` 实际打印

**THEN** 三仓 UI 一致 · web-impl 弹窗与 12.2 sys_dict 字典 admin 页风格统一 · android-impl Toast 文案按角色本地化

```sql
-- V57__sys_print_log.sql (V1.3.9 Sprint 12 · 12.4 打印留痕 · 与 12.1 共表)

CREATE TABLE sys_print_log (
  id                  BIGSERIAL PRIMARY KEY,
  operator_user_id    BIGINT NOT NULL,
  printed_at          TIMESTAMP NOT NULL DEFAULT NOW(),
  code_type           VARCHAR(20) NOT NULL,                          -- GD / LZ / SB / WW / WL / DRAWING
  code_value          VARCHAR(200) NOT NULL,                         -- GD-260614-001 等
  count               INTEGER NOT NULL DEFAULT 1,                    -- 1-100
  printer_ip          VARCHAR(45) NULL,                              -- 模式二为 NULL
  printer_name        VARCHAR(100) NULL,                             -- 模式二为 "普通浏览器"
  mode                VARCHAR(20) NOT NULL,                          -- ZPL_DIRECT / PDF_BROWSER
  status              VARCHAR(20) NOT NULL DEFAULT 'PENDING',       -- SUCCESS / FAILED / PENDING
  error_msg           VARCHAR(500) NULL,
  reference_log_id    BIGINT NULL,                                   -- 补打时指向原始
  remark              VARCHAR(200) NULL,
  created_at          TIMESTAMP NOT NULL DEFAULT NOW(),
  CONSTRAINT fk_print_log_user FOREIGN KEY (operator_user_id) REFERENCES sys_user(id),
  CONSTRAINT fk_print_log_ref  FOREIGN KEY (reference_log_id)  REFERENCES sys_print_log(id),
  CONSTRAINT chk_print_code_type CHECK (code_type IN ('GD','LZ','SB','WW','WL','DRAWING')),
  CONSTRAINT chk_print_mode     CHECK (mode IN ('ZPL_DIRECT','PDF_BROWSER')),
  CONSTRAINT chk_print_status   CHECK (status IN ('SUCCESS','FAILED','PENDING'))
);
CREATE INDEX idx_print_log_user       ON sys_print_log(operator_user_id);
CREATE INDEX idx_print_log_code       ON sys_print_log(code_type, code_value);
CREATE INDEX idx_print_log_printed_at ON sys_print_log(printed_at);
CREATE INDEX idx_print_log_status     ON sys_print_log(status);
CREATE INDEX idx_print_log_reference  ON sys_print_log(reference_log_id) WHERE reference_log_id IS NOT NULL;

-- 注释
COMMENT ON TABLE  sys_print_log            IS 'V1.3.9 新增 · 打印留痕 · 与 sys_workflow_event 同 sys_ 命名空间';
COMMENT ON COLUMN sys_print_log.code_type  IS 'GD/LZ/SB/WW/WL 标签码 · DRAWING 图纸（12.1 共表）';
COMMENT ON COLUMN sys_print_log.mode      IS 'ZPL_DIRECT=模式一直连 · PDF_BROWSER=模式二浏览器';
COMMENT ON COLUMN sys_print_log.printer_name IS '模式一为 sys_printer.name · 模式二固定为"普通浏览器"';
COMMENT ON COLUMN sys_print_log.reference_log_id IS '补打时指向原始 sys_print_log.id · 形成补打链';
```

## 7. 风险

| 风险 | 等级 | 缓解 |
|------|------|------|
| 🔴 **浏览器无法跨域调打印机 Socket → 必须后端 Socket 代理** | 🟢 低 | 架构明确：web-impl 仅调 REST · Socket 仅后端 `ZplSocketClient` |
| 客户机房打印机 IP 变化 · admin 需手动改配置（暂不支持 DHCP）| 🟡 中 | V1.4 backlog · mDNS 自动发现 |
| ZPL 与 TSPL 指令集部分字段不兼容 | 🟢 低 | `ProtocolAdapter` 抽象（`renderZpl()` vs `renderTspl()`）· V57 protocol 字段已存 |
| 一键补打若历史记录被归档 → 仅 SUCCESS 可重放 | 🟢 低 | 业务规则明确 · UI 屏蔽 FAILED |
| A4 PDF 浏览器打印依赖浏览器实现 · 不同浏览器弹窗样式差异 | 🟢 低 | 仅依赖 `window.print()` 标准 API · 不深度定制 |
| 高并发打印（100+ QPS）Socket 连接池性能 | 🟢 低 | V1.3.9 客户规模低 · V1.4 引入连接池 |
| sys_print_log 数据增长（每日 1000+ 条 6 个月 = 18 万行）| 🟢 低 | 索引已建（printed_at + code_type + operator_user_id）· 月度归档 V1.4 |
