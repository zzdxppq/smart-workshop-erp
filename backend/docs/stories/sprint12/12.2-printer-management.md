## 1. PM 反馈来源

PM 范蠡 6/14 客户反馈 · `docs/prd-feedback-v1.3.9.md#反馈-2打印机管理`

### 1.1 两类打印机

| 类型 | 用途 | 协议 | 代表型号 |
|------|------|------|---------|
| **普通 (NORMAL)** | 报价单 PDF / 对账单 PDF / 图纸 A4 | OS 打印队列（无需 IP 配置） | 任意 HP/Canon/Brother |
| **标签 (LABEL)** | 50mm×30mm 不干胶标签 | ZPL（Zebra）/ TSPL（TSC） · Socket 9100 | 得力 DL-888B · 斑马 ZD420 · TSC TTP-244 Pro |

**明确不支持**：ESC/POS 票据打印机（文档说明）

### 1.2 配置项

- `name`（必填 · 唯一 · 1-50 字符）
- `type`（NORMAL / LABEL）
- `ip`（LABEL 必填 · IPv4 格式 · NORMAL 可空）
- `port`（默认 9100 · 范围 1-65535）
- `protocol`（NORMAL=`PDF_BROWSER` / LABEL=`ZPL` 或 `TSPL` · 设备型号决定）
- `modelSuggestion`（`DELI_DL888B` / `ZEBRA_ZD420` / `TSC_TTP244PRO` / `OTHER`）
- `enabled`（1/0）

### 1.3 用户点击打印逻辑（在 12.4 共用）

- **1 台同类型**：直接打印
- **多台**：弹选择框（web-impl `<el-dialog>` / android-impl `<AlertDialog>`）
- **未配**：前端拦截 · 提示"管理员先配置"

## 4. Acceptance Criteria

### AC-12.2.1 — 配置 CRUD（admin only）

**GIVEN** admin 已登录 · V1.3.7 sys_global_threshold 双轨 ACL 模式已 ship
**WHEN** admin 调用下列端点：
- (a) `POST /api/v1/printers` 创建（NORMAL · 缺 ip 允许）→ 201 返回 id
- (b) `POST /api/v1/printers` 创建（LABEL · 缺 ip）→ 422 `LABEL 类型必填 ip`
- (c) `POST /api/v1/printers` 创建（name 重复）→ 409 `打印机名称已存在`
- (d) `GET /api/v1/printers?type=LABEL&status=ONLINE` → 200 分页列表
- (e) `PUT /api/v1/printers/{id}` 修改 port → 200
- (f) `DELETE /api/v1/printers/{id}` 删除（无关联 sys_print_log）→ 204
- (g) `DELETE /api/v1/printers/{id}` 删除（有关联 sys_print_log）→ 409 · 提示"改为 enabled=0"
**THEN** 7 子场景全部通过 · `@PreAuthorize("hasAuthority('sys:admin')")` 拦截非 admin · 100% 审计日志

### AC-12.2.2 — 心跳检测（60s 调度 + TCP Socket 探活）

**GIVEN** V55 sys_printer 表已建 + `@EnableScheduling` 已启用 + `@Scheduled(fixedRate = 60000)` 实装
**WHEN** 调度器每 60 秒执行：
- 对每条 `enabled=1` 的 LABEL 打印机 · 创建 `Socket(ip, port)` · 2 秒 connect 超时
- 连接成功 → `status=ONLINE` + `lastHeartbeatAt=NOW()`
- 连接失败 → 累计 `fail_count++` · 达 2 次连续失败 → `status=OFFLINE`（容差防误标）
- 心跳结束后 NORMAL 类型保持 `status=UNKNOWN`（OS 打印队列无 IP · 不探活）
**THEN** 心跳调度稳定 · sys_printer 表状态字段实时更新 · admin UI 列表自动刷新（web-impl 每 30s 轮询）

### AC-12.2.3 — admin UI 页（与 sys_dict 同款风格）

**GIVEN** V1.3.7 Story 1.3 sys_dict 字典 admin 页已 ship（增/改/删/启停 · Element Plus Table）
**WHEN** 访问 `/admin/printers` 路由：
- 顶部筛选：类型下拉 + 状态下拉 + 启停
- 表格列：名称 / 类型 / IP / 端口 / 协议 / 型号 / 状态（红/绿 tag）/ 最后心跳 / 操作（测试/编辑/删除/启停）
- 新增按钮 → `<el-dialog>` 表单（与 sys_dict 同款 · 含字段必填校验）
- 测试按钮 → 调用 `/test` 端点 · loading 2 秒后弹"成功（延迟 23ms · 协议 ZPL）"或"失败 · 请检查 IP/端口/网线"
- 心跳显示红/绿：ONLINE 绿色 tag · OFFLINE 红色 tag
**THEN** UI 与 sys_dict 字典 admin 页风格一致 · 操作流畅 · 移动端响应式（Element Plus 内置）

CREATE TABLE sys_printer (

## 7. 风险

| 风险 | 等级 | 缓解 |
|------|------|------|
| 心跳调度若客户机房网络瞬断（如 1 分钟）会误标 OFFLINE | 🟢 低 | 容差策略：连续失败 2 次才标 OFFLINE · 单次失败仅 fail_count++ |
| web-impl 跨域调用后端 9100 不可能（CORS + 浏览器不允许 Socket） | 🟢 低 | 架构明确：Socket 必须在后端代理 · web-impl 仅调 `/test` 与 `/available` REST 端点 |
| ESC/POS 票据打印机客户已购 · 但 V1.3.9 不支持 | 🟡 中 | 文档明确告知 · PM 范蠡沟通客户 · V1.4 backlog（待评估 ROI）|
| 多台 ZPL 型号协议差异（ZD420 与 TTP-244 Pro ZPL 子集不同） | 🟢 低 | V55 仅记录 `protocol` 字段 · 实际指令生成由 12.4 `ProtocolAdapter` 处理 |
| 心跳调度在高并发场景（数百台打印机）性能 | 🟢 低 | `@Scheduled` 单线程 · V1.3.9 客户规模 5-10 台 · V1.4 可改为线程池 |
