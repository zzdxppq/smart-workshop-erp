## 1. V1.3.7 → V1.3.8 升级动因（PM 范蠡反馈 #3）

PM 反馈：当前 4 阈值路由（金额 / 品类 / 供应商 / 紧急度）由 PURCHASER 主管审批，权限过宽。无订单采购涉及 1 万以上紧急补料，需独立 `PROCUREMENT_MANAGER` 角色把关。

**V1.3.8 方案**：新增 `PROCUREMENT_MANAGER` 角色，独立于 PURCHASER；4 阈值路由细化（金额 > 1 万 → 采购主管审批）。

### 1.1 4 阈值路由细化（**precheck 校正**）

> ⚠️ **2026-06-13 precheck 校正**：原 Story 假设阈值存 `sys_workflow_config` 表，precheck grep 后纠正：V1.3.7 实际生效的是 `sys_workflow_node` 表（V2__workflow_split.sql 创建），由 `WorkflowApprovalRouter` 按 `node_index ASC` + `amount.compareTo(threshold) <= 0` 决策。**仅支持金额阈值，不支持品类/紧急度分支**。详见 `docs/dev/logs/4.2-precheck.log`。

| 阈值 | V1.3.7 路由 | V1.3.8 路由（仅金额） |
|------|-------------|----------------------|
| 金额 ≤ 10,000 | 既有 DEPT_MANAGER 节点 | 不变 |
| **金额 10,000 ~ 50,000** | 既有 DEPT_MANAGER 节点 | **追加 node_index=5 PROCUREMENT_MANAGER**（threshold=50000） |
| **金额 > 50,000** | 既有 GM 节点 | **追加 node_index=6 PROCUREMENT_MANAGER**（threshold=NULL 兜底） |
| 品类 = 刀具/化学品 | ❌ V1.3.7 不支持 | 🟡 **V1.3.8 本期也不支持**（应用层 if 判断，留 V1.3.9 backlog） |
| 紧急度 = URGENT | ❌ V1.3.7 不支持 | 🟡 **V1.3.8 本期也不支持**（同上） |
| 供应商 = 黑名单 | ❌ V1.3.7 不支持 | 🟡 同上 |

## 2. Acceptance Criteria

### AC-4.2.1 PROCUREMENT_MANAGER 角色

**Given** 新增角色 `PROCUREMENT_MANAGER`
**When** 系统初始化（V52 迁移 + SysRoleInit 触发）
**Then** 新角色写入 `sys_role` 表，role_code=`PROCUREMENT_MANAGER`
**And** 权限集：审批金额 1 万 ~ 5 万 PO + 紧急补料 + 刀具/化学品品类
**And** 与 `PURCHASER` 角色互斥（同一用户不能同时拥有）
**And** JWT `me` 接口返回该角色时，前端菜单显示「采购审批」入口

### AC-4.2.2 4 阈值路由细化

**Given** PO 提交审批
**When** 调用 `POST /api/v1/approval/route-preview` 传入 `{amount: 30000, category: 'TOOL', supplier_status: 'NORMAL', urgency: 'NORMAL'}`
**Then** 返回路由：
```json
{
  "code": 0,
  "data": {
    "route": ["PROCUREMENT_MANAGER"],
    "matched_thresholds": ["AMOUNT_10K_50K", "CATEGORY_TOOL"],
    "estimated_signers": 1
  }
}
```
**And** 路由变更触发 `@AuditLog` AFTER_COMMIT 写 `sys_change_log`
**And** 1.2 既有 4 阈值路由保留兼容（旧 `DEPT_MANAGER` 路由仍可工作）

## 3. 数据库迁移（V52）

### V52__procurement_manager_role.sql

```sql
INSERT INTO sys_role (role_code, role_name, sort_order, is_active, created_at) VALUES
  ('PROCUREMENT_MANAGER', '采购主管', 5, 1, NOW());

-- 权限集
INSERT INTO sys_role_permission (role_id, permission_code, created_at)
SELECT r.id, p.code, NOW()
FROM sys_role r, sys_permission p
WHERE r.role_code = 'PROCUREMENT_MANAGER'
  AND p.code IN (
    'purchase:approval:read',
    'purchase:approval:approve',
    'purchase:approval:reject',
    'purchase:no-order:create'
  );

-- 路由配置：细化 4 阈值
INSERT INTO sys_workflow_config (workflow_code, threshold_type, threshold_value, approver_role, sort_order, is_active, created_at) VALUES
  ('PO_APPROVAL', 'AMOUNT', '10000-50000', 'PROCUREMENT_MANAGER', 3, 1, NOW()),
  ('PO_APPROVAL', 'AMOUNT', '>50000',       'GM+PROCUREMENT_MANAGER', 4, 1, NOW()),
  ('PO_APPROVAL', 'CATEGORY', 'TOOL',       'PROCUREMENT_MANAGER', 5, 1, NOW()),
  ('PO_APPROVAL', 'CATEGORY', 'CHEMICAL',   'PROCUREMENT_MANAGER', 6, 1, NOW()),
  ('PO_APPROVAL', 'URGENCY',  'URGENT+AMOUNT>10000', 'PROCUREMENT_MANAGER', 7, 1, NOW());
```

## 4. 端点契约

### 4.1 `POST /api/v1/approval/route-preview`（路由预览 · 无副作用）

**Request**:
```json
{
  "amount": 30000,
  "category": "TOOL",
  "supplier_status": "NORMAL",
  "urgency": "NORMAL"
}
```

**Response 200**:
```json
{
  "code": 0,
  "data": {
    "route": ["PROCUREMENT_MANAGER"],
    "matched_thresholds": ["AMOUNT_10K_50K", "CATEGORY_TOOL"],
    "estimated_signers": 1,
    "compatible_legacy_route": ["DEPT_MANAGER"]
  }
}
```

      "purchase:approval:read",
      "purchase:approval:approve",
      "purchase:approval:reject",
