## Epic Context

- **WHY（来自 PRD）**：CNC 加工厂的工程师在图纸上标注的"工艺要求/公差/技术注释"是工艺路线的源头。工程转化是把图纸的"几何信息"翻译成"工艺信息"，是连接设计与生产的桥梁。
- **业务红线**（V1.3.7）：
  ① 标注必须挂载图纸版本（防 v1→v2 标注丢失，V1.3.7 P1 修补 1）
  ② 工程转化前必须校验图纸状态 RELEASED（V1.3.7 P1 修补 2）
  ③ PDF 预览水印（工程师姓名 + 时间戳 + 公司 LOGO，V1.3.7 P2 修补 3 闭环）
  ④ 工程转化 → BOM 单号 `BOM{yyyyMMdd}{seq:4}`（100 并发不重复）
  ⑤ 工艺路线 5 段成本聚合（V1.3.4 闭环 hook，1.8 触发 BOM 自动生成）
- **解耦约束**：工程转化服务是 `erp_business` schema 业务模块（与 `erp_platform` 工作流中台解耦），通过 **Feign 客户端** 调用下游 BOM 服务（Epic 3.3）；下游工艺库 / 工序引用 1.8 的标注数据

## Acceptance Criteria

> 渲染源：PRD `docs/prd/epic-3-图纸与物料.yaml#3.2` + `docs/prd/2-需求.md#FR-3-2`（FR-3-2-1/2/3）

### AC-3.2.1 — 图纸标注（FR-3-2-1）

**GIVEN** 图纸已就位（含 DRAFT/RELEASED/ARCHIVED/OBSOLETE 4 状态），Story 1.7 图纸版本管理已实装
**WHEN** 工程师或工艺师执行：
- (a) `POST /api/v1/drawings/{id}/annotations` 新增标注（请求体含 `version` 必填 + `type` 4 选 1：DIMENSION/TOLERANCE/PROCESS_REQ/TECH_NOTE + `content` + `color` + `x/y/width/height` 坐标 + `priority`）
- (b) `GET /api/v1/drawings/{id}/annotations?version=v1` 查询标注列表（按版本过滤）
- (c) `PUT /api/v1/drawings/{id}/annotations/{aid}` 修改标注（仅 `DRAFT/RELEASED` 状态可改）
- (d) `DELETE /api/v1/drawings/{id}/annotations/{aid}` 删除标注（仅 `DRAFT` 状态可删）

**THEN** 4 类型标注 + 坐标定位 + 颜色分类 + 优先级排序 + 挂载图纸版本；标注变更 100% 留痕（`@AuditLog` AFTER_COMMIT）

**AND** P1 修补 1：标注必须挂载图纸版本（防 v1→v2 标注丢失）；新增 v2 时，v1 标注自动归档为 `version=v1`，查询时按 `?version=` 过滤

### AC-3.2.2 — 工程转化（FR-3-2-2）

**GIVEN** 图纸已 `RELEASED` 状态，工艺路线 5 段成本聚合已就位
**WHEN** 工程师执行：
- (a) `POST /api/v1/drawings/{id}/convert` 触发工程转化（请求体含 `bomType` 默认 `STANDARD` + `targetQty` 默认 1 + `comment`）
- (b) 后端流程：① 校验图纸 `RELEASED` 状态（40904 守卫，P1 修补 2）② 调 `DrawingPdfExportService.aggregateProcessRouteCost` 求 BOM 总成本 ③ 调 `DocNoGenerator.nextBomNo()` 生成 `BOM{yyyyMMdd}{seq:4}` ④ 调下游 BOM 服务（Feign 客户端）创建 BOM 主表 + 5 段工艺路线子表 ⑤ 写 `eng_conversion_history` 留痕

**THEN** 工程转化成功后图纸状态 `RELEASED → CONVERTED`（新增第 5 状态，V1.3.7 P1 修补 2 配套）；BOM 单号实时生成（DB 唯一索引守）；下游 BOM 接收 5 段成本数据（120+80+200+150+90=640 等）

**AND** 工程转化失败回滚：图纸状态保持 `RELEASED`；记录 `eng_conversion_history.status=FAILED` + `error_message`

### AC-3.2.3 — PDF 预览水印（FR-3-2-3 · V1.3.7 P2 修补 3 闭环）

**GIVEN** 图纸已 `RELEASED` 状态，Story 1.7 PDF 导出已实装（简化文本格式）
**WHEN** 工程师或系统执行 `GET /api/v1/drawings/export/{id}?format=pdf`：
- (a) PDF 第一页加水印：工程师姓名 + 当前时间戳 + 公司 LOGO（`昆山佰泰胜机械有限公司`）
- (b) 水印字体：Arial Unicode 18pt，旋转 30°，半透明（alpha=0.3）
- (c) 工艺路线 5 段渲染（步骤名 + 单价 + 总成本）复用 Story 1.7

**THEN** PDF 导出含水印 + 工艺路线 + 签字扫描件解密嵌入（AES-256-GCM）+ 1h 缓存命中
