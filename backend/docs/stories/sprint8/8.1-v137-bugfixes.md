## 2. Acceptance Criteria

### AC-8.1.1 类型 A：BigDecimal 精度修复

**Given** ProcessService / ProcessIntegrationTest 期望 `new BigDecimal("1500.00")`
**When** 测试代码用 `assertEquals(1500.00, expected)` 或 `BigDecimal.equals`
**Then** 修复后断言改用 `compareTo`（scale-insensitive 比较）
**And** ProcessServiceTest.cost_aggregation_5_segments PASS
**And** ProcessIntegrationTest.crossmodule_5_segment_aggregator_for_1_9 PASS

### AC-8.1.2 类型 B：HashMap/List 契约一致

**Given** ConversionService 实际返回 `Map<String, Object>` 但测试期望 `List`
**When** 修复 Service 返回类型 OR 测试断言类型（决策：改 Service 返回 List<Map>）
**Then** 12 个 ConversionService / ConversionIntegrationTest 测例 PASS
**And** 5 端点契约不变（OpenAPI 兼容）

---

## 3. 修复策略

### 3.1 类型 A：测试侧修复（推荐）

**理由**：业务代码 `new BigDecimal("1500")` 是合理的（用户输入），不应该改业务。改测试断言为 `assertEquals(0, expected.compareTo(actual))`。

### 3.2 类型 B：服务侧修复

**理由**：服务返回 `Map<String, Object>` 是历史 bug（V1.3.7 1.8 工程转化的 list 实际是 batch 列表）。改 Service 返回 `List<Map<String, Object>>` 更符合语义。
