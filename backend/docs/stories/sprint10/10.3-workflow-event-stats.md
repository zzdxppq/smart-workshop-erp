## 2. 端点契约

### GET /api/v1/workflow/events/stats

**Query 参数**：
- `workflow_code`（必填，如 PO_APPROVAL / QUOTE_APPROVAL / OUTSOURCE_APPROVAL）
- `approver_role`（可选，如 PROCUREMENT_MANAGER / DEPT_MANAGER / GM / PURCHASER）
- `start_date`（ISO 日期，可选，默认 30 天前）
- `end_date`（ISO 日期，可选，默认今天）

**Response 200**：
```json
{
  "code": 0,
  "data": {
    "totalCount": 23,
    "byEventType": {
      "CREATED": 12,
      "APPROVED": 8,
      "REJECTED": 2,
      "DELEGATED": 1
    },
    "byApproverRole": {
      "PROCUREMENT_MANAGER": 15,
      "GM": 5,
      "GM+PROCUREMENT_MANAGER": 3
    },
    "period": {
      "startDate": "2026-05-14",
      "endDate": "2026-06-13"
    }
  }
}
```

**权限**：仅 GM + ADMIN（`@PreAuthorize("hasAnyRole('GM', 'ADMIN')")`）

**错误码**：
- `40001` workflow_code 缺失
- `40001` start_date > end_date
- `40401` workflow_code 未知
- `40304` 非 GM/ADMIN（与 1.40 工序分配职责越权同码体系）

## 3. 数据源与索引

### 3.1 表

```sql
-- V53 Sprint 8.3 迁移
CREATE TABLE sys_workflow_event (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  workflow_code VARCHAR(64) NOT NULL,
  biz_id BIGINT NOT NULL,
  biz_no VARCHAR(64),
  event_type VARCHAR(32) NOT NULL,    -- CREATED/APPROVED/REJECTED/DELEGATED/PREVIEWED/AWARDED
  approver_role VARCHAR(64),
  from_node VARCHAR(64),
  to_node VARCHAR(64),
  comment VARCHAR(512),
  matched_node_index INT,
  matched_threshold VARCHAR(64),
  operator_id BIGINT,
  operator_name VARCHAR(64),
  created_at DATETIME NOT NULL,
  INDEX idx_wfe_code_role_time (workflow_code, approver_role, created_at),
  INDEX idx_wfe_biz (workflow_code, biz_id)
);
```

### 3.2 聚合 SQL

```sql
-- byEventType 分组
SELECT event_type, COUNT(*) AS cnt
FROM sys_workflow_event
WHERE workflow_code = ? AND created_at BETWEEN ? AND ?
GROUP BY event_type;

-- byApproverRole 分组
SELECT approver_role, COUNT(*) AS cnt
FROM sys_workflow_event
WHERE workflow_code = ? 
  AND approver_role IS NOT NULL
  AND created_at BETWEEN ? AND ?
GROUP BY approver_role;

-- 进一步按 approver_role 过滤时，WHERE 加 approver_role = ?
```

**性能**：复合索引 `idx_wfe_code_role_time` 覆盖 WHERE + GROUP BY · 预计 30 天数据 1000 行内 < 50ms

## 4. Acceptance Criteria

### AC-10.3.1 按 workflow_code 聚合

测例：mock 5 条 CREATED + 3 条 APPROVED + 2 条 REJECTED 事件（PO_APPROVAL），验证返回 byEventType 正确分组（5/3/2 = 10 · 与 9.1 NO_ORDER 触发一致）。

### AC-10.3.2 按 approver_role 过滤

测例：approver_role=PROCUREMENT_MANAGER 时只返回该角色的事件（mock 数据含 PROCUREMENT_MANAGER + GM + DEPT_MANAGER 混合）。

## 5. 风险

| 风险 | 等级 | 缓解 |
|------|------|------|
| 聚合 SQL 性能 | 🟢 低 | workflow_code + approver_role + created_at 复合索引（V53 已建） |
| 时间范围缺省值 | 🟢 低 | 30 天前 ~ 今天（前端可覆盖） |
| 并发触发 recordEvent 影响统计 | 🟢 低 | 读已 commit 数据 · MVCC 隔离 · 不需加锁 |
| 与 9.1 触发端点的数据时序 | 🟢 低 | 9.1 触发 AFTER_COMMIT · 本 Story 聚合读不到未 commit 数据（预期） |
