## 1. 现状

`android-impl/` 仓根当前有：
- `build.gradle.kts` (8KB · V1.3.7 AGP 配置)
- `gradle.properties` (212B)
- `settings.gradle.kts` (626B)
- `.gradle/`（本地缓存，已 .gitignore）
- ❌ 无 `gradlew` / `gradlew.bat` / `gradle/wrapper/gradle-wrapper.{jar,properties}`

→ 任何新接手 dev 必须本地装 Gradle 8.7+ 才能跑构建 · 与 `docs/architecture/6-部署架构docker-compose本地化部署.md` 的"一键构建"原则不一致

## 2. 目标

落地标准 Android Gradle Wrapper，使 `./gradlew build` 在无系统级 Gradle 的环境下也可工作：

| 文件 | 来源 | 提交到 git |
|------|------|------------|
| `gradlew` | 官方脚本（Unix shell） | ✅ text |
| `gradlew.bat` | 官方脚本（Windows batch） | ✅ text |
| `gradle/wrapper/gradle-wrapper.jar` | Gradle 8.7 发行版 | ✅ binary |
| `gradle/wrapper/gradle-wrapper.properties` | 自写 | ✅ text |
| `gradle/wrapper/gradle-wrapper-main-class.gradle` | （可选 · 已废弃） | ❌ |

## 3. 验收点

### AC-10.4.1 产物落地

- 4 个文件全部存在
- `gradle-wrapper.jar` SHA256 与官方 Gradle 8.7 发行版一致（`https://services.gradle.org/distributions/gradle-8.7-wrapper.jar`）
- `gradle-wrapper.properties` 内容：
  ```
  distributionBase=GRADLE_USER_HOME
  distributionPath=wrapper/dists
  distributionUrl=https\://services.gradle.org/distributions/gradle-8.7-bin.zip
  networkTimeout=10000
  validateDistributionUrl=true
  zipStoreBase=GRADLE_USER_HOME
  zipStorePath=wrapper/dists
  ```

### AC-10.4.2 可执行

- `chmod +x gradlew` · `./gradlew --version` 退出码 0
- 输出含 "Gradle 8.7"
- `gradle-wrapper.properties` 中 distributionUrl 哈希记录在 `android-impl/.gradle-wrapper-sha256.txt`

## 4. 风险

| 风险 | 等级 | 缓解 |
|------|------|------|
| gradle-wrapper.jar 二进制提交 diff 噪声 | 🟢 低 | 业界标准做法 · V1.3.7 .gitignore 已含 `*.jar` 例外 |

| Windows CRLF / Unix LF 行尾 | 🟢 低 | `.gitattributes` 强制 `*.bat text eol=crlf` `gradlew text eol=lf` |
