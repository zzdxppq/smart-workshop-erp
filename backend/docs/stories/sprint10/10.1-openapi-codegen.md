## 1. Sprint 9.2 现状

Sprint 9.2 已把 openapi-typescript-codegen 接入 web-impl 仓，**但存在 3 个问题**：
- 8 个 `.vue` 仍用 `any`（V1.3.8 PM 反馈）→ Story 10.5 修
- `src/api/generated/` 未纳入 `tsc --noEmit` 强类型检查（CI 不拦截类型漂移）
- 契约变更（`openapi.yaml` 改动）无自动回归 → 改完字段前端不感知

## 2. 目标

把 codegen 输出纳入 web-impl 强类型门禁 + 提供 npm scripts 触发：

1. **生成**：`npm run gen:api` → 调用 `openapi-typescript-codegen -i ../backend/spec/openapi.yaml -o src/api/generated -c fetch`
2. **编译检查**：`npm run typecheck`（`tsc --noEmit`）含 `src/api/generated/**`
3. **变更检测**：CI 增加 step：codegen 后 `git diff --exit-code src/api/generated`（契约变更必须显式 commit）
4. **收敛 any**：与 Story 10.5 协同，本 Story 落地类型基础设施

## 3. 验收点（端点契约 · 不新增业务端点）

### 3.1 npm script 链路

```jsonc
// web-impl/package.json (新增)
{
  "scripts": {
    "gen:api": "openapi-typescript-codegen -i ../backend/spec/openapi.yaml -o src/api/generated -c fetch --useUnionTypes",
    "typecheck": "vue-tsc --noEmit",
    "typecheck:ci": "vue-tsc --noEmit && git diff --exit-code src/api/generated"
  },
  "devDependencies": {
    "openapi-typescript-codegen": "^0.29.0",
    "vue-tsc": "^2.1.10"
  }
}
```

### 3.2 tsconfig 覆盖

```jsonc
// web-impl/tsconfig.json
{
  "include": ["src/**/*.ts", "src/**/*.vue", "src/api/generated/**/*.ts"],
  "exclude": ["node_modules", "dist"]
}
```

## 4. Acceptance Criteria

### AC-10.1.1 codegen 可执行且输出稳定

测例：`npm run gen:api` 成功，产出 `src/api/generated/{models,services}/` 包含 V1.3.7 132 端点对应类型与 service stub。

### AC-10.1.2 typecheck 把 generated 纳入检查

测例：故意改 `backend/spec/openapi.yaml` 给 `QuoteDTO.totalAmount` 加 `nullable: true`，重跑 `npm run gen:api`，再 `npm run typecheck` 触发未处理 null 报错。

### AC-10.1.3 契约变更显式 commit（CI gate）

测例：在 PR 中跑 `npm run typecheck:ci`，若 `src/api/generated` 有未 commit 变更则退出码 1。

### AC-10.1.4 与 Sprint 8.4 / 9.2 衔接

测例：Sprint 9.2 已存在的 `src/api/quote.ts` 等手写 fetch 仍可工作（不强制迁移到 generated），与 10.5 渐进替换。
