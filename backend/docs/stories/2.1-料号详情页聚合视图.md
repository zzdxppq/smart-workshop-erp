## 1. V1.3.7 → V1.3.8 升级动因（PM 范蠡反馈 #1）

PM 反馈：仓管/采购/总经理在 ERP 内查一个料号需要在 6+ 模块间跳转，效率低、易漏数据。需求：**单页聚合 7 Tab，一屏看全**。

### 1.1 7 Tab 设计

| Tab | 数据源（V1.3.7 Story） | DTO 字段 | 缓存键 |
|-----|------------------------|----------|--------|
| 1. 基本信息 | 1.11 material 表 + classification | material_no · name · spec · unit · category · 默认仓库 | mat:detail:{id}:base |
| 2. 工艺路线 | 1.10 工艺库 + material_process 关联 | process_no · step_seq · workcenter · std_minutes · 设备 | mat:detail:{id}:process |
| 3. 图纸 | 1.7 图纸 + version | dwg_no · version · status · pdf_url · latest_flag | mat:detail:{id}:drawing |
| 4. 价格 | 1.33 价格控制 + price_history | current_price · avg_30d · min_30d · max_30d · 趋势点 12 个 | mat:detail:{id}:price |
| 5. 材料成本 | 1.40 料号成本聚合 5 段 | material_cost · scrap_rate · effective_cost | mat:detail:{id}:cost |

| 7. 外协成本 | 1.40 料号成本聚合 委外段 | outsource_cost · outsource_supplier · lead_days | mat:detail:{id}:outsource |

## 2. Acceptance Criteria

### AC-2.1.1 详情聚合端点

**Given** 调用 `GET /api/v1/materials/{id}/detail`
**When** 传入有效 material_id
**Then** 返回 7 Tab 聚合 DTO（含 base/process/drawing/price/cost/labor/outsource 全部字段）
**And** HTTP 200，response time P95 < 300ms（5min 缓存命中）
**And** material 不存在时 HTTP 404

### AC-2.1.2 7 Tab 前端渲染

**Given** PC 端 Element Plus 详情页 / APP 端 Vant 详情页
**When** 进入料号详情
**Then** 7 Tab 全部正确渲染（数据来自 AC-2.1.1 DTO）
**And** Tab 切换无白屏、无 N+1 请求
**And** 权限隔离：仓管可见基本/图纸/工时，采购可见基本/价格/外协，总经理可见全部

### AC-2.1.3 缓存策略

**Given** 详情聚合端点配置 Redis 5min 缓存（key=`mat:detail:{id}`）
**When** 同一 material 在 5min 内第二次访问
**Then** cache HIT，response time < 50ms
**And** 缓存失效：material/process/drawing/price 任一变更触发 `@CacheEvict`（AFTER_COMMIT）
**And** K6 压测 50 并发 P95 < 300ms 通过

## 4. 技术实现要点

### 4.1 后端聚合 DTO

```java
@Data
public class MaterialDetailDTO {
    private BaseInfo base;
    private ProcessInfo process;
    private DrawingInfo drawing;
    private PriceInfo price;
    private CostInfo cost;
    private LaborInfo labor;
    private OutsourceInfo outsource;
}
```

### 4.2 Redis 缓存

```java
@Cacheable(value = "mat:detail", key = "#id", ttl = 300)  // 5min
public MaterialDetailDTO getDetail(Long id) { ... }

@CacheEvict(value = "mat:detail", key = "#materialId")
public void onMaterialChange(Long materialId, ...) { ... }  // AFTER_COMMIT
```

### 4.3 权限隔离

复用 V1.3.7 `@PreAuthorize` 注解 + 角色矩阵：

| Tab | WAREHOUSE | PURCHASER | GM |
|-----|-----------|-----------|----|
| 基本信息 | ✅ | ✅ | ✅ |
| 工艺路线 | ✅ | ❌ | ✅ |
| 图纸 | ✅ | ❌ | ✅ |
| 价格 | ❌ | ✅ | ✅ |
| 材料/工时/外协成本 | ❌ | ✅（仅外协） | ✅ |

### 4.4 性能基线（K6）

- 50 并发 GET `/api/v1/materials/{id}/detail`
- 缓存命中：P95 < 50ms
- 缓存未命中：P95 < 300ms
- 错误率 < 0.1%
