## 1. 测试设计原则

| 维度 | 策略 | 理由 |
|------|------|------|
| **单元** | DrawingAuthz 7×5 矩阵全覆盖 | 35 cell 单元测试（每角色正/反例）+ 灰度开关旁路测试 |
| **集成** | 3 端点 + 5 业务单据 JOIN | V54 Flyway 迁移验证 · crm_drawing_link 5 表回填 · 端到端 ACL |
| **E2E** | web-impl DrawingViewer 7 角色 + android-impl DrawPermissionInterceptor | UI 权限位渲染一致性 · APP Toast 文案 |
| **性能** | SpEL 鉴权 P95 < 50ms | @Cacheable 5min + Redis 7 工序缓存 + idx_drawing_link 索引 |
| **契约** | 错误码统一 40304（评审建议）| 与 1.40 工序分配同码 · 错误码空间管理 |
| **灰度** | 4 阶段 feature flag 验证 | sys_dict `draw.acl.gray.{ROLE}` 灰度开关逐角色验证 |
| **数据迁移** | 备份 + 99% 校验 + uk_biz_ref 防重 | V54 末尾 crm_drawing_link_backup 备份 · 可回滚 |
| **审计** | @AuditLog DENY 100% 留痕 | sys_change_log module=DRAWING_ACL_DENY 验证 |

### 测例组 3：数据迁移（3 测例）

| ID | 场景 | 步骤 | 期望 | 优先级 |
|----|------|------|------|--------|
| **TC-12.1.3.1** | 5 表 JOIN 回填率 | admin 执行 `data-migrate-drawing-link.sql`（含 crm_order_item/po_item/incoming_order_item/inspection_item/workorder_process 5 表 JOIN crm_drawing.material_code）| 回填率 ≥ 99% · 5 类 biz_type 均有数据 · 验收 SQL 输出 coverage_pct | P0 |
| **TC-12.1.3.2** | uk_biz_ref UNIQUE 无重复 | 重复插入同一 `(drawing_id, biz_type, biz_id)` 二次 | DB 抛 DuplicateKeyException · 不污染 crm_drawing_link 表 | P0 |
| **TC-12.1.3.3** | 回滚（`crm_drawing_link_backup`）可恢复 | 故意制造数据错误 → 执行 `TRUNCATE crm_drawing_link; INSERT INTO crm_drawing_link SELECT * FROM crm_drawing_link_backup;` | 数据回滚到迁移前状态 · 记录数一致 | P0 |

**关键约束**：
- V54 末尾必须执行 `INSERT INTO crm_drawing_link_backup SELECT * FROM crm_drawing_link;`
- INSERT IGNORE + uk_biz_ref 防重（无需额外去重脚本）
- ON DELETE RESTRICT（评审建议）· 删除图纸时若有关联单据 → 提示"请先解绑"

## 4. 性能基线

| 操作 | 期望 | 实测容忍 | 备注 |
|------|------|----------|------|
| `DrawingAuthz.canView` 单次（DB 查询）| < 50ms | < 100ms | 含 Spring Security 评估 + SpEL 执行 |
| `@Cacheable` 命中（Redis）| < 5ms | < 10ms | key=`#auth.name + ':' + #drawingId` · TTL 5min |
| `UserProcessService.getCurrentProcess`（Redis 命中）| < 3ms | < 8ms | OPERATOR 高速扫码场景 |
| 数据迁移 5 表 10000 行 | < 30s | < 60s | `data-migrate-drawing-link.sql` |
| 24 测例全跑 | < 3min | < 5min | 35 单元 + 6 集成 + 3 E2E |
| E2E web-impl DrawingViewer 渲染 | < 1s | < 2s | 包含权限位计算 + 按钮渲染 |
| android-impl Toast 拦截 | < 200ms | < 500ms | 用户体验阈值 |

**性能优化栈**：
- `@Cacheable(value = "drawingAuthz", key = "#auth.name + ':' + #drawingId")` · TTL 5min
- `idx_drawing_link (drawing_id, biz_type, biz_id)` 索引 · O(1) 查询
- Redis 7 缓存 `user_id → current_process_id` · TTL 5min
- uk_biz_ref 防重 · 避免应用层去重开销

## 6. 回归策略

### 6.1 0 业务代码破坏原则

- **纯 ACL 加固**：仅在 DrawingAuthz 拦截层 + crm_drawing_link 关联层 · 不修改业务逻辑
- 既有 1.7 图纸查看/上传/加密功能保留
- 既有 1.3 金额 ACL 不受影响（命名空间独立）

### 6.2 既有测例保持 PASS

| Story | 测例 | 12.1 后状态 |
|-------|------|------------|
| **1.7** 图纸上传 + 加密 | 12 测例 | PASS（V54 数据迁移后 · 既有 crm_drawing 表结构不变）|
| **2.1** 料号详情聚合 7 Tab | 18 测例 | PASS（图纸 Tab 按 12.1 ACL 过滤 · 工程师仍可见）|
| **1.1** 10 角色 + JWT | 24 测例 | PASS（角色枚举不变 · 7 角色映射新增 sys_role_permission）|
| **1.3** 4 金额阈值 | 14 测例 | PASS（sys_global_threshold 表不变 · 与 draw:* 命名空间独立）|
| **1.4** 5 类码扫描壳 | 8 测例 | PASS（MaterialBarcodeScan 入口不变 · 鉴权后置）|

### 6.3 9.2 4 happy path 回归

- 工程师上传图纸 → 加密签字扫描 → 加密 → 入库（PASS）
- 工程师发布图纸 → RELEASED 状态 → 全员可见（PASS · ENGINEER 5/5 true）
- 业务员查看关联订单图纸 → 预览 → 打印（PASS · SALES linked canView=true）
- 财务角色无图纸入口 → 不见图纸（PASS · FINANCE scope=NONE）

### 6.4 灰度观察期

- 每角色灰度观察 1-2 天 · admin 控制 sys_dict feature flag
- @AuditLog DRAWING_ACL_DENY 日志监控 · 拒绝率突增 → 立即回滚
- 客户投诉（"看不到部分图纸"）→ feature flag false 即时回退
