## 1. 测试设计原则

| 维度 | 策略 | 理由 |
|------|------|------|
| **单元** | PrinterService CRUD + heartbeat probe 逻辑 | 16 测例主力 · fail_count 状态机分支 |
| **集成** | 6 端点 + admin UI | 与 1.3 sys_dict 同款（端点 + UI 联动 + 字典种子）|
| **E2E** | 不新增（admin UI 已覆盖） | 与 9.2 风格一致 · 16 测例聚焦核心 |
| **性能** | 心跳 P95 < 5s（N 台并发） | @Scheduled 单线程 · V1.3.9 客户规模 5-10 台 |
| **契约** | 错误码 40001/40401/40304/40901/40902/42201/50201 | 沿用 V1.3.7 1.1 + 12.2 新增 40901/50201 |
| **心跳** | fail_count ≥ 2 才标 OFFLINE | 防瞬断误标 · 单次失败仅 fail_count++ |
| **多租户** | INSERT/UPDATE 必带 tenant_id | V1.3.7 1.1 RBAC 已 ship · 查询走 `WHERE tenant_id = ?` |
| **审计** | @AuditLog AFTER_COMMIT 100% 留痕 | module=printer · admin 写操作全覆盖 |

- TC-12.2.1.5 验证 `@PreAuthorize("hasAuthority('sys:admin')")` 拦截 · 沿用 1.1 RBAC 错误码 40304

### 测例组 2：心跳探活（4 测例）

| ID | 场景 | 步骤 | 期望 | 优先级 | 备注 |
|----|------|------|------|--------|------|
| **TC-12.2.2.1** | 真实 ZD420 IP 探活成功 | mock TCP connect 成功（端口 9100 监听） · 等待 @Scheduled 60s 周期或手动触发 | status=ONLINE · fail_count=0 · lastHeartbeatAt=NOW() | P0 | mock `Socket.connect()` 返回成功 |
| **TC-12.2.2.2** | 不可达 IP 探活失败 1 次 | mock TCP connect 失败（端口拒绝） · 第 1 次调度 | fail_count=1 · status 仍 ONLINE（容差） | P0 | 验证 fail_count++ 单次不立即 OFFLINE |
| **TC-12.2.2.3** | 连续 2 次失败标 OFFLINE | mock TCP connect 失败 · 第 2 次调度 | fail_count=2 · status=OFFLINE | P0 | 关键容差验证 · 防瞬断误标 |
| **TC-12.2.2.4** | 探活超时 2s（firewall drop） | mock TCP connect 阻塞 → 2s 超时 → 抛出 SocketTimeoutException | fail_count++ · @Scheduled 不阻塞 · 下次调度正常 | P0 | 验证 timeout 设置 · 避长卡 |

**补充说明**：
- 单元测试用 Mockito 模拟 `Socket.connect()` 行为；集成测试可起一个本地 `ServerSocket(9100)` mock 真实 ZD420
- TC-12.2.2.3 关键：连续 2 次失败（不是累计 2 次）· 第 3 次成功应将 fail_count 重置为 0
- TC-12.2.2.4 关键：超时不阻塞调度线程 · 验证 2s 后正常抛 IOException 并被 catch

## 4. 性能基线

| 操作 | 期望 | 实测容忍 | 备注 |
|------|------|----------|------|

| **TCP 探活单台** | < 2s | < 5s | Socket.connect timeout=2s · 防止 firewall 长卡 |

## 5. 风险与缓解

| # | 风险 | 等级 | 缓解 | 测例 |
|---|------|------|------|------|
| 1 | **瞬断误标 OFFLINE**（心跳 60s 周期遇到网络瞬断）| 🟢 低 | `fail_count ≥ 2` 容差 · 单次失败仅 `fail_count++` | TC-12.2.2.3 |
| 2 | **TCP firewall 拦截**（客户机房 9100 端口被防火墙 drop）| 🟡 中 | 2s connect timeout + 异步重试 · V1.4 backlog 改 SNMP | TC-12.2.2.4 |
| 3 | **admin UI 与 1.3 sys_dict 不一致**（设计走样）| 🟢 低 | 沿用 1.3 已 ship 的 Element Plus el-table + el-form 模式 | TC-12.2.4.1~4.2 |
| 4 | **12.2 与 12.4 启动顺序错位**（sys_print_log 未就位）| 🟡 中 | 12.2 先 ship（V55 表就位）· 12.4 后（V57 sys_print_log） | TC-12.2.1.4（mock 外键引用）|
| 5 | **ZPL vs TSPL 协议差异** | 🟢 低 | 12.2 仅存 `protocol` 字段 · 12.4 抽象 `ProtocolAdapter` | TC-12.2.4.2（联动）|
| 6 | **多租户串数据**（tenant_id 缺失）| 🟢 低 | V1.3.7 1.1 RBAC 已 ship · 12.2 INSERT/UPDATE 必带 tenant_id | TC-12.2.1.1（带 tenant_id=1）|
