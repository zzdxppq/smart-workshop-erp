## 5. 心跳 fail_count ≥ 2 容差验证

### 5.1 关键代码（SysPrinterService.probeHeartbeat）
```java
} catch (IOException e) {
    // 失败
    int newFail = (p.getFailCount() == null ? 0 : p.getFailCount()) + 1;
    String newStatus = newFail >= FAIL_COUNT_THRESHOLD  // 2
            ? SysPrinter.STATUS_OFFLINE
            : p.getStatus(); // 保持不变（容差）
    p.setFailCount(newFail);
    p.setStatus(newStatus);
    printerMapper.updateHeartbeat(p.getId(), newStatus, newFail, p.getLastHeartbeatAt());
}
```

### 5.2 状态机分支
| 当前 fail_count | 当前 status | TCP 探活 | 新 fail_count | 新 status | 行为 |
|----|----|----|----|----|----|
| 0 | ONLINE | 成功 | 0 | ONLINE | fail_count=0 · last_heartbeat_at=NOW() |
| 0 | ONLINE | 失败 | 1 | ONLINE（保持）| fail_count++ · status 不变 |
| 1 | ONLINE | 失败 | 2 | **OFFLINE** | 触发容差阈值 |
| 2 | OFFLINE | 成功 | 0 | **ONLINE** | fail_count 重置 |
| 2 | OFFLINE | 失败 | 3 | OFFLINE | 持续 OFFLINE |
| N/A | UNKNOWN | N/A | 0 | UNKNOWN | NORMAL 类型不探活 · return |

### 5.3 TC 覆盖
- TC-12.2.2.1: `127.0.0.1` 端口 OK 分支
- TC-12.2.2.2: `127.0.0.2:1` 第 1 次失败 → fail_count=1 · status 保持 ONLINE
- TC-12.2.2.3: `127.0.0.2:1` 第 2 次失败 → fail_count=2 · status=OFFLINE
- TC-12.2.2.4: NORMAL 类型走 type guard · return · verify never called
