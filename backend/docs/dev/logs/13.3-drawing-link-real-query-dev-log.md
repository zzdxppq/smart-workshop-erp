## 3. 5 类 link JOIN SQL 落地（CrmDrawingLinkMapper 新增 6 查询）

### 3.1 端点 1 · 4 类 link JOIN（业务单据主表 → 图纸）

```sql
-- ORDER：crm_drawing → crm_drawing_link(ORDER) → crm_order_item(material) → crm_order
SELECT DISTINCT o.id AS biz_id
FROM crm_drawing d
JOIN crm_drawing_link dl ON dl.drawing_id = d.id AND dl.biz_type = 'ORDER'
JOIN crm_order_item oi ON oi.material = d.material_code
JOIN crm_order o ON o.id = oi.order_id AND o.owner_user_id = #{userId}
WHERE d.id = #{drawingId}
  AND o.status NOT IN ('DRAFT','CANCELLED')
  AND o.is_deleted = 0

-- PO：JOIN crm_purchase_order_item.material_code + purchase_order.created_by
-- INCOMING：JOIN crm_incoming_order_item.material_code + wms_inbound.operator_user_id
-- INSPECTION：JOIN crm_inspection_item.material_code + qc_inspection.inspector_user_id
```

### 3.2 端点 2 · 业务单据 → 图纸列表（MyBatis `<script>` 动态 SQL 分发）

```sql
SELECT DISTINCT d.id, d.drawing_no, d.title, d.version, d.pdf_path
FROM crm_drawing d
JOIN crm_drawing_link dl ON dl.drawing_id = d.id AND dl.biz_type = #{bizType}
JOIN crm_order_item oi ON oi.material = d.material_code  -- 5 类分支
WHERE oi.order_id = #{bizId}    -- 5 类分支
```

### 3.3 端点 3 · 工序 → 图纸列表（含工序/工单信息 · IN_PROGRESS 过滤）

```sql
SELECT DISTINCT d.id, d.drawing_no, d.title, d.version, d.pdf_path,
       wp.id, wp.process_code, wp.process_name,
       wp.work_order_id, wo.workorder_no, wp.status, wp.operator_user_id
FROM crm_workorder_process wp
JOIN crm_workorder wo ON wo.id = wp.work_order_id
JOIN crm_workorder_item woi ON woi.work_order_id = wo.id
JOIN crm_drawing d ON d.material_code = woi.material_code
WHERE wp.id = #{processId}
  AND wp.status = 'IN_PROGRESS'
```

## 4. Redis 缓存设计（IMPL 注意事项 #2 · 防角色权限漂移）

### 4.1 Key 设计（含 role + user_id）

| 端点 | Key 模式 | TTL | 失效触发 |
|------|---------|-----|----------|
| 端点 2 | `drawing:link:{bizType}:{bizId}:{role}:{user_id}` | 300s | crm_drawing_link 写入 · @CacheEvict |
| 端点 3 | `user:current_process:{user_id}` | 300s | crm_workorder_process 状态变更 · @CacheEvict |

### 4.2 降级策略

```java
// DrawingLinkQueryService.fail_count >= 3 退化 DB
private boolean shouldDegrade(String cacheName) {
    AtomicInteger c = redisFailCount.get(cacheName);
    return c != null && c.get() >= REDIS_FAIL_THRESHOLD;  // 3
}
```

- Redis 正常 → 命中缓存（cacheHit=true · 响应 < 5ms）
- Redis 失败 1-3 次 → 退化 DB（fail_count 累计）
- Redis 失败 ≥ 4 次 → 强制走 DB（shouldDegrade 返 true）· DB QPS < 100 安全

### 4.3 @CacheEvict 一致性

```java
@CacheEvict(value = {"drawingLink:PERM"}, allEntries = true)
public void evictAllLinkCaches() {
    log.info("[13.3] evictAllLinkCaches · crm_drawing_link 写入触发");
}
```

- crm_drawing_link INSERT/UPDATE/DELETE → 调用 evictAllLinkCaches()
- TTL 兜底：5min 强制过期 · 防止 @CacheEvict 漏掉导致长期脏读
