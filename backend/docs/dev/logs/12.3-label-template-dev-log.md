## 4. V56 Flyway 落地确认

```sql
CREATE TABLE label_template (
    id            BIGINT NOT NULL AUTO_INCREMENT,        -- PK
    `type`        VARCHAR(10) NOT NULL,                  -- GD/LZ/WW/WL（SB 代码层 fallback）
    color_strip   VARCHAR(7) NOT NULL,                   -- #1E40AF/#16A34A/#EA580C/#000000
    factory_name  VARCHAR(20) NOT NULL DEFAULT '昆山佰泰胜精密加工',
    layout_json   VARCHAR(500) NOT NULL,                 -- 三区坐标 + 字体 + DPI
    dpi           SMALLINT NOT NULL DEFAULT 300,          -- 203 / 300
    enabled       TINYINT(1) NOT NULL DEFAULT 1,
    tenant_id     BIGINT NOT NULL DEFAULT 1,
    created_at    DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    updated_at    DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (id),
    UNIQUE KEY uk_label_type (`type`, tenant_id),
    KEY idx_tenant (tenant_id),
    CONSTRAINT chk_label_type CHECK (`type` IN ('GD','LZ','WW','WL')),
    CONSTRAINT chk_label_dpi  CHECK (dpi IN (203, 300))
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

- ✅ **10 字段**：PK + 业务 7（type/color_strip/factory_name/layout_json/dpi/enabled/tenant_id）+ 审计 2
- ✅ **3 索引**：PRIMARY KEY + UNIQUE KEY uk_label_type + KEY idx_tenant
- ✅ **2 CHECK**：chk_label_type + chk_label_dpi
- ✅ **4 行 seed**（INSERT ... ON DUPLICATE KEY UPDATE 幂等）：
  - GD #1E40AF
  - LZ #16A34A
  - WW #EA580C
  - WL #000000
- ✅ **SB- 不入库**（4 行 seed）· 代码层 `LabelTemplateService.getTemplate("SB")` fallback 到 GD + 覆盖 color_strip=#6B7280

## 5. SB- fallback 单测证据

`LabelTemplateServiceTest.TC_12_3_1_2_sb_fallback_gd_grey_strip`：

```java
// Arrange: SB- DB 不存在 + GD 行存在
when(templateMapper.selectByType(eq("SB"), eq(1L))).thenReturn(null);
LabelTemplate gd = makeTemplate("GD", "#1E40AF", 300);
when(templateMapper.selectByType(eq("GD"), eq(1L))).thenReturn(gd);

// Act
LabelTemplateService svc = new LabelTemplateService(templateMapper, qrCodeGenerator);
LabelTemplate got = svc.getTemplate("SB", 1L);

// Assert
assertNotNull(got, "SB- fallback 必须返回 GD 模板");
assertEquals("GD", got.getType());
assertEquals("#6B7280", got.getColorStrip(), "严格只覆盖 color_strip=#6B7280");
assertEquals(gd.getLayoutJson(), got.getLayoutJson(), "layout_json 不覆盖");
assertEquals(gd.getDpi(), got.getDpi(), "dpi 不覆盖");
assertEquals(gd.getFactoryName(), got.getFactoryName(), "factory_name 不覆盖");
```

**通过标准**：SB- fallback 后色条严格覆盖为 #6B7280，layout_json/dpi/factory_name 与 GD 完全一致。

## 7. 多仓跨端一致证据

| 仓 | 调用方式 | 接收内容 | 一致性保证 |
|----|---------|---------|-----------|
| backend | `LabelTemplateService.preview()` 渲染 | base64 PNG | 权威源 · ZXing + 三区布局 |
| web-impl | `<LabelPreview>` 调 `/preview` | base64 → `<img :src="data:image/png;base64,...">` | 100% 一致（同一后端权威源）|
| android-impl | `LabelPreviewFragment` 调 `/preview` | base64 → Bitmap 显示 | 100% 一致（同一后端权威源）|

**视觉一致性机制**（architect §2.4）：
- 三端一律 `<img>` 渲染 base64 PNG · 不用前端 qrcode.js/ZXing 即时生成
- 后端是唯一权威源 · 前端/移动端仅消费 base64
- **本地预览**（草稿态）可用前端 qrcode.js / ZXing 即时生成 · 但**最终打印前必须用后端 preview**

## 8. 与 12.4 协同指引

12.4 双模式打印启动时需引用：

### 8.1 复用资产

| 资产 | 路径 | 12.4 用法 |

| OpenAPI spec | `spec/openapi.v139-label-templates.yaml` | 12.4 端点命名空间延续 `/print/labels/...` |

### 8.2 12.4 启动时序

1. **读 12.3 spec**（`openapi.v139-label-templates.yaml` + `*-schemas.yaml`）
2. **复用 LabelTemplateService.preview()** · 输出 base64 PNG
3. **新建 sys_print_log 表**（V57 Flyway）· 外键 `type` 引用 `label_template.type`
4. **新增端点 `/api/v1/print/labels`**（POST 打印触发）· 复用 12.3 preview + 12.2 sys_printer
5. **集成 web-impl `<LabelPreview>`** · 在工单/流转/委外/设备管理页加"打印预览"按钮 → 调 `/print/labels`

### 8.3 12.4 决策点

- **PDF 格式**：12.3 仅支持 PNG（LabelPngRenderer 用纯 JDK BufferedImage）· 12.4 引入 OpenPDF 后扩展 PDF 路径
