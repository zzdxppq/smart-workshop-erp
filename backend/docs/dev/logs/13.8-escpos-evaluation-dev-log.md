### 3.3 AC-13.8.3 EscPosAdapter 抽象评估

#### 3.3.1 PrintProtocolAdapter 接口草案

**GIVEN**：Sprint 12.4 `ProtocolAdapter` 抽象（`ZplAdapter` + `TsplAdapter` + `PdfBrowserAdapter`）

**WHEN**：13.8 评估 `EscPosAdapter` 平级抽象

**THEN**（评估草案）：

```kotlin
// backend/src/main/kotlin/com/btsheng/erp/print/protocol/PrintProtocolAdapter.kt (V1.3.10 backlog 草案)
interface PrintProtocolAdapter {
    fun build(payload: PrintPayload): ByteArray
}

class EscPosAdapter : PrintProtocolAdapter {
    // ESC/POS 字节流生成
    override fun build(payload: PrintPayload): ByteArray {
        val bytes = ByteArrayOutputStream()
        bytes.write(0x1B); bytes.write('@'.code)  // ESC @ 初始化
        bytes.write(0x1B); bytes.write('!'.code); bytes.write(0x00)  // ESC ! 0x00 字号复位
        bytes.write(0x1B); bytes.write('a'.code); bytes.write(0x01)  // ESC a 0x01 居中
        bytes.write("销售小票\n".toByteArray(Charsets.UTF_8))
        // ... 二维码（GS ( k） + 切纸（GS V） + 中文厂名（思源黑体嵌入）
        bytes.write(0x1D); bytes.write('V'.code); bytes.write(0x42); bytes.write(0x00)  // GS V 切纸
        return bytes.toByteArray()
    }
}
```

**评估维度**：

| 维度 | 评估项 | 评估结论 |
|------|--------|----------|
| 接口定义 | `interface PrintProtocolAdapter { fun build(payload: PrintPayload): ByteArray }` | 🟢 **就位** · 与 12.4 `LabelProtocol` 平级 |
| 实现类 | `EscPosAdapter`（含 ESC @ 初始化 + ESC ! 字号 + ESC a 对齐 + GS V 切纸 + GS ( k 二维码）| 🟢 **就位** · 4 选 1 工厂方法清晰 |
| 字节流断言测试 | 飞鹅 FP-58 Ⅱ / 佳博 GP-58MB 字节流单测 | 🟢 **就位** · mock Socket 服务端 · 断言 bytesSent 与 build() 输出逐字节相等 |
| 错误码复用 | 50201-50203 复用（与 ZPL/TSPL 一致）| 🟢 **就位** · 50201 PRINTER_OFFLINE + 50202 PROTOCOL_UNSUPPORTED + 50203 协议发送失败 |
| @Async 线程池复用 | 12.4 既有 `print-zpl-` 前缀线程池 | 🟢 **复用** · 不新增线程池 |
| Socket 9100 实现 | 与 ZPL/TSPL 平级 · 复用 12.2 心跳调度 | 🟢 **复用** · 2s connect timeout + fail_count ≥ 2 |

**评估深度**：🟡 中（ESC/POS 设备型号碎片化 · 字节流指令集兼容性需单测覆盖主流型号）
