## 6. ZPL Socket 代理架构（@Async + ProtocolFactory + 3s 超时）

### 6.1 完整流程（PrintService.printZpl + asyncSendZpl）

```java
// 1. 同步预检（HTTP 线程内）
public Result<ZplPrintResult> printZpl(ZplPrintRequest req, Long userId, String userName, Long tenantId) {
    // 1.1 业务校验
    if (req.getCount() < 1 || req.getCount() > 100) {
        return Result.fail(40003, "count 范围 1-100");
    }
    // 1.2 查 sys_printer
    SysPrinter printer = printerMapper.selectById(req.getPrinterId());
    if (!"ONLINE".equals(printer.getStatus())) {
        return Result.fail(50201, "PRINTER_OFFLINE: " + printer.getName());
    }
    // 1.3 协议预检
    if (!protocolFactory.has(printer.getProtocol())) {
        return Result.fail(50202, "PROTOCOL_UNSUPPORTED: " + printer.getProtocol());
    }
    // 2. 创建 PENDING log
    SysPrintLog log = new SysPrintLog();
    log.setLogNo(generateLogNo(tenantId));
    log.setStatus("PENDING");
    log.setPrintMode("ZPL_DIRECT");
    printLogMapper.insert(log);
    Long logId = log.getId();
    // 3. @Async 异步执行（独立线程池 · 不阻塞 HTTP）
    asyncSendZpl(logId, req, printer, userId, userName, tenantId);
    return Result.ok(new ZplPrintResult(printLogId=logId, logNo, ...));
}

@Async("printZplExecutor")
public void asyncSendZpl(Long logId, ZplPrintRequest req, SysPrinter printer, ...) {
    try {
        // 3.1 ProtocolFactory 选 ZplProtocol / TsplProtocol
        LabelProtocol protocol = protocolFactory.get(printer.getProtocol());
        byte[] payload = protocol.render(data, req.getCount());

        // 3.2 Socket 3s 硬性超时
        try (Socket socket = new Socket()) {
            socket.connect(new InetSocketAddress(printer.getIp(), printer.getPort()), 3000);
            socket.getOutputStream().write(payload);
            socket.getOutputStream().flush();
            Thread.sleep(200);  // 等待打印机缓冲
        }
        // 3.3 写 SUCCESS log
        updateLogSuccess(logId, payload.length, latencyMs);
    } catch (SocketTimeoutException e) {
        updateLogFailed(logId, "SocketTimeout: " + e.getMessage());
    } catch (IOException e) {
        updateLogFailed(logId, e.getClass().getSimpleName() + ": " + e.getMessage());
    }
}
```

### 6.2 @Async 线程池配置（PrintAsyncConfig）

```java
@Bean(name = "printZplExecutor")
public Executor printZplExecutor() {
    ThreadPoolTaskExecutor executor = new ThreadPoolTaskExecutor();
    executor.setCorePoolSize(4);          // core=4
    executor.setMaxPoolSize(16);          // max=16
    executor.setQueueCapacity(200);       // queue=200
    executor.setThreadNamePrefix("print-zpl-");
    executor.setRejectedExecutionHandler(new ThreadPoolExecutor.CallerRunsPolicy());
    executor.setAllowCoreThreadTimeOut(true);
    executor.setKeepAliveSeconds(60);
    return executor;
}
```

### 6.3 关键设计要点

- **3s 硬性超时**：`SOCKET_CONNECT_TIMEOUT_MS = 3000` · 验证（IMPL 注意事项 3）
- **200ms 缓冲等待**：`SOCKET_BUFFER_WAIT_MS = 200` · 等待打印机内部缓冲
- **try-with-resources**：保证 Socket.close() · 防连接泄漏
- **异常兜底**：`PrintAsyncExceptionHandler` · 防 @Async 静默吞异常（architect R4 重点）
- **协议适配**：`ProtocolFactory.get(name)` 失败 → 50202 PROTOCOL_UNSUPPORTED

## 7. 27 标签/页 PDFBox 解析证据

### 7.1 计算依据

- 标签尺寸：50mm × 30mm

    }

    SysPrintLog newLog = new SysPrintLog();
    newLog.setReferenceLogId(sourceId);
    printLogMapper.insert(newLog);

    ...
}
