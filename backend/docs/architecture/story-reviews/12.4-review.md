### 1.4 错误码

| 错误码 | 含义 | 触发场景 | HTTP |
|--------|------|----------|------|
| **40001** | 参数校验失败 | count>100 / items>30 / lines>6 | 422 |
| **40950** | `PRINT_CONNECTION_FAILED` | 模式一 IP 不通 / 端口拒 / 协议不匹配 | 409/502 |
| **40951** | `PRINT_SOCKET_IO` | Socket connect refused / 写入失败 | 502 |
| **40952** | `PRINT_PROTOCOL_MISMATCH` | NORMAL 打印机调 ZPL 端点 | 409 |

### 2.2 ZPL Socket 代理架构（模式一）

```
┌─────────────┐  REST  ┌──────────────────┐  Socket  ┌─────────────┐
│  web-impl / │ ─────▶ │  PrintController │ ──────▶  │  ZPL/TSPL   │
│ android-impl│        │  (12.4 backend)  │  9100     │  标签打印机  │
└─────────────┘        └──────────────────┘          └─────────────┘
                                │
                                ▼
                       ┌──────────────────┐
                       │ PrintService     │
                       │  .sendZpl()      │
                       │  @Async (线程池) │
                       └──────────────────┘
                                │
                  ┌─────────────┼─────────────┐
                  ▼             ▼             ▼
            ┌─────────┐  ┌──────────┐  ┌────────────┐
            │sys_printer│ │ sys_print │ │ sys_change │
            │  .ip:9100│  │   _log    │  │   _log     │
            │  .protocol│ │ status    │  │ SYS_PRINT  │
            │  .status  │  │           │  │   _LOG     │
            └─────────┘  └──────────┘  └────────────┘
```

**`PrintService.sendZpl(printerId, zplData)` 流程**
1. 查 `sys_printer` → IP + port（默认 9100）· 校验 `status=ONLINE` · `protocol ∈ {ZPL, TSPL}`
2. `ProtocolAdapter.render(LabelData)` 选 `ZplProtocol` 或 `TsplProtocol` → `byte[]` 字节流
3. `new Socket()` · `connect(new InetSocketAddress(ip, port), 3000)` · **3 秒硬性超时**
4. `OutputStream.write(bytes)` + `flush()`
5. `Thread.sleep(200)` 等待打印机缓冲（@Async 线程 · 不阻塞 HTTP）
6. `socket.close()` · 写 `sys_print_log status=SUCCESS bytesSent latencyMs` · 返回
7. 失败 → 抛 `BizException(50203, "ZPL 发送失败：" + e.getMessage())` · **@Async 异常** → 写 `sys_print_log status=FAILED error_msg=e.getMessage()`（不让异常静默吞掉）

**关键设计要点**
- **@Async 线程池配置**：`@Configuration class AsyncConfig { ThreadPoolTaskExecutor { core=4 · max=16 · queue=200 } }` · 防 Socket 阻塞占满 HTTP 线程
- **Socket 资源释放**：`try-with-resources` 保证 `close()` · 防连接泄漏
- **协议适配器依赖 sys_printer.protocol 字段**：V57 已存 `ZPL` / `TSPL` 字符串（Story 12.2 引入）
- **测试 Mock 策略**：`@MockBean SocketFactory` · 注入假 Socket 服务端（绑定本地 0 号端口）· 断言 `bytesSent` 与 `latencyMs`

### 2.3 A4 PDF 排版生成（模式二）

**使用 OpenPDF 1.3.34**（Story 1.5/1.6 既有依赖 · V1.3.7 验证）

**标签尺寸 + 排版计算（PM prompt 修正版）**
- 标签尺寸：50mm × 30mm（GD/LZ/SB/WW/WL 共用）
- A4 纸：210mm × 297mm
- 边距设计：左/右 30mm（每列 50mm · 3 列 = 150mm · A4 宽 210mm − 150mm = 60mm · 两侧各 30mm ✓）
- 上下：每行 30mm × N 行 = 30N mm ≤ 297mm → N ≤ 9.9 → **N=9 行/页**
- **每页标签数 = 3 列 × 9 行 = 27 标签/页**（PM prompt 修正 · Story 文件 30 标签/页不可达）

**OpenPDF 实现要点**
```java
Document doc = new Document(new Rectangle(210 * 2.83f, 297 * 2.83f));  // mm → pt
PdfWriter writer = PdfWriter.getInstance(doc, baos);
doc.open();
PdfPTable table = new PdfPTable(3);
table.setWidthPercentage(100);
table.setWidths(new float[]{1, 1, 1});  // 3 等宽列
for (LabelItem item : items) {
    PdfPCell cell = new PdfPCell();
    cell.setFixedHeight(30 * 2.83f);  // 30mm → 85pt
    cell.setBorder(Rectangle.NO_BORDER);
    cell.addElement(renderCell(item));  // 嵌入 ZXing QR + lines
    table.addCell(cell);
}
// 不足 27 项 → 补空 cell 保持栅格
while (table.size() < 27) table.addCell(emptyCell());
doc.add(table);
doc.close();
return baos.toByteArray();
```

**关键约束**
- `items.size()` ≤ 30（OpenAPI 强校验）· 超过 → 40001 错误
- 不足 27 项 → 留白（不缩印）· 简化裁剪计算
- **实测要求**：dev 启动后用 PDF viewer 验证 27 标签/页（与预期一致）· 修正 Story 文件的 3×10=30 描述

### 2.4 ProtocolAdapter 抽象

```java
// V57 新增 · backend/protocol/label/LabelProtocol.java
public interface LabelProtocol {
    /** 根据 LabelData 生成打印机指令字节流 */
    byte[] render(LabelData data);

    /** 协议名 · 与 sys_printer.protocol 字段对应 */
    String protocolName();
}

// V57 新增 · backend/protocol/label/ZplProtocol.java
@Component("ZPL")
public class ZplProtocol implements LabelProtocol {
    @Override public byte[] render(LabelData d) {
        String zpl = String.format(
            "^XA^FO20,20^BCN,80,Y,N,N^FD%s^FS" +    // QR
            "^FO20,120^A0N,28,28^FD%s^FS" +          // line[0]
            "^FO20,160^A0N,24,24^FD%s^FS" +          // line[1]
            "^XZ", d.qrContent(), d.lines().get(0), d.lines().get(1));
        return zpl.getBytes(StandardCharsets.UTF_8);
    }
    @Override public String protocolName() { return "ZPL"; }
}

// V57 新增 · backend/protocol/label/TsplProtocol.java
@Component("TSPL")
public class TsplProtocol implements LabelProtocol {
    @Override public byte[] render(LabelData d) {
        // TSPL 语法: SIZE 50 mm,30 mm · CLS · BITMAP ... · TEXT ...
        String tspl = String.format(
            "SIZE 50 mm,30 mm\r\n" +
            "CLS\r\n" +
            "QRCODE 20,20,M,6,A,0,\"%s\"\r\n" +
            "TEXT 20,130,\"3\",0,1,1,\"%s\"\r\n" +
            "PRINT 1\r\n", d.qrContent(), d.lines().get(0));
        return tspl.getBytes(StandardCharsets.UTF_8);
    }
    @Override public String protocolName() { return "TSPL"; }
}
```

**ProtocolFactory**（按 `sys_printer.protocol` 字段注入）
```java
@Component
public class ProtocolFactory {
    private final Map<String, LabelProtocol> protocols;
    public ProtocolFactory(List<LabelProtocol> impls) {
        this.protocols = impls.stream()
            .collect(Collectors.toMap(LabelProtocol::protocolName, p -> p));
    }
    public LabelProtocol get(String name) {
        LabelProtocol p = protocols.get(name);
        if (p == null) throw new BizException(50202, "PROTOCOL_UNSUPPORTED: " + name);
        return p;
    }
}
```

- TSC TTP-244 Pro → `TSPL`

- 12.2 sys_printer.protocol 字段已存 `ZPL` / `TSPL` 字符串 · dev 仅需实现两个 `LabelProtocol` bean

### 2.5 补打 reference_log_id 链

**写入逻辑**
```java
@Transactional
public PrintLogDTO replay(Long sourceId, ReprintRequest req) {
    PrintLog source = printLogRepo.findById(sourceId)
        .orElseThrow(() -> new BizException(404, "日志不存在"));
    if (source.getStatus() != SUCCESS)
        throw new BizException(40954, "PRINT_REPLAY_FORBIDDEN: 仅 SUCCESS 可补打");
    if (source.getReferenceLogId() != null)
        throw new BizException(40954, "PRINT_REPLAY_FORBIDDEN: 补打记录不可再次补打（防递归）");

    PrintMode targetMode = req.getTargetMode() == SAME
        ? source.getMode() : req.getTargetMode();
    // ... 调原 mode 或新 mode 打印 · 写新 log
    PrintLog newLog = new PrintLog();
    newLog.setReferenceLogId(sourceId);  // 指向原始
    return printLogRepo.save(newLog);
}
```

**审计追溯链**
- 原始 log (a) ← 补打 (b, ref=a) ← 补打 (c, ref=b) ❌ **禁止**（c 的 source.referenceLogId = b.ref ≠ null）
- UI 显示：列表行 `referenceLogId != null` → 红色"补打" tag · 鼠标悬停显示原始编码

## 6. IMPL 注意事项（5 条）

1. **ProtocolAdapter 单元测试覆盖 3 型号**：Zebra ZD420 (ZPL) / TSC TTP-244 Pro (TSPL) / 启邦 DL-888B (TSPL) · mock 假 Socket 服务端接收字节 · 断言 `bytesSent` 与协议 `render()` 输出 **逐字节相等**（防 ZPL/TSPL 语法错位 · 不可上线后才发现）
2. **A4 PDF 9 行/页实测**：dev 启动后用 iText/OpenPDF 生成 1 页 27 标签 PDF · PDFBox 3.x 解析验证 `PdfPTable.size() == 27`（不是 30）· 修正 Story §3 "3×10=30"描述 → 实际"3×9=27"（PM prompt 已修正）
3. **Socket 超时 3 秒硬性**：避免主线程阻塞 · `@Async` 线程池配置 `core=4 max=16 queue=200`（Story §4 AC-12.4.1 写 2 秒 · 建议 3 秒更稳妥）· IMPL 前 SM 萧何与 PO 范蠡确认 2s vs 3s
4. **补打链禁止递归**：补打记录的 `referenceLogId != null` → 拒绝再次补打（40954）· `@Transactional` 内校验 · 防止"补打的补打"形成死链
5. **sys_print_log 与 sys_workflow_event 共用索引命名规范**：`idx_{table}_{col}` · 索引列顺序遵循"高选择性在前"原则（`operator_user_id, printed_at DESC`）· 加 `tenant_id` 复合索引（8.3 多租户范式）
