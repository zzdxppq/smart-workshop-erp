### 1.1 端点契约（3 端点）

| # | 方法 | 路径 | 鉴权 | 说明 |
|---|------|------|------|------|
| 1 | `GET` | `/api/v1/drawings/{id}/permission` | JWT（任意角色） | 返回当前用户对该图纸的权限位（view/print/download/upload/delete）|

### 1.4 错误码（统一规范）

| HTTP | 错误码 | 场景 |
|------|--------|------|
| 403 | **40304** | 角色无图纸权限（FINANCE 全拒 / SALES 不关联订单 / OPERATOR 工序不关联） |

```sql
-- V54__drawing_acl.sql · 12.1 图纸权限矩阵

CREATE TABLE IF NOT EXISTS crm_drawing_link (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    drawing_id BIGINT NOT NULL COMMENT '图纸 ID',
    biz_type VARCHAR(20) NOT NULL COMMENT 'ORDER/PO/INCOMING/INSPECTION/WORKORDER_PROCESS',
    biz_id BIGINT NOT NULL COMMENT '业务单据 ID',
    created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    created_by BIGINT NOT NULL COMMENT '创建人',
    UNIQUE KEY uk_biz_ref (biz_type, biz_id, drawing_id) COMMENT '业务单据 + 图纸 唯一',
    KEY idx_drawing_link (drawing_id, biz_type, biz_id) COMMENT '图纸 → 业务单据 反向查询',
    KEY idx_biz_lookup (biz_type, biz_id) COMMENT '业务单据 → 图纸 正向查询',
    CONSTRAINT fk_draw_link_drawing FOREIGN KEY (drawing_id) REFERENCES crm_drawing(id) ON DELETE RESTRICT
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='V1.3.9 12.1 图纸与业务单据关联表';
```

**字段**：`id`, `drawing_id`, `biz_type` (ORDER/PO/INCOMING/INSPECTION/WORKORDER_PROCESS), `biz_id`, `created_at`, `created_by`

**索引**：
- `uk_biz_ref (biz_type, biz_id, drawing_id)` UNIQUE · 防重复关联
- `idx_drawing_link (drawing_id, biz_type, biz_id)` · 图纸 → 业务单据 反向（DrawAclService 鉴权用）
- `idx_biz_lookup (biz_type, biz_id)` · 业务单据 → 图纸 正向（5 类业务单据详情页"查看关联图纸"用）

**外键**：`fk_draw_link_drawing` ON DELETE RESTRICT · 删除图纸时若有关联单据 → 提示"请先解绑"

-- 1) 销售订单 → 图纸（crm_order_item.material_code = crm_drawing.material_code）

#### 2.3.2 DrawingAuthz.canView 内部分支（7 角色）

```java
@Component("drawingAuthz")
public class DrawingAuthz {
    @Autowired private CrmDrawingLinkMapper linkMapper;
    @Autowired private UserProcessService userProcessService; // Redis 缓存 user → current_process

    /**
     * 鉴权逻辑（顺序短路）：
     * 1. ENGINEER/PROD_PLANNER → true（全量）
     * 2. SALES → 查 crm_drawing_link WHERE drawing_id=? AND biz_type='ORDER'
     *            AND biz_id IN (sales 自己的订单)
     * 3. PURCHASER → 查 PO 关联
     * 4. WAREHOUSE → 查 INCOMING 关联
     * 5. QC → 查 INSPECTION 关联
     * 6. OPERATOR → 查 WORKORDER_PROCESS 关联且 user 持有该工序
     * 7. FINANCE → false
     */
    @Cacheable(value = "drawingAuthz", key = "#auth.name + ':' + #drawingId")
    public boolean canView(Authentication auth, Long drawingId) {
        String role = auth.getAuthorities().stream()
            .filter(a -> a.getAuthority().startsWith("ROLE_"))
            .map(a -> a.getAuthority().substring(5))
            .findFirst().orElse("");

        switch (role) {
            case "ENGINEER":
            case "PROD_PLANNER":
                return true;
            case "SALES":
                return hasLinkedBiz(auth.getName(), drawingId, "ORDER",
                    () -> orderService.findOrderIdsByUser(auth.getName()));
            case "PURCHASER":
                return hasLinkedBiz(auth.getName(), drawingId, "PO",
                    () -> purchaseOrderService.findPoIdsByUser(auth.getName()));
            case "WAREHOUSE":
                return hasLinkedBiz(auth.getName(), drawingId, "INCOMING",
                    () -> incomingOrderService.findIncomingIdsByUser(auth.getName()));
            case "QC":
                return hasLinkedBiz(auth.getName(), drawingId, "INSPECTION",
                    () -> inspectionService.findInspectionIdsByUser(auth.getName()));
            case "OPERATOR":
                String currentProcess = userProcessService.getCurrentProcess(auth.getName());
                if (currentProcess == null) return false;
                return hasLinkedBiz(auth.getName(), drawingId, "WORKORDER_PROCESS",
                    () -> List.of(Long.parseLong(currentProcess.replace("P", ""))));
            case "FINANCE":
            default:
                return false;
        }
    }

    public boolean canDownload(Authentication auth, Long drawingId) {
        return hasRole(auth, "ENGINEER");
    }

    private boolean hasLinkedBiz(String username, Long drawingId, String bizType,
                                 Supplier<List<Long>> userBizIdsSupplier) {
        List<Long> userBizIds = userBizIdsSupplier.get();
        if (userBizIds.isEmpty()) return false;
        return linkMapper.existsByDrawingAndBizTypeAndBizIdIn(drawingId, bizType, userBizIds) > 0;
    }
}
```

#### 2.3.3 性能优化

- `@Cacheable(value = "drawingAuthz", key = "#auth.name + ':' + #drawingId")` · 5min TTL
- 索引 `idx_drawing_link (drawing_id, biz_type, biz_id)` 保证查询 O(1)
- Redis 缓存 `user_id → current_process_id` 5min（OPERATOR 高速扫码场景）
