## 灵魂一致性声明（UX 计成）

**为什么是这套 UI**：CNC 一线是戴手套、站着、车间灯光不均匀的操作工；办公室是键盘密、表格多、效率优先的业务员。两套范式，一套灵魂——**能 3 步做完的绝不做第 4 步，能扫一下解决的绝不让用户找菜单**。UX 的存在，是把"图号怎么跟、工时怎么算、首件谁签"这些老师傅脑子里的隐性知识外化为可见的 UI 规则，让新人和老人能在同一张图纸上协作。

## 2. UX Goals & Principles 愿景与原则

### 2.1 UX 愿景（≤150 字）

> **"让操作工奶奶 30 秒学会扫码报工；让业务员键盘流 3 步完成报价。"**

UI 不是给人"看"的，是给人在车间"用"的。一线戴手套、光线差、站着操作；办公室键盘流、表格密、效率优先。两种范式，一套灵魂：**能 3 步做完的事绝不做第 4 步，能扫一下解决的绝不让用户找菜单**。

### 2.2 设计原则（5 条）

#### 原则 1：扫码即一切（APP 端）

**一句话**：任何业务都能在 3 步内通过"扫码 + 确认"完成。
**反例**：操作工进入 APP 后要选"业务类型 → 工序 → 工单 → 机台"四级菜单。**不做**。
**正向案例**：E5-S2 扫码开工——扫工单码 → 自动识别机台 → 按"开工"按钮（0 表单填写）。

#### 原则 2：键盘流优先（Web 端）

**一句话**：高频操作必须有快捷键，Tab 序从主到次，单据保存用 `Ctrl+S`。
**反例**：业务员录完一条报价要点 3 次鼠标才能提交。**不做**。
**正向案例**：E2-S2 报价录入——`F8` 图号联想、`Ctrl+S` 保存、`Alt+T` 含税切换、`Tab` 切明细行，键盘走完全程。

#### 原则 3：信息密度友好（Web 列表）

**一句话**：行高 32px、列宽可调、列可隐藏、过滤条件可见可改。
**反例**：每页只显示 10 条数据，要点"加载更多"才能继续。**不做**。
**正向案例**：E5-S1 工单列表默认 50 条/页，行高 32px，列可拖拽排序、可隐藏。

#### 原则 4：离线可放心（APP 端）

**一句话**：扫码后断网不卡顿、不丢数；状态条明牌告诉用户"离线多少条待上传"。
**反例**：扫码报工后断网，结果丢失、用户不知情。**不做**。
**正向案例**：E1-S4 离线模式——本地 SQLite 缓存 500 条单据 + 扫码记录；状态条 32px 顶置，橙色横幅"离线模式 5 条待上传"。

#### 原则 5：拒绝伪需求装饰

**一句话**：不做"皮肤主题切换、动画转场、骨架屏加载 800ms"等与业务无关的视觉。
**反例**：报价提交后弹 3 秒花瓣飘落动画。**不做**。
**正向案例**：保存草稿后立即 Toast 提示 + 列表行更新；无 Loading 动画（直接 disable 按钮 + 文字"保存中"）。

## 4. Usability Goals 可用性目标

### 4.1 量化指标

| 指标 | 目标值 | 测量方式 | 优先级 |
|------|--------|----------|--------|
| **操作工扫码报工完成时间** | ≤ 30 秒（P95） | 用户测试 + 埋点 | P0 |
| **业务员报价录入完成时间** | ≤ 5 分钟（P95，含 BOM） | 用户测试 + 埋点 | P0 |
| **生管排产拖拽成功率** | ≥ 95%（首次拖到正确位置） | 埋点（拖拽 → 提交） | P0 |
| **关键页面 LCP（Largest Contentful Paint）** | ≤ 2.5s（P75） | Lighthouse / Sentry | P0 |
| **扫码响应时间** | ≤ 1s（P95） | APP 端埋点 | P0 |
| **离线 → 在线自动同步成功率** | ≥ 99% | APP 端埋点 | P0 |
| **键盘快捷键覆盖率** | ≥ 80% 高频操作 | 用例清单对账 | P0 |
| **WCAG AA 合规** | 100%（关键流程） | axe-core 自动化扫描 | P0 |
| **新用户上手时间** | ≤ 30 分钟（看 1 个 5 分钟视频） | 用户测试 | P1 |
| **NPS（净推荐值）** | ≥ 40（V1.0 上线 3 个月） | 问卷 | P2 |

### 4.2 灵魂测试

| 测试名称 | 参与者 | 任务 | 通过标准 |
|----------|--------|------|----------|
| **操作工奶奶测试** | 王奶奶（55 岁，车间 10 年，不熟悉 APP） | 扫码报工：扫工单码 → 输 3 个数字 → 提交 | 30 秒内完成；不出现"找不到按钮" |
| **业务员 5 分钟测试** | 业务员新人 | 客户录入 → 报价 → 提交审批 | 5 分钟内完成 |
| **生管 1 屏测试** | 生管 | 打开生产工作台 | 5 秒内知道：今天有多少逾期、哪些设备忙 |

### 4.3 错误容忍

| 错误类型 | 恢复策略 | UI 反馈 |
|----------|----------|---------|
| 表单字段缺失 | 提交时聚焦第一个错误字段 + 红色提示 | 字段下方红字 + 字段红框 |
| 金额非法字符 | 实时过滤（仅数字 + 小数点） | 输入框光标抖动 |
| 扫码失败 | 自动重试 3 次 + 提示"请重试" | 震动 2 次 + 弹窗 |
| 网络中断 | 切换离线模式 + 状态条提示 | 顶部橙色横幅 |
| 单据已关闭 | 提示"该单据已关闭"+ 返回列表 | Toast + 自动返回 |
| 权限不足 | 按钮置灰 + 鼠标悬停提示"无权限" | Tooltip |

## 7. User Flows 用户流程

> 选取 4 个核心 Story 的关键流程作为示范。

### 7.1 流程 1：业务员录入报价（E2-S2）—— 3 步完成

#### 7.1.1 流程图

```
开始
  ↓
[1] 点击顶部菜单「销售」→「报价单」→「新建」
  ↓
[2] 填写客户（自动联想 / F2 选历史）
  ↓
[3] F8 触发图号联想 → 选图号 → 自动带出 BOM
  ↓
[4] 调整明细（数量、单价）→ Alt+T 切换含税
  ↓
[5] Ctrl+S 保存草稿 或 Ctrl+Enter 提交审批
  ↓
[6] 金额 < 5 万 → 业务员自审通过 → 结束
[6] 金额 5-20 万 → 自动路由部门经理 → 结束（待办推送给经理）
[6] 金额 > 20 万 → 自动路由总经理 → 结束
[6] 客户为黑名单 → 直接驳回 + Toast 提示
  ↓
结束
```

#### 7.1.2 关键节点

| 节点 | 触发 | UI 反馈 |
|------|------|---------|
| 选客户 | 文本框 + 联想下拉 | 客户编码 + 名称 + 信用额度 |
| F8 图号联想 | 1 字符触发远程搜索 | 下拉：图号 + 最新版本 + 缩略图 |
| 选图号 | 鼠标点击 / 回车 | BOM 自动带出 + 成本自动汇总 |
| 切换含税 | Alt+T | 金额实时重算 + 标签切换 |
| 保存 | Ctrl+S | 右上角 Toast "草稿已保存" + 单据状态变"草稿" |
| 提交 | Ctrl+Enter | 弹窗"已提交审批" + 跳转到列表页 |

#### 7.1.3 边界 case

| Case | 处理 |
|------|------|
| 客户为黑名单 | 提交时弹窗拦截 + Toast "该客户为黑名单，无法报价" |
| 金额阈值跨档 | 5 万 / 20 万边界走向上取整档（5.00 万 = 5-20 万档） |
| 客户保护期内 | 选客户时显示红色 Tag "保护中" + 提示"该客户由 X 保护" |
| 图号无 BOM | 提示"该图号未维护 BOM"+ 引导到 BOM 维护页 |
| 网络中断 | 提交失败时 Toast + 草稿本地缓存，恢复后自动重试 |

### 7.2 流程 2：操作工扫码报工（E5-S2）—— 3 步完成

#### 7.2.1 流程图

```
开始
  ↓
[1] 打开 APP，默认进入「扫码」Tab（80% 区域扫码框）

[2] 摄像头对准工单码 → 自动识别

   识别成功 → 震动 + 滴声 → 自动路由到「扫码结果页」
  ↓
[3] 选择报工（自动识别为工单码 → 默认进入报工页）
  ↓
[4] 填写 3 个数字：完工数、合格数、报废数（大数字键盘）
  ↓
   校验：合格 + 报废 = 完工 ✓
  ↓
[5] 点击"提交"按钮（高度 64px）
  ↓
[6] 提交成功 → Toast "已报工"+ 自动跳回扫码页
  ↓
结束
```

#### 7.2.2 关键节点

| 节点 | 触发 | UI 反馈 |
|------|------|---------|
| 摄像头启动 | APP 启动后 1s 内 | 扫码框 + 4 角动画 |
| 识别成功 | 摄像头解析到二维码 | 震动 200ms + 滴声 + 屏幕闪绿 200ms |
| 识别失败 | 模糊/无码 | 震动 2 次（连续）+ Toast "识别失败" |
| 报工页 | 路由切换 | 上滑动画 300ms |
| 大数字键盘 | 点击数字输入框 | 弹出系统数字键盘 |
| 校验失败 | 合格+报废≠完工 | 输入框红框 + 错误提示 |
| 报废率 > 10% | 实时计算 | 黄色提示 "报废率 12%，将触发品检介入" |
| 提交 | 大按钮 | 按钮变"提交中..." 1s → Toast + 跳回 |

#### 7.2.3 边界 case

| Case | 处理 |
|------|------|
| 扫码失败 | 自动重试 3 次 → 弹窗"请重试"+ 闪光灯按钮 |
| 未知码 | 弹窗"未知码，请确认" + 手工输入兜底 |
| 单据已关闭 | 弹窗"该工单已关闭" + 跳回扫码页 |
| 离线扫码 | 顶部橙色横幅"离线模式 5 条待上传"+ 数据入本地队列 |
| 报废率 > 10% | 提交后自动推送品检待办 + Toast "已通知品检" |
| 报工数量 > 派工数量 | 输入框红框 + 提示"超过派工数量" |

### 7.3 流程 3：生管排产（E5-S1）—— 拖拽式

#### 7.3.1 流程图

```

  ↓
[5] 点击"确认"→ 后端保存 + WebSocket 广播
  ↓
[6] 其他生管的看板同步刷新（5s 轮询 / WS 推送）
  ↓
结束
```

#### 7.3.2 关键节点

| 节点 | 触发 | UI 反馈 |
|------|------|---------|
| 拖拽开始 | 鼠标按下 + 移动 5px | 卡片半透明 + 跟随鼠标 |
| 拖拽中 | 实时计算目标位置 | 高亮可放置区（绿）/ 不可放置（红） |
| 释放 | 鼠标松开 | 弹窗确认 / 自动归位 |
| 预校验 | 后端 API 调用 | Loading 100ms 内 |
| 冲突 | 后端返回 409 | 红框 + Toast + 自动归位 |
| 成功 | 后端返回 200 | 绿色高亮 200ms + 卡片变色（草稿→已排产） |
| 实时刷新 | WS 推送 | 卡片动画插入（300ms） |

#### 7.3.3 边界 case

| Case | 处理 |
|------|------|

| 时间冲突 | 目标位置红框 + Toast "该机台当日已满载" |
| 超载预警 | 黄色进度条（> 80%）+ 可放下但提示"超载" |
| 网络中断 | 拖拽失败 → 卡片归位 + Toast "网络异常，已重试" |
| 并发冲突 | A 生管已排 → B 生管拖拽时红框 + Toast "该时段已被 X 排产" |

| 节点 | 触发 | UI 反馈 |
|------|------|---------|
| 检测离线 | 网络变化监听 | 状态条 1s 内变橙 |
| 写入本地 | 扫码提交时 | 立即 Toast + 状态条 +1 |
| 检测在线 | 网络恢复 | 状态条变蓝 + 同步动画 |
| 同步进度 | WorkManager 回调 | 状态条实时更新 X/Y |
| 冲突 | 服务端状态与本地不一致 | 弹窗二选一（覆盖/合并） |
| 同步完成 | 全部上传成功 | 绿色横幅 + 数字归零 |

#### 7.4.3 边界 case

| Case | 处理 |
|------|------|
| 本地队列 > 500 条 | 弹窗拦截"本地缓存已满，请连接网络同步" |
| 服务端拒绝 | 红色横幅"同步失败"+ 列表 + 重试按钮 |
| 时间戳异常 | 冲突弹窗 + 用户选择（覆盖/合并/取消） |
| APP 杀进程后重启 | WorkManager 唤醒同步任务 + 状态条恢复 |
| 同步中再次断网 | 暂停同步 + 状态条变橙 + 恢复后继续 |

## 11. Accessibility 无障碍

### 11.1 Compliance Target 合规目标

- **WCAG 2.1 AA**：100% 关键流程通过

### 11.2 Key Requirements 关键要求

| 维度 | 要求 | 实现方式 |
|------|------|----------|
| **对比度** | 正文 ≥ 4.5:1；大字 ≥ 3:1 | 色板已对齐（10.2 节） |

## 13. Animation & Motion 动画与动效

### 13.1 Motion Principles 动效原则

> **克制 + 目的明确**：动效只用于"反馈状态变化"，不用于装饰。

| 原则 | 描述 |
|------|------|
| **目的性** | 每个动效必须有目的（提示状态变化、引导视线） |
| **时长 ≤ 300ms** | 95% 动效在 200-300ms 内 |
| **缓动函数** | `cubic-bezier(0.4, 0, 0.2, 1)`（Material Standard） |
| **不阻挡操作** | 不在动效期间 disable 按钮 |
| **可被关闭** | `prefers-reduced-motion` 媒体查询时禁用 |

### 13.2 Key Animations 关键动效

| 场景 | 动效 | 时长 | 缓动 |
|------|------|------|------|
| Modal 打开 | 缩放 + 淡入 | 200ms | ease-out |
| Drawer 滑入 | 从右滑入 | 300ms | ease-out |
| Toast | 从顶部滑入 + 2s 后自动淡出 | 200ms + 2s + 200ms | ease-out |
| 拖拽归位 | 弹性回弹 | 200ms | spring |
| 拖拽成功 | 绿色高亮闪烁 | 200ms × 2 | linear |
| 状态变更 | 背景色过渡 | 200ms | ease-in-out |
| Loading | 旋转（24px 圆环） | 800ms 循环 | linear |
| 扫码成功 | 屏幕闪绿 + 震动 | 200ms | - |
| 看板卡片插入 | 上滑 + 淡入 | 300ms | ease-out |
| 列表行更新 | 黄色高亮闪烁 | 200ms × 2 | linear |

### 13.3 禁用动效的场景

- **数据加载**：不显示骨架屏（直接 disable 按钮 + 文字"加载中"）
- **表单提交**：不显示 Loading 动画（按钮文字变"提交中"）
- **页面切换**：不淡入淡出（直接切换）

### 13.4 `prefers-reduced-motion` 支持

```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

## 14. Performance 性能

### 14.1 Performance Goals 性能目标

| 指标 | 目标 | 测量 |
|------|------|------|
| **首屏 LCP**（Largest Contentful Paint） | ≤ 2.5s（P75） | Lighthouse / Sentry |
| **可交互 TTI**（Time to Interactive） | ≤ 3.5s（P75） | Lighthouse |
| **FID**（First Input Delay） | ≤ 100ms（P75） | Lighthouse |
| **CLS**（Cumulative Layout Shift） | ≤ 0.1（P75） | Lighthouse |
| **扫码响应** | ≤ 1s（P95） | APP 端埋点 |
| **离线同步** | ≤ 5s/条（P95） | APP 端埋点 |
| **查询接口 P95** | ≤ 2s（普通） | 后端埋点 |
| **报表 P95** | ≤ 10s（多维聚合） | 后端埋点 |

### 14.2 Design Strategies 设计策略

#### 14.2.1 Web 端

| 策略 | 描述 |
|------|------|
| **路由懒加载** | 所有页面 `() => import()` |
| **组件按需引入** | Element Plus `unplugin-vue-components` |
| **图片懒加载** | `loading="lazy"` |
| **大表格虚拟滚动** | VxeTable virtual scroll |
| **大列表分页** | 默认 50 条/页，可切换 100/200 |
| **图表按需加载** | ECharts 按需 import |
| **Web Worker** | 复杂计算（MRP 展开）放 Worker |
| **防抖/节流** | 搜索联想 300ms debounce；滚动 16ms throttle |
| **Service Worker** | V1.1 评估（V1.0 不做） |
| **CDN 加速** | 静态资源走 CDN |

#### 14.2.2 APP 端

| 策略 | 描述 |
|------|------|
| **本地缓存** | Room 缓存工单/客户/物料基础数据 |
| **图片压缩** | Glide + WebP 格式 |
| **网络切换** | OkHttp 自动重试 + 网络切换监听 |
| **弱网策略** | 减少请求频率、合并请求、本地优先 |
| **后台同步** | WorkManager 智能调度（充电/WiFi 优先） |
| **APK 体积** | ≤ 30MB |

#### 14.2.3 数据加载

| 策略 | 描述 |
|------|------|
| **骨架数据** | 首屏先渲染 10 条 + "加载更多"按钮 |
| **乐观更新** | 拖拽排产、扫码提交先更新 UI，失败回滚 |
| **预加载** | 鼠标悬停菜单项 200ms 触发预加载 |
| **缓存** | 列表数据 stale-while-revalidate |

### B.2 WebSocket / SSE 实时推送需求

| 场景 | 端 | 协议 | 频道/事件 | 频率 | 数据量 |

### B.3 文件上传分片规则

| 类型 | 大小限制 | 分片大小 | 并发数 | 重试 |
|------|----------|----------|--------|------|
| **图纸 PDF** | ≤ 100MB | 5MB | 3 | 3 次 |
| **DXF/STEP/NC** | ≤ 50MB | 5MB | 3 | 3 次 |
| **图片（头像/凭证）** | ≤ 5MB | 不分片 | 1 | 3 次 |
| **合同附件** | ≤ 20MB | 2MB | 3 | 3 次 |
| **检测报告** | ≤ 20MB | 2MB | 3 | 3 次 |

> **分片上传流程**：
> 1. 前端计算 MD5 → 调用 `/api/v1/files/init` → 返回 `uploadId`
> 2. 分片上传 → `/api/v1/files/chunk?uploadId=xxx&chunk=N`
> 3. 全部完成 → `/api/v1/files/complete` → 返回 `fileUrl`
> 4. 失败分片重试 → `/api/v1/files/chunk` 重传

### B.4 移动端网络切换/弱网策略

| 场景 | 策略 |
|------|------|
| **WiFi → 4G 切换** | OkHttp 监听 NetworkCallback，自动重试未完成请求 |
| **4G → 无网络** | 切换离线模式 + 状态条变橙 + 数据入本地队列 |
| **无网络 → 4G** | WorkManager 触发同步任务 + 状态条变蓝 |
| **弱网（< 100KB/s）** | 减少请求频率 + 合并请求 + 禁用图片上传（只缓存） |
| **超时（> 10s）** | 弹窗"网络较慢" + 允许用户取消 |
| **断点续传** | 文件分片上传支持断点续传 |
| **请求去重** | 同一请求 1s 内去重（防双击） |
