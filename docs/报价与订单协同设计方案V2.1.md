# CNC加工厂ERP系统 · 报价与订单协同设计方案（V2.1）

> 基于 V2.0 版本，加入料号生成时机与逻辑修正。

## 一、核心原则

| 序号 | 原则 | 说明 |
| :--- | :--- | :--- |
| 1 | **报价流程需要审批** | 报价是价格谈判工具，涉及定价，必须走审批流程 |
| 2 | **订单流程不需要审批** | 订单是客户已确认的指令，"订单为大"，直接生效 |
| 3 | **图号不存在时直接新建** | 订单阶段有新图号直接创建，不需要退回报价 |
| 4 | **报价数据自动带入** | 如有历史报价，系统自动带入产品信息/价格/工艺预览 |
| 5 | **工艺由工程师填写** | 无论是报价阶段还是订单阶段，工艺均由工程师定义，业务员不填工艺 |
| 6 | **订单提交后进入工程转化** | 工程师细化工艺+BOM，完成后进入待转产池 |
| 7 | **生管点「转工单」进入排产** | 工程转化完成后，生管在待转产订单列表中点击「转工单」，进入工序分配与排产 |
| 8 | **料号在销售订单阶段生成** | 报价阶段不生成料号，料号在订单提交时生成，且仅生成一次 |

## 二、料号生成时机（核心）

### 2.1 明确规则

| 阶段 | 是否生成料号 | 说明 |
| :--- | :--- | :--- |
| 报价阶段 | :x: **不生成** | 报价是价格谈判工具，可能最终不下单，此时生成料号会污染主数据 |
| **销售订单提交** | :white_check_mark: **生成** | 订单是客户已确认的指令，此时正式生成料号，进入企业生产体系 |
| 工程转化阶段 | :x: **不重复生成** | 复用订单阶段已生成的料号 |

### 2.2 料号生成逻辑

```
业务员提交销售订单
    │
    ▼
系统对订单中每个明细行执行：
    │
    ▼
检查该图号是否已有料号？
    │
    ├─── 有 ──→ 直接复用（说明该产品之前下过单）
    │
    └─── 无 ──→ 生成新料号
                    │
                    ├── 料号编码：WL-{分类前缀}-{流水号}
                    ├── 写入物料主数据（crm_material）
                    └── 关联该图号（DWG-...）
    │
    ▼
料号写入销售订单明细行
    │
    ▼
工程转化时直接引用该料号
```

### 2.3 料号生成示例

| 场景 | 图号 | 是否已有料号 | 动作 |
| :--- | :--- | :--- | :--- |
| 客户首次下单 | DWG-20260620-0001 | 无 | **生成 WL-A001** |
| 同一产品再次下单 | DWG-20260620-0001 | 有（WL-A001） | **复用 WL-A001** |
| 报价单有 10 个图号，客户只下 3 个 | DWG-001~DWG-010 | 无 | **只对 3 个下单的生成料号** |
| 报价单 A（已审批）一直未下单 | DWG-XXX | 无 | **始终不生成料号** |

## 三、角色职责划分

| 角色 | 报价阶段 | 订单阶段 | 工程转化阶段 | 排产阶段 |
| :--- | :--- | :--- | :--- | :--- |
| **业务员** | 上传图纸、填基础数据、提交工程师、提交审批 | 输入图号、确认/调整数量/交期/价格、**提交订单（无需审批）→ 系统自动生成料号** | 只读查看 | 只读查看 |
| **工程师** | **填写工艺及时间**（车床/CNC/放电/线割等工时）、确认工艺路线 | — | **工艺明细化**（详细参数）、**编制BOM**（引用已生成的料号） | — |
| **生管** | — | — | — | **查看待转产订单列表，点击「转工单」→ 工序分配 → 排产** |
| **销售经理** | 审批报价单 | — | — | — |

## 六、料号与各阶段数据关联

| 阶段 | 数据载体 | 料号状态 | 说明 |
| :--- | :--- | :--- | :--- |
| 报价阶段 | 报价单明细行 | **无料号** | 只有图号（DWG-） |
| 订单提交时 | 销售订单明细行 | **首次生成** | 生成 WL-，写入物料主数据 |
| 工程转化 | BOM | **引用料号** | 成品料号已有，子件料号可在此阶段创建 |
| 排产/生产 | 工单 | **引用料号** | 所有生产单据引用同一料号 |

## 八、状态机设计

### 8.1 销售订单状态机

```
DRAFT（草稿）
       │
       │ 业务员提交订单（无需审批）
       ▼
APPROVED（已生效）
       │
       │ 工程师开始工程转化
       ▼
PROCESSING（工程转化中）
       │
       │ 工程师完成工艺+BOM提交
       ▼
PENDING_PRODUCTION（待转产）
       │
       │ 生管点击「转工单」
       ▼
IN_PRODUCTION（已转工单）
       │
       │ 生产完成
       ▼
COMPLETED（已完成）
```

### 8.2 工单状态机（复用PRD E5-S1）

```
待排产（生管点击转工单后生成）
       │
       │ 生管排产确认
       ▼
已排产（排入甘特图）
       │
       │ 操作工扫码开工
       ▼
加工中
       │
       │ 报工完成
       ▼
已报工
       │
       │ 品检通过
       ▼
已检验
       │
       │ 成品入库
       ▼
已入库 → 已关闭
```
