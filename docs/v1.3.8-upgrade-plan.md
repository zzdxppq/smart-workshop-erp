## 2. 升级策略

### 2.1 数据库迁移

```sql
-- V49 batch 表
CREATE TABLE crm_batch (
  id BIGINT NOT NULL AUTO_INCREMENT,
  batch_no VARCHAR(30) NOT NULL COMMENT 'YYYYMMDD-流水',
  material_id BIGINT NOT NULL,
  po_id BIGINT NOT NULL,
  po_item_id BIGINT NOT NULL,
  quantity INT NOT NULL,
  arrived_at DATETIME NOT NULL,
  PRIMARY KEY (id),
  UNIQUE KEY uniq_batch_no (batch_no),
  KEY idx_material (material_id),
  KEY idx_po (po_id)
) ENGINE=InnoDB;

-- V50 物料码映射
CREATE TABLE crm_material_barcode_batch (
  id BIGINT NOT NULL AUTO_INCREMENT,
  batch_id BIGINT NOT NULL,
  material_id BIGINT NOT NULL,
  barcode_no VARCHAR(50) NOT NULL COMMENT 'WL-XXXX-YYYYMMDD-流水',
  is_active TINYINT(1) DEFAULT 1,
  PRIMARY KEY (id),
  UNIQUE KEY uniq_barcode (barcode_no)
) ENGINE=InnoDB;
```

## 3. 升级影响

### 3.1 业务影响

- 仓管：扫码从 PO 粒度改为物料粒度
- 采购：新增无订单采购入口

### 3.2 技术影响

- 数据库：+4 表
- 业务模块：+2 新模块（batch / purchase-reason）
- 升级模块：3 个（incoming / outsourcing / purchase）
