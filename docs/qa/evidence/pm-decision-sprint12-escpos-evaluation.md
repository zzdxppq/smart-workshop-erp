## 1. 背景与现状

### 1.1 客户需求

客户（昆山佰泰胜）已于 **V1.3.12 合同** 中采购 ESC/POS 票据打印机（典型用途：销售小票 / 发货单 / 委外加工单 / 品质报告单等 80mm 或 58mm 票据打印）。

**合同附录 C 硬件清单**（`docs/contract-appendix-c-hardware.md`）：
- 工业标签打印机：启邦 DL-888B（ZPL）/ 斑马 ZD420（ZPL）/ TSC TTP-244 Pro（TSPL）· V1.3.9 Sprint 12 12.2/12.4 支持
- **ESC/POS 票据打印机**（销售小票 / 发货单）：客户自购 · 80mm 或 58mm 票据 · ESC/POS 协议 · **Sprint 12 12.2 明确不支持**

### 1.2 Sprint 12 12.2 / 12.4 协议范围

| 协议 | 用途 | 12.2 是否支持 | 12.4 是否支持 | V55 protocol 枚举 |
|------|------|---------------|---------------|-------------------|
| **ZPL**（Zebra Programming Language）| 工业标签（条形码 + 二维码 + 文本）| ✅ 是 | ✅ 是 | `ZPL` |
| **TSPL**（TSC Printer Language）| 工业标签（条形码 + 二维码 + 文本）| ✅ 是 | ✅ 是 | `TSPL` |
| **ESC/POS**（Epson Standard Code for Point of Sale）| 票据 / 小票 / 收据 | ❌ 否 | ❌ 否 | （未纳入） |
| **A4 PDF**（browser print）| A4 纸型报告 / 委外加工单 | ❌ 否（输出非打印机）| ✅ 是（模式二）| `PDF_BROWSER` |

**协议依据**：
- `docs/architecture/story-reviews/12.2-12.4-review.md` §3.2 协议适配：明确仅 ZPL/TSPL
- `backend/src/main/java/com/erp/print/protocol/ProtocolAdapter.java`：抽象类仅含 ZPL/TSPL 实现
- V55 Flyway 迁移：`sys_dict` 表 `protocol` 字典项仅含 `ZPL` / `TSPL` / `PDF_BROWSER`

### 1.3 12.4 模式二 PDF_BROWSER 替代方案

12.4 双模式打印中模式二（`PDF_BROWSER`）支持：
- A4 PDF 生成（PDFBox 渲染 + 思源黑体嵌入 · 13.2 已 Sharded）
- 浏览器原生打印对话框（`window.print()`）
- 用户可选激光打印机 / 喷墨打印机 / 针式打印机 / 票据打印机（任何支持 PDF 打印的设备）

**关键事实**：客户普通激光打印机可打印 PDF → 销售小票 / 发货单可用 PDF 模式二 + 客户激光打印机打印（前提：客户接受 A4 纸型 + 切纸为 1/2 或 1/3 票据规格）。

## 2. 选项分析（3 选项 · 详尽对比）

### 2.1 选项 A · 不支持 ESC/POS（沿用 12.2 ZPL/TSPL + 12.4 PDF_BROWSER 替代）

#### 2.1.1 方案描述

- **V1.3.9 Sprint 12 IMPL 维持**：仅 ZPL/TSPL 标签 + A4 PDF 模式二
- **客户票据打印路径**：
  - 销售小票 / 发货单 → 12.4 模式二（PDF_BROWSER）→ 客户激光打印机打 PDF → 客户手工切纸或全 A4 留存
  - 委外加工单 / 品质报告单 → 同上
- **V1.3.10 backlog**：评估支持 ESC/POS（待客户 V1.3.9 灰度反馈）

#### 2.1.2 优势

- ✅ **零代码改动**：Sprint 12 12.2/12.4 IMPL 不变 · 集成 E 验证 86 测例不动
- ✅ **V1.3.9 FAT 准入无影响**：6 项委派中 #5 仅决策记录 · 不影响 IMPL
- ✅ **客户当前可用**：客户普通激光打印机 + PDF 浏览器打印已可满足 80% 票据需求（销售小票、发货单、委外加工单等可用 A4 PDF）
- ✅ **协议抽象稳定**：V55 protocol 枚举不动 · V57 code_type 枚举不动
- ✅ **合同义务满足**：V1.3.12 合同仅要求"票据打印能力" · 未强制 ESC/POS 协议

#### 2.1.3 劣势

- ❌ **用户体验降级**：销售小票需 A4 纸 → 客户成本增加（每张小票浪费 80% 纸张）
- ❌ **票据规格受限**：客户激光打印机打 A4 PDF → 无法输出 80mm / 58mm 标准小票
- ❌ **客户期望落差**：客户已购 ESC/POS 票据打印机 · V1.3.9 未启用 → 客户可能反馈"V1.3.9 没用上"
- ❌ **打印速度慢**：激光打印机 PDF 打印速度远低于 ESC/POS 票据打印机（点行打印）
- ❌ **二次切纸工序**：客户需手工切纸 · 增加工序时间

#### 2.1.4 工作量与风险

- **工作量**：0 天（仅决策记录）
- **风险**：客户 V1.3.9 灰度阶段（6/30-7/14）反馈"V1.3.9 没用上 ESC/POS 票据机" → 紧急插单 V1.3.9.1+ ESC/POS 支持
- **客户接受度**：🟡 中等（取决于客户是否愿意 A4 切纸）

### 2.2 选项 B · V1.3.10 Sprint 15 评估支持 ESC/POS 票据（新增协议）

#### 2.2.1 方案描述

- **V1.3.10 backlog 评估**：在 Sprint 15（V1.3.9 上线后）启动 ESC/POS 票据打印机支持
- **新增协议**：扩 V55 protocol 枚举（新增 `ESC_POS` 项）+ 扩 V57 code_type 枚举（新增 `ESC_POS_TICKET` 项）

## 4. 决策

### 4.1 推荐选项 · **选项 A · 不支持 ESC/POS（沿用 12.2 ZPL/TSPL + 12.4 PDF_BROWSER 替代）**

#### 4.1.1 决策依据

1. **V1.3.9 集成 E 验证 CONDITIONAL GO 闭环**
   - Sprint 12 4 Story 已 Reviewed · 86 测例待执行 · 6 项委派中 5 项待回复
   - 本决策不影响 Sprint 12 IMPL · 集成 E 验证不动

2. **12.4 模式二 PDF_BROWSER 替代方案满足客户 80% 票据需求**
   - 销售小票 / 发货单 / 委外加工单 / 品质报告单均可 A4 PDF 输出
   - 客户普通激光打印机已可打印 · 无需新增硬件
   - 客户可手工切纸或全 A4 留存（取决于客户流程）

3. **V1.3.10 backlog 候选 #1 已就位**
   - ESC/POS 票据打印机已列入 V1.3.10 backlog（`docs/orchestrix-pm-audit-2026-06-14.md` §4.2 #4）
   - 估算工时 5-7 天 · 🔴 P0 · Sprint 14 启动后 IMPL

4. **客户合同义务满足**
   - V1.3.12 合同附录 C 硬件清单仅要求"票据打印能力" · 未强制 ESC/POS 协议
   - 选项 A 满足合同要求 · 选项 B 增强体验

5. **客户反馈通道清晰**
   - V1.3.9 灰度阶段（6/30-7/14）· 客户黄梓昀 + 潘强日常反馈通道（163 邮箱 + 飞鹅 IM）
   - 若客户反馈"V1.3.9 没用上 ESC/POS 票据机" → V1.3.9.1+ 紧急插单（与选项 B 等价）

#### 4.1.2 决策结论

- **V1.3.9 Sprint 12 IMPL 维持 12.2 ZPL/TSPL + 12.4 PDF_BROWSER**
- **V1.3.10 backlog 候选 #1 保留 ESC/POS 票据打印机** · Sprint 14 启动后根据客户反馈决策
- **客户沟通**：本决策纳入 `docs/prd-feedback-v1.3.9.md` 附录 · PO 范蠡 6/15 前邮件同步客户黄梓昀 + 潘强

### 4.2 备选选项（不建议但保留）

- **选项 B**（V1.3.10 Sprint 15 评估）：若客户 V1.3.9 灰度阶段强烈反馈需 ESC/POS → 启动选项 B（与 V1.3.9.1+ 紧急插单等价）
