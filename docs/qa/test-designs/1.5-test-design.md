## 五、性能与安全测试设计

### 5.1 k6 性能基线（3 场景 · 部署阶段 docker 实测）

| 场景 | 目标 | 工具 |
|------|------|------|
| 路由 P95<50ms | 1000 并发单次 | k6 + `QuoteApprovalRouter` |
| 转订单 P95<1s | 100 并发 | k6 + `OrderConversionService` |
| PDF 导出 P95<2s | 50 并发 | k6 + OpenPDF |

### 5.2 OWASP ZAP 安全扫描（6 类越权）

| 扫描类型 | 配置 | 通过条件 |
|---------|------|---------|
| Baseline Scan | `zap-baseline.py -t http://localhost:8080 -l High` | 0 High |
| IDOR / 越权 | 业务调 admin 端点 | 40303 ACCESS_DENIED |
| 状态机越权 | DRAFT 调 `/approve` | 40904 STATE_INVALID |
| 金额越权 | 业务提交 25万 试图自审 | 路由到 gm |
| 候选人越权 | 非候选人调 `/approve` | 40303 NOT_IN_CANDIDATES |
| 审计完整性 | 任意写 → 查 `crm_quote_history` | 100% 留痕 |

### 5.3 SonarQube 4 项规则

- 无 `log.*password` / `log.*customerId` / `log.*totalAmount`
- `BigDecimal.compareTo` 而非 equals（grep `\.equals\(.*amount` → 0 命中）
- `@AuditLog` 覆盖所有 `QuoteService` 写方法
- 无 Critical 漏洞

### 5.4 V1.3.7 5 条 UI 红线（Frontend V1.3.7 §8 · Grep 验证）

- 业务自审 < 5万 列表不显示金额
- > 20万 二次密码确认（admin 密码）
- 黑名单客户高亮
- 转订单预览
- 导出审计
