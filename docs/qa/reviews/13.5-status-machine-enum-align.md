## 一、测试设计原则

| 维度 | 策略 | 理由 |
|------|------|------|
| 单元 | 不适用 | pure schema enum 调整 · 无业务逻辑 |
| 集成 | codegen + typecheck + grep | openapi.yaml → TS 类型的契约一致性 |
| E2E | 不新增（10.2 端到端覆盖） | enum 改动 ≠ 业务流转改动 |
| 性能 | codegen < 10s · typecheck < 10s | CI 增量 < 20s |
| 契约 | 与 Sprint 10.1 codegen baseline 对齐 | 端点/类型漂移是核心风险 |

> **核心策略**：3 处 enum + 1 处 description（评审新增 L3049）+ codegen regen + typecheck:ci 三重门禁
