### 测例组 1：5 类 link 真实查询（8 测例）

> 核心 SQL：V54 crm_drawing_link JOIN 5 业务单据 item 表 via material_code · 复用 12.1 已 ship 五元组

#### TC-13.3.1.1 SALES 真实查询（端点 2）

- **前置**：SALES userId=100 + crm_drawing_link 已有订单关联 · crm_order_item 含 material_code
- **步骤**：
  1. SALES userId=100 调用 `GET /api/v1/drawings/accessible?biz_type=ORDER&biz_id=123`
- **期望**：
  - HTTP 200
  - 返回 `[drawingId, drawingCode, drawingName, version, thumbnailUrl, permissionLevel]`
  - 非 mock `[10,20,30]` · 真实图纸列表
- **断言**：
  ```bash
  curl -H "Authorization: Bearer ${SALES_TOKEN}" \
    "http://localhost:8080/api/v1/drawings/accessible?biz_type=ORDER&biz_id=123" | \
  jq '.data.drawings | length'  # > 0
  ```

#### TC-13.3.1.2 PURCHASER 真实查询

- **步骤**：PURCHASER userId=200 调用 `?biz_type=PO&biz_id=456`
- **期望**：返回真实 PO 关联图纸（非 mock）
- **断言**：
  ```bash
  curl -H "Authorization: Bearer ${PURCHASER_TOKEN}" \
    "http://localhost:8080/api/v1/drawings/accessible?biz_type=PO&biz_id=456" | \

  ```

#### TC-13.3.1.3 WAREHOUSE 真实查询

- **步骤**：WAREHOUSE userId=300 调用 `?biz_type=INCOMING&biz_id=789`
- **期望**：返回真实入库单关联图纸

#### TC-13.3.1.4 QC 真实查询

- **步骤**：QC userId=400 调用 `?biz_type=INSPECTION&biz_id=321`

#### TC-13.3.1.5 OPERATOR 当前工序（端点 3 · Redis 命中）

### 测例组 2：Redis 7 缓存（5 测例）

> Key 设计：`drawing:link:{drawing_id}:{role}:{user_id}` + `user:current_process:{user_id}` · TTL 300s · 含 role 防权限漂移

#### TC-13.3.2.1 端点 2 缓存命中

- **步骤**：
  1. SALES userId=100 第 1 次查询 → DB 真实
  2. SALES userId=100 第 2 次查询（间隔 < 5min）→ 期望 Redis 命中
- **期望**：
  - 第 2 次 `cacheHit: true`
  - Redis `EXISTS drawing:link:*:SALES:100` = 1

#### TC-13.3.2.2 端点 3 缓存命中（OPERATOR 扫码）

- **步骤**：
  1. OPERATOR userId=500 扫码工序 2 次
- **期望**：
  - 第 2 次 `cacheHit: true`
  - 端到端响应 < 200ms（含扫码 + 网络 + 后端）

#### TC-13.3.2.3 TTL=300 验证

- **步骤**：
  1. `redis-cli TTL "drawing:link:123:SALES:100"`
- **期望**：295-300 秒（5min TTL · 容忍误差 ±5s）

#### TC-13.3.2.4 @CacheEvict 一致性

- **步骤**：
  1. SALES userId=100 第 1 次查询 → 缓存写入
  2. admin 触发 crm_drawing_link INSERT（`@CacheEvict allEntries=true`）
  3. SALES userId=100 下次查询 → 期望缓存清空 · 重新加载 DB
- **期望**：
  - Redis `EXISTS` = 0（清空后）
  - 第 3 次查询 `cacheHit: false`

#### TC-13.3.2.5 Redis 7 失效降级

- **步骤**：
  1. 模拟 Redis 7 故障（docker stop redis7）
  2. SALES 连续查询 4 次
- **期望**：
  - 前 3 次返回 503 错误（fail_count < 4）
  - 第 4 次退化到 DB 真实查询（200 OK）· DB QPS < 100 安全范围

### 测例组 3：性能（V58 5 条部分索引 · 4 测例）

#### TC-13.3.3.1 EXPLAIN 部分索引命中

- **前置**：V58 5 条部分索引已 ship（`idx_drawing_link_order` / `_po` / `_incoming` / `_inspection` / `_process`）

  - `key=idx_drawing_link_order`

#### TC-13.3.3.3 缓存命中 < 5ms

- **步骤**：同一 userId 第 2 次查询（Redis 命中）

## 五、风险与缓解

| 风险 | 等级 | 缓解 | 测例 |
|------|------|------|------|
| 5 类 link JOIN 大表性能 | 🟡 中 | V58 5 条部分索引 + material_code JOIN + Redis 5min 缓存 + EXPLAIN 验证 | TC-13.3.3.1/2/3/4 |
| **灰度阶段时序错位**（12.1 阶段 2-4 早于 13.3 ship）| **🔴 P0** | **SM 萧何协调**：13.3 ship 日期 ≤ 12.1 阶段 2 开启日期 · 任意延期同步顺延 | TC-13.3.4.1/2/3 |
| Redis 7 失效 | 🟢 低 | fail_count 容差 3 次 · 第 4 次退化 DB · DB QPS < 100 安全范围 | TC-13.3.2.5 |
| OPERATOR 工序查询高频（扫码 200ms 内）| 🟡 中 | Redis 5min 缓存（Key `user:current_process:{user_id}`）· 命中 < 5ms | TC-13.3.2.2/3.5.2 |
| 数据迁移回填率 < 99%（V54 crm_drawing_link）| 🟢 低 | V54 已 ship · 已知回填率 99.5% · 残余 0.5% 通过 admin 对账端点人工补录 | TC-13.3.1.6 |
| V58 部分索引 PG vs MySQL dialect 差异 | 🟡 中 | PG 原生 `WHERE biz_type='X'` 部分索引 · MySQL 5.7+ 函数索引 / 虚拟列 · `application.yml` 切分 dialect | TC-13.3.3.1 |
| Redis 缓存 Key 仅含 user_id（权限漂移风险）| 🔴 P0 | **IMPL 注意事项 #2**：Key 含 `role + user_id`（`drawing:link:{drawing_id}:{role}:{user_id}`）· 禁止仅 user_id | TC-13.3.2.1/4.4 |
| 缓存击穿（同一 userId 大量并发首次查询）| 🟢 低 | `@Cacheable(sync=true)` · 只允许一个线程查询 DB | TC-13.3.2.1 |
| 缓存雪崩（5 类 link 同时 TTL 过期）| 🟢 低 | 5min TTL 错峰（实际 4-6min 随机）· V58 migration 加 `EXPIRE key seconds 300±60` | TC-13.3.2.3 |
