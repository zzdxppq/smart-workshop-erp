## 一、测试设计原则

| 维度 | 策略 | 理由 |
|------|------|------|
| 单元 | 不适用 | 生成性任务，无业务逻辑可单测 |
| 集成 | codegen 编译 + typecheck | openapi.yaml → TS 类型的契约一致性 |
| E2E | 不新增（10.2 落 14 端点 E2E） | 避免双 E2E 维护 |
| 性能 | codegen < 5s · typecheck < 5s | CI 增量 < 10s |
| 契约 | 与 backend 132 端点计数对齐 | 端点漂移是核心风险 |

### 测例组 2：typecheck null 报错（2 测例）

#### TC-10.1.1.2.1 nullable 触发类型错误
- **前置**：在 `backend/spec/openapi.yaml` 找到 `QuoteDTO.totalAmount` 字段
- **步骤**：
  1. 临时改 `totalAmount` 字段加 `nullable: true`
  2. `npm run gen:api`（重新生成）
  3. 在 `src/views/quote/Detail.vue` 引用 `quote.totalAmount.toFixed(2)`（不守卫 null）
  4. `npm run typecheck`
- **期望**：
  - vue-tsc 退出码 2
  - 错误信息包含 "Object is possibly 'null'"
- **断言**：
  ```bash
  ! npm run typecheck 2>&1 | grep -q "Object is possibly 'null'"
  ```

#### TC-10.1.1.2.2 修复后通过
- **步骤**：在 `Detail.vue` 补守卫 `if (quote.totalAmount) { quote.totalAmount.toFixed(2) }`
- **期望**：`npm run typecheck` 退出码 0
- **断言**：
  ```bash
  npm run typecheck
  ```

## 三、CI 集成

### GitLab CI 配置

```yaml
# .gitlab-ci.yml (web-impl 仓新增)
e2e_codegen:
  stage: verify
  image: node:18
  script:
    - npm ci
    - npm run gen:api
    - npm run typecheck:ci
    - npm run build
  artifacts:
    when: on_failure
    paths:
      - playwright-report/
      - src/api/generated/
    expire_in: 7 days
  only:
    - merge_requests
    - main
```
