## 二、测例详细（8 测例）

### 测例组 1：byEventType 聚合（3 测例）

#### TC-10.3.1.1 标准 5/3/2 分组
- **前置**：`@BeforeEach` 插入 10 条 PO_APPROVAL 事件（5 CREATED + 3 APPROVED + 2 REJECTED）
- **步骤**：
  1. `GET /api/v1/workflow/events/stats?workflow_code=PO_APPROVAL`
- **期望**：
  ```json
  {
    "code": 0,
    "data": {
      "totalCount": 10,
      "byEventType": { "CREATED": 5, "APPROVED": 3, "REJECTED": 2 },
      "byApproverRole": {},
      "period": { "startDate": "2026-05-14", "endDate": "2026-06-13" }
    }
  }
  ```
- **断言**：
  ```java
  assertEquals(10, resp.getData().getTotalCount());
  assertEquals(5, resp.getData().getByEventType().get("CREATED"));
  assertEquals(3, resp.getData().getByEventType().get("APPROVED"));
  assertEquals(2, resp.getData().getByEventType().get("REJECTED"));
  ```

#### TC-10.3.1.2 含 PREVIEWED / AWARDED / DELEGATED 边界
- **前置**：插入 1 条 PREVIEWED（9.1 4.2 触发）+ 1 条 AWARDED（9.1 1.32 触发）+ 1 条 DELEGATED
- **步骤**：`GET /api/v1/workflow/events/stats?workflow_code=PO_APPROVAL`
- **期望**：byEventType 包含 5 类事件计数

#### TC-10.3.1.3 0 事件返回
- **前置**：workflow_code=NO_SUCH_WORKFLOW 无任何事件
- **步骤**：`GET /api/v1/workflow/events/stats?workflow_code=NO_SUCH_WORKFLOW`
- **期望**：
  - HTTP 200
  - `totalCount: 0`
  - `byEventType: {}`（空 Map 不为 null）
  - `byApproverRole: {}`

### 测例组 2：byApproverRole 过滤（2 测例）

#### TC-10.3.2.1 单角色过滤
- **前置**：插入 10 条事件，approver_role 分布 PROCUREMENT_MANAGER 5 + GM 3 + DEPT_MANAGER 2
- **步骤**：`GET /api/v1/workflow/events/stats?workflow_code=PO_APPROVAL&approver_role=PROCUREMENT_MANAGER`
- **期望**：
  - `totalCount: 5`
  - `byEventType` 仅包含 PROCUREMENT_MANAGER 触发的 CREATED/APPROVED/REJECTED 计数
  - `byApproverRole: { "PROCUREMENT_MANAGER": 5 }`

#### TC-10.3.2.2 复合角色值（4 阈值路由 GM+PROCUREMENT_MANAGER）
- **前置**：插入 3 条 `approver_role='GM+PROCUREMENT_MANAGER'`（4.2 路由预览产生）
- **步骤**：`GET /api/v1/workflow/events/stats?workflow_code=PO_APPROVAL&approver_role=GM%2BPROCUREMENT_MANAGER`（URL encode +）
- **期望**：
  - `totalCount: 3`
  - `byApproverRole: { "GM+PROCUREMENT_MANAGER": 3 }`（**不要拆分** · IMPL 注意 2）

### 测例组 3：时间范围（2 测例）

#### TC-10.3.3.1 默认 30 天
- **前置**：插入 31 天前 5 条 + 15 天前 5 条
- **步骤**：不传 start_date / end_date
- **期望**：`totalCount: 5`（仅 15 天前的 5 条）

#### TC-10.3.3.2 自定义范围
- **步骤**：`GET /api/v1/workflow/events/stats?workflow_code=PO_APPROVAL&start_date=2026-05-01&end_date=2026-05-31`
- **期望**：
  - `period.startDate: "2026-05-01"`
  - `period.endDate: "2026-05-31"`
  - 计数仅含 5 月份数据

#### TC-10.3.3.3 start_date > end_date 边界
- **步骤**：`start_date=2026-06-13&end_date=2026-05-01`
- **期望**：HTTP 400 · `code: 40001`

### 测例组 4：权限 + 错误码（1 测例）

#### TC-10.3.4.1 非 GM/ADMIN 鉴权
- **前置**：以 sales 角色登录（accessToken with role=SALES）
- **步骤**：`GET /api/v1/workflow/events/stats?workflow_code=PO_APPROVAL`
- **期望**：HTTP 403 · `code: 40304`（与 1.40 工序分配同码体系）
