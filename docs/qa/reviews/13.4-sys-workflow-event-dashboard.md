## 一、测试设计原则

| 维度 | 策略 | 理由 |
|------|------|------|
| 单元 | composable `useWorkflowStats` 1 测例 | 封装 loading/error/data · 验证 loading state |
| 集成 | ECharts 4 图渲染 · 端点消费 · 权限校验 | 数据链路完整闭环最后一环（写 → 存 → 聚 → 显）|
| E2E | 路由深链接 + 浏览器后退/前进 | Vue Router 4 watch route.query.tab 验证 |
| 性能 | 4 图 lazy load + tree-shaking ~150KB | 按需 import · 不影响首屏 |
| 契约 | 0 新增端点 · 消费 Sprint 10.3 既有 8/8 PASS | 与 10.3 严格对齐 |

> **核心策略**：4 子组件独立拆分（单一职责 · 便于复用）+ ECharts shallowRef + onUnmounted dispose
