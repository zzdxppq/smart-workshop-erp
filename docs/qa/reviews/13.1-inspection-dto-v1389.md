## 一、测试设计原则

| 维度 | 策略 | 理由 |
|------|------|------|
| 单元 | 不适用 | pure schema 调整 · 无业务逻辑 |
| 集成 | codegen + typecheck 双门禁 | openapi.yaml → TS 类型的契约一致性 |
| E2E | 不新增（10.2/13.3 端到端覆盖） | 避免双 E2E 维护 |
| 性能 | codegen < 10s · typecheck < 10s | CI 增量 < 20s |
| 契约 | 与 Sprint 10.1 codegen baseline 对齐 | 端点/类型漂移是核心风险 |

> **核心约束**：13.5 enum drift 必须先 ship（已 ✅）· 否则 codegen baseline 不稳 → InspectionDTO schema 联合类型漂移
