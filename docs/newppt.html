08 · BUSINESS RULES
把"老师傅心里的算盘"写成公式

计件工资：3 行公式胜过 30 页文档

通用 ERP 的薪资模块只有"基本工资+绩效"。CNC 车间要算的是合格件 × 单价 + 工时补差 − 报废扣款，还得叠加工序系数、设备系数、夜班系数。把这些规则白纸黑字写进系统，是车间不扯皮的关键。 

// 计件工资 = 合格件 × 单价 + 工时补差 − 报废扣款
WAGE =
    qualified_qty × piece_price
    + actual_hours × hour_rate
    − scrapped_qty × deduct_rate

// 工序系数：粗加工 1.0 / 精加工 1.3 / 钳工 1.5
WAGE ×= process_factor[workorder.process]

// 夜班系数：白班 1.0 / 夜班 1.2
WAGE ×= shift_factor[shift]

// 良品率预警：< 95% 自动报品检介入
IF yield_rate < 0.95
    THEN notify(quality_inspector)
// 行业经验 通用 ERP 通常只能算到"本月工资 = X 元"。

本方案能精确到："张三在工单 GD-20260604-001 第 3 道精加工工序做了 87 件合格件，合格品单价 2.5 元，工时 6.5 小时拿补差 65 元，报废 2 件扣 20 元，夜班系数 1.2，本工序工资 = 87×2.5 + 65 − 20 = 262.5 × 1.3 × 1.2 = 409.5 元。"

09 · STATE MACHINES
把"什么时候该做什么"写成状态机

工单的 7 态生命周期

通用 ERP 的"工单状态"通常只有 3 个：进行中、已完成、已关闭。CNC 工厂的状态有 7 个——这 7 个状态之间的允许转换、触发动作、回退条件都必须严格定义，否则就会出现"工单已关闭但还在扫码报工"的脏数据。 

[已创建] ──创建工单──▶ [待排产]
                          │
                          │ 生管分配机台
                          ▼
                      [已排产] ──操作工扫码──▶ [加工中]
                                                       │
                                                       │ 扫码报工
                                                       ▼
                                                [已报工] ──品检通过──▶ [已检验]
                                                                       │
                                                                       │ 扫码过站 → 末道 → 入库
                                                                       ▼
                                                                [已入库] ──发货完成──▶ [已关闭]

# 关键规则：
✗ 加工中 → 已关闭     # 禁止跳关（强制走"已入库"）
✗ 已报工 → 加工中       # 禁止回退（需走返工流程，建新工单）
✓ 加工中 → 加工中       # 允许切换工序（生成新扫码记录）
✓ 已报工 → 已报工       # 允许追加报工（数量合并，自动校验合格+报废=完工）

离线扫码：500 条缓存 + 自动重试
