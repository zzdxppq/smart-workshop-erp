### 1.2 本次反馈两大新需求（V1.3.4 核心）

#### 需求 A：委外订单全流程深化（含工序委外 + 整单委外区分 + 7 状态机 + 返修闭环）

客户原话精简："送货员到工厂打开 APP 选订单 → 点送达 → 扫工厂门口 / 待检区二维码 → 弹确认框 → 系统状态变'待检' → 自动通知品管 + 生管 → 后续质检 / 入库 / 退货维修 / 已通知返修 → 同一委外单返修 ≥ 2 次自动预警高层。"

补充要求："工序委外订单比较多，业务流程要做完整；要复用三码（GD/LZ/SB）；生管要能看到委外是否完成 + 下一道工序排程；委外历史交期要能预估；PC 端要能一屏看所有委外订单。"

#### 需求 B：料号价格 / 成本检索

客户原话："客户想通过直接搜料号，查特定料号的价格和成本明细，材料成本整个明细。"

补充要求："PC + APP 双端；权限隔离（一线只看自己工序工时，管理层看全量）；支持 Excel/PDF 导出；要看到价格 / 材料 / 工时 / 表处 / 外协 / 管理费分摊 / 总成本完整结构。"

## 五、灵魂一致性自评（PM 范蠡）

> **范蠡本次 PRD 深化是否保持了"一码到底、一数到底"灵魂？**
>
> **一句话回答**：**一码到底从"物料"扩展到"物理空间"（AREA- 区域码贯穿厂内每一寸土地），一数到底从"订单"扩展到"料号生命周期"（5 Tab 成本结构穿透每张订单背后的"这料赚不赚"），灵魂不仅一致，且被两次放大**。
>
> **自评分数**：9.2 / 10（V1.3.3 为 9.0，本次 +0.2）
> **加分项**：
> - **委外 7 状态机 + 区域码 = 把厂外送货员到厂的每一步"无形交接"全部码化**，彻底消灭"打电话叫人"和"门卫等 20 分钟"两个老问题
> - **料号成本聚合 + 5 Tab 面板 = 让老板真正能"用数据驱动决策"**，而不是"凭感觉觉得这料赚"
> - **返修次数 ≥ 2 自动预警高层 = 把"事后总账"变"事中干预"**，与 V1.3.3 利润预警形成"事中干预双引擎"
>
> **扣分项**：
> - 返修预警的 SMS 通道默认走企业微信 webhook，真 SMS 需另签约（已列入开放问题 §8.4 #9，暂扣 0.5 分）
> - 料号成本面板 36 个月以上历史查询需要冷热分层（已列入开放问题 #10，暂扣 0.3 分）
