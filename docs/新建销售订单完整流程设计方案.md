## 一、核心原则

| 序号 | 原则 | 说明 |
|------|------|------|
| 1 | **订单来源图纸库** | 新建销售订单时，业务员从图纸库选择客户图号；图号经报价流程沉淀完整产品数据，**本流程不允许新建/上传图号** |
| 2 | **订单不需要审批** | 订单是客户已确认的指令，「订单为大」，提交后直接生效 |
| 3 | **图纸库数据自动带出** | 选图号后自动带出材质、尺寸、重量、工艺路线预览 |
| 4 | **业务员确认/调整数量、交期、价格** | 数量、交期、单价由业务员确认（单价调整需销售经理权限） |
| 5 | **交期在订单阶段录入** | 销售订单为正式订单，**期望交期必填** |
| 6 | **订单提交后进入工程转化** | 提交后工程师收到待办，进行工艺细化 + BOM |
| 7 | **料号在订单提交时生成** | 提交时按系统图号生成/复用 WL 料号，报价阶段不生成 |

---

## 二、角色职责

| 角色 | 职责 | 操作页面 |
|------|------|----------|
| **业务员** | ① 选择客户 ② 从图纸库搜索/选择客户图号 ③ 确认数量、交期、价格 ④ 提交订单 | 销售 → 销售订单 → 新建 |
| **系统** | ① 自动生成订单号 ② 从图纸库带出产品数据 ③ 提交时生成/复用料号 ④ 通知工程师 | 自动 |
| **工程师** | ① 收到订单待办 ② 工程转化（细化工艺 + BOM） | 工程 → 订单工程转化 |

## 三、图纸库数据说明

图纸库图号已包含完整产品数据，供订单直接引用：

| 数据项 | 字段 | 说明 |
|--------|------|------|
| 客户图号 | `crm_drawing.customer_drawing_no` | 客户图纸编号 |
| 系统图号 | `crm_drawing.drawing_no` | 系统内部唯一标识 |
| 产品名称 | `crm_drawing.title` | 产品名称 |
| 材质 | `material_grade` / 关联物料 | 材质信息 |
| 尺寸规格 | `spec_size` | 产品尺寸 |
| 单件重量 | `unit_weight` | 单件重量(kg) |
| 工艺路线 | `process_route` | 已确认工艺路线 JSON/文本预览 |

> 图纸库图号均已走过报价流程，数据完整可信；订单选用时 **禁止上传新图纸**（`DrawingPicker allowUpload=false`）。

## 四、完整销售订单流程

### 4.1 流程总览

```
客户正式下单
    ↓
① 业务员打开「新建销售订单」
    ↓
② 选择客户
    ↓
③ 从图纸库搜索客户图号（customer_drawing_no 模糊匹配）
    └── 选用后自动带出：系统图号、产品名称、材质、尺寸、重量、工艺预览
    ↓
④ 业务员填写/确认：数量、交期（必填）、单价、需求描述
    ↓
⑤ 提交订单（无需审批）
    ├── 生成订单号 XS{yyyyMMdd}-{seq}
    ├── 按系统图号生成/复用料号 → 写入明细 + 物料主数据
    ├── 状态 DRAFT → APPROVED
    └── 创建工程转化工作台 → 通知工程师
    ↓
工程师工程转化 → PENDING_PRODUCTION → 生管转工单
```

### 4.2 与报价流程边界

| 阶段 | 能否上传新图 | 是否填交期 | 是否生成料号 | 是否审批 |
|------|-------------|-----------|-------------|---------|
| 报价 | ✅ 可上传 | ❌ 不填 | ❌ 不生成 | ✅ 多级审批 |
| **销售订单** | ❌ 仅图纸库选用 | ✅ 必填 | ✅ 提交时生成 | ❌ 无需审批 |

## 五、页面详细设计

### 5.1 新建销售订单页面

- 路由：`/sales/orders/new`
- 组件：`OrderForm.vue`
- 按钮：`保存草稿` | `提交订单（无需审批）`
- 明细：`DrawingPicker`（`allowUpload=false`）按行添加
- 底部：料号预检表（调用 `GET /orders/check-material-nos`）
- 工艺预览区：只读，来自图纸 `process_route`

### 5.2 关键交互

| 操作 | 系统行为 |
|------|----------|
| 输入客户图号搜索 | `GET /drawings?keyword=` 搜 `customer_drawing_no` + 系统图号 |
| 选用图号 | 带出材质/规格/重量/工艺；**不可改产品信息** |
| 调整单价 | 业务员只读；销售经理/管理员可改 |
| 提交前 | 校验交期、明细含 `drawing_id`、数量>0、单价≥0 |

## 六、状态机

```
DRAFT（草稿）
    ↓ POST /orders/{id}/submit（无需审批）
APPROVED（已生效）
    ↓ 工程师开始工程转化
PROCESSING（工程转化中）
    ↓ 工程师完成工艺+BOM
PENDING_PRODUCTION（待转产）
    ↓ 生管「转工单」
IN_PRODUCTION（已转工单）
    ↓ …
CLOSED / CANCELLED
```

兼容旧状态：`CONFIRMED` / `PRODUCING` 等仍保留读取。

## 八、API 设计（V2.1 订单流）

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/orders` | 创建草稿 |
| PUT | `/orders/{id}/draft` | 保存草稿（含明细行） |
| POST | `/orders/{id}/submit` | 提交：交期校验 + 料号生成 + APPROVED + 工程师待办 |
| GET | `/orders/check-material-nos?drawingNos=` | 料号预检（复用/将新建） |
| GET | `/drawings?keyword=` | 图纸库搜索（含客户图号） |
