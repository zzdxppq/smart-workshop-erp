**触发器与通知规则（V1.3.7 改版：所有"通知厂商"统一改为"163 邮箱"，删除短信列）**：

| From | To | Trigger | 通知对象 |
|------|----|---|---------|
| PENDING_SHIP | SHIPPING | 厂商电话/系统外接单 + 仓管代录（V1.3.5 无送货员 APP） | **厂商：163 邮箱**（V1.3.7 单渠道，V1.3.6 短信列删除；含 PDF 委外订单详情）+ 系统记录 |
| SHIPPING | PENDING_INSPECTION | **仓管在 APP "到货扫码"扫委外单码（V1.3.5 新）** | **生管 + 品质**（APP + 企业微信 + PC 红点） |
| PENDING_INSPECTION | INSPECTING | **品质员去仓库找仓管领料（V1.3.5 简化）** | 生管（CC 通知） |
| INSPECTING | QUALIFIED_STORAGE | 品管录入"合格" | 仓管员（APP） |
| INSPECTING | REPAIR_REQUESTED | 品管录入"不合格 + 退回维修" | 采购员（APP + 企业微信）+ **厂商：163 邮箱**（V1.3.7 单渠道，V1.3.6 短信列删除；通知"质检不合格 + 不良原因"） |
| QUALIFIED_STORAGE | STORED | **仓管扫物料码 WL- + 选库位 + 录良品数（V1.3.5 收回区域码）** | 生管（触发工序接续 E5-S6）+ 财务（对账归集 E6-S1） |
| REPAIR_REQUESTED | NOTIFIED_REPAIR | 采购生成返修单 | **厂商：163 邮箱**（V1.3.7 单渠道，V1.3.6 短信列删除）+ 生管 |
| NOTIFIED_REPAIR | SHIPPING | 厂商再送货（is_rework_reinspection=true） | 仓管（待下次扫码） |
| _ANY_ → 返修次数 ≥ 2 | _ANY_ | 自动触发 | **高层 + 采购主管 + 生管主管（APP + PC 红点 + 企业微信 webhook）** |

### 附录 B-3：月度对账状态机详图（V1.3.6 新增 · 线下 + 拍照上传）

```
                          ┌──────────────────┐
                          │ 采购 PC 端        │
                          │ "生成对账单"     │
                          └────────┬─────────┘
                                   │ 系统自动汇总当月已 STORED 委外单
                                   │ 生成对账单 PDF（含厂商签字栏位）
                                   ↓
                          ┌──────────────────┐
                          │ 待对账            │  邮件发厂商（含 PDF）
                          │ (DRAFT → SENT)    │  + 短信提醒（V1.3.6）
                          └────────┬─────────┘
                                   │ 采购线下带纸去厂商处
                                   │ 厂商核对 + 签字盖章
                                   │ 采购拍照/扫描上传系统
                                   ↓
                          ┌──────────────────┐
                          │ 对账已确认         │  通知财务 + 生管
                          │ (CONFIRMED)       │  签字扫描件 AES-256 入库
                          └────────┬─────────┘
                                   │ 财务校验：双方金额一致
                                   │ 点"生成付款申请"
                                   ↓
                          ┌──────────────────┐
                          │ 付款中            │  走原有 >10万双签流程
                          │ (PAYING)          │  （E9-S3）
                          └────────┬─────────┘
                                   │ 财务执行付款
                                   ↓
                          ┌──────────────────┐
                          │ 已付款            │  终态 + 通知厂商（邮件+短信）
                          │ (PAID)            │
                          └──────────────────┘

                                   ↺ 异常分支：
                          ┌──────────────────┐
                          │ 已驳回            │  对账金额不一致 / 附件缺失
                          │ (REJECTED)        │  → 退回采购重对
                          └──────────────────┘
```
