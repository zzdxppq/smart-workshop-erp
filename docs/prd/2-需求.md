# 2. 需求

### 2.1 功能性需求（FR1...FR60+）

> 编号规则：FR-{Epic}-{Story}-{AC序号}，例如 FR-2-1-3 表示 Epic 2 客户与销售、Story 1 客户档案、第 3 条 AC。

#### Epic 1 基础设施与权限

- **FR-1-1 用户与角色权限**
  - FR-1-1-1 管理员可创建/编辑/禁用用户，用户名全局唯一、密码 ≥ 8 位含大小写数字。
  - FR-1-1-2 角色支持菜单权限（哪些菜单可见）、数据权限（按本人/本部门/全部）、操作权限（增/改/删/审/导/打）。
  - FR-1-1-3 金额权限：业务员、经理、总经理三类内置角色，金额阈值在角色管理可调。
  - FR-1-1-4 部门/职位与角色解耦：同一个人可同时是业务员（业务侧）和报价员（技术侧），按模块取角色并集。
- **FR-1-2 审批工作流配置**
  - FR-1-2-1 审批流支持节点（审批人/角色/发起人自选/部门经理）+ 条件分支（金额/客户类型/订单类型）。
  - FR-1-2-2 内置四套模板：报价审批、订单审批、采购审批、付款审批；可复制改。
  - FR-1-2-3 审批超时提醒：待办超过 24h 未处理，系统消息 + 邮件提醒（邮件 V1.1）。
- **FR-1-3 系统参数与数据字典**
  - FR-1-3-1 数据字典：物料分类、工序类型、表处类型、班别、仓库、币种等。
  - FR-1-3-2 系统参数：单据编号规则（年/月/日/流水）、打印模板、APP 端离线缓存时长。
  - FR-1-3-3 金额阈值全局配置：报价/订单/采购/付款阈值集中管理，变更走变更日志。
- **FR-1-4 APP 端基础（登录/消息/扫码壳）**
  - FR-1-4-1 APP 支持账号密码登录 + 记住密码 + 指纹解锁（V1.0）+ 生物识别（V1.1）。
  - FR-1-4-2 消息中心：审批通知、逾期提醒、异常上报、扫码记录回执；离线消息本地缓存，上线后同步。
  - FR-1-4-3 扫码壳：统一扫码入口，扫到的码自动识别类型（工单码/流转码/设备码/物料码）并路由到对应业务页。
  - FR-1-4-4 离线模式：本地 SQLite 缓存单据与扫码记录，本地时间戳 + 上传时冲突解决（以服务端最新状态为准，冲突弹窗提示用户选择"覆盖"或"合并"）。

#### Epic 2 客户与销售

- **FR-2-1 客户档案与 CRM**
  - FR-2-1-1 客户档案字段：客户编码（自动生成 KH+年月日+3 位流水）、名称、行业分类、信用额度（默认 0）、开票信息、地址、负责人、保护状态。
  - FR-2-1-2 客户保护机制：业务员领用后进入保护期（默认 30 天可配），保护期内他人只读；管理员可强制收回。
  - FR-2-1-3 洽谈进展：跟进时间、跟进内容、进展阶段（初步接触/需求确认/报价中/议价中/已签约/已流失）、下次跟进时间。
  - FR-2-1-4 客户共享：跨部门协作，主负责不变，被共享人拥有读写权限。
- **FR-2-2 报价与多级审批**
  - FR-2-2-1 报价单字段：报价单号（BJ+年月日+4 位流水）、客户、交期、币种、明细（图号/材质/尺寸/数量/单价/金额/是否FA/是否新件）、审批状态。
  - FR-2-2-2 审批阈值：单笔金额 < 5 万 → 业务员自审；5-20 万 → 部门经理；> 20 万 → 总经理；客户为黑名单直接驳回。
  - FR-2-2-3 报价单导出 PDF（带公司 logo + 报价明细 + 审批签字栏）+ Excel。
  - FR-2-2-4 报价转订单：审批通过后一键转订单，图纸/明细/数量/单价自动带出，可调整后下单。
- **FR-2-3 订单与合同**
  - FR-2-3-1 销售订单字段：订单号（XS+年月日+4 位流水）、客户、交期、订单类型（标准/急单/试制）、是否 FA、合同附件、审批状态。
  - FR-2-3-2 订单审批阈值：同报价；新增信用额度校验：订单未收金额 + 本次金额 > 信用额度 → 升级总经理审批。
  - FR-2-3-3 订单变更：数量/交期/工艺变更，变更前后差异对比，变更后自动通知关联生管/客户。
  - FR-2-3-4 订单状态机：草稿→待审→已审（生产中）→部分发货→已发货→已结算→已关闭→已取消。
- **FR-2-4 合同回款与利润**
  - FR-2-4-1 回款计划：按合同条款自动分期（例 30%/40%/30%），每期计划回款日期与金额。
  - FR-2-4-2 回款控制：到期前 3 天消息提醒业务员/财务；逾期 7 天升级到总经理。
  - FR-2-4-3 合同利润核算：成本（材料+人工+委外+费用）= 总收款 - 利润；利润率自动计算。

#### Epic 3 图纸与物料

- **FR-3-1 图纸与版本管理**
  - FR-3-1-1 图纸档案：图号（客户图号 + 厂内图号）、版本号（v1.0/v1.1）、文件路径、PDF/原始格式、关联业务单。
  - FR-3-1-2 版本管理：每次变更生成新版本，旧版本可查不可改；变更说明必填（≥ 10 字符）。
  - FR-3-1-3 在线预览：PDF 在线查看，支持缩放/翻页/打印（无下载）。
- **FR-3-2 工程转化**
  - FR-3-2-1 客户图纸 → 厂内图纸：复制原图，新增厂内标注（加工要求/材料/工艺路线），保存为新版本。
  - FR-3-2-2 CAD/CAM 关联：图纸支持挂载 .dxf/.step/.nc 文件，加工车间可下载。
- **FR-3-3 BOM 多级维护**
  - FR-3-3-1 物料编码：分类（原材料/半成品/成品/辅料/包材）+ 料号 + 尺寸规格；例 "YL-45-D80" 表示圆料 45#钢 直径 80mm。
  - FR-3-3-2 BOM 支持不限层级（多级子件）；损耗系数按层级独立维护。
  - FR-3-3-3 设计 BOM 一键转生产 BOM：复制为生产版本，工程师微调后发布。
- **FR-3-4 工艺库与工序**
  - FR-3-4-1 工序库：工序名称（车/CNC/铣/钳/表处/热处理/三次元/清洗/包装）、标准工时、设备类型、单价。
  - FR-3-4-2 工艺路线：从工序库选择工序，按顺序组装为产品工艺路线；支持复制历史工艺。
- **FR-3-5（V1.3.5 已删除）固定区域二维码管理**——V1.3.4 引入，V1.3.5 客户反馈"取消区域码"整章回退。详见 §3.4 委外加工管理 · 工序分配职责（V1.3.7 新增）+ E12-S1/S2 仓管到货扫码。

#### Epic 4 扫码仓储一体化

- **FR-4-1 物料条码生成**
  - FR-4-1-1 入库时自动生成物料码（编码规则：物料编码+批次号+入库日期+流水）；支持一维码/二维码。
  - FR-4-1-2 批量打印：选中多个物料码一键打印（按 50mm×30mm 标签纸）。
- **FR-4-2 APP 扫码出入库**
  - FR-4-2-1 扫码入库：扫描物料码 → 选入库类型（采购/生产/委外/其他）→ 选库位 → 录入数量 → 提交。
  - FR-4-2-2 扫码出库：扫描物料码 → 选出库类型（领料/发货/委外/其他）→ 选库位/批次 → 录入数量 → 提交。
  - FR-4-2-3 离线策略：本地缓存单据（最多 500 条），本地时间戳 + 上传时冲突解决。
- **FR-4-3 库位批次与多仓库**
  - FR-4-3-1 多仓库：仓库/库区/库位三级；库位编码规则 W-A-01-03（一号仓库 A 区 01 排 03 位）。
  - FR-4-3-2 批次管理：批次号规则 YYYYMMDD-流水；支持先进先出；批次追溯。
- **FR-4-4 安全库存与预警**
  - FR-4-4-1 安全库存：物料维度配置上下限；库存低于下限触发预警。
  - FR-4-4-2 库存查询：实时库存 = 在库 + 在途 - 冻结；按物料/仓库/批次/库位多维查询。

#### Epic 5 生产执行核心

- **FR-5-1 工单与排产**
  - FR-5-1-1 工单：销售订单自动转工单；字段：工单号（GD+年月日+4 位流水）、销售订单、产品、数量、计划开始/结束、工序路线、状态。
  - FR-5-1-2 工单状态机：草稿→待排产→已排产→加工中→已报工→已检验→已入库→已关闭→已取消。
  - FR-5-1-3 排产：拖拽式排产（Gantt 视图）；按机台维度看负荷。
  - FR-5-1-4 物料需求分析（MRP）：工单 BOM 展开 → 库存比对 → 生成缺料清单 → 触发采购申请。
- **FR-5-2 扫码开工/报工/过站**
  - FR-5-2-1 扫码开工：扫工单码 → 自动识别操作人/机台 → 记录开工时间；工单状态变"加工中"。
  - FR-5-2-2 扫码报工：扫工单码 → 录入完工/合格/报废数量 → 校验合格+报废=完工 → 提交；自动通知品检。
  - FR-5-2-3 扫码过站：扫流转码（区别于工单码）→ 录入下工序/交接数量/交接人 → 生成下工序开工触发。
  - FR-5-2-4 三码区分（**V1.3.4 升级为四码 / V1.3.5 收回区域码 · 回到三码**）：
    - **工单码 GD-**：贴在工单卡，记录加工人与时间。
    - **流转码 LZ-**：上工序报工合格后自动生成，贴在物料包装，记录物料去向。**工序委外时每道委外工序生成独立 LZ-，贯穿厂内外**（V1.3.4）。
    - **设备码 SB-**：贴在设备，快速录入机台号，可选。
    - ~~**区域码 AREA-（V1.3.4 新增，V1.3.5 已删除）**~~：V1.3.5 客户反馈"取消区域码"——区域码失去服务对象。改由仓管扫**委外订单二维码 WW-**（V1.3.5）触发到货（详见 E12-S2）。
  - FR-5-2-5 报工校验：报工数量 ≤ 派工数量；合格 + 报废 = 完工；报废率 > 10% 触发品检介入。
- **FR-5-3 委外下单基础**（V1.3.4 拆清整单 vs 工序委外 / **V1.3.7 严格分离生管/采购分工**）
  - **工序分配职责（V1.3.7 新增，严格分离生管 vs 采购）**：
    - **生管职责（工序划分）**：看订单的工序清单 → 决定每道工序是「厂内自制」还是「委外」→ 在工单上勾选/标记每道工序归属 → **不指定厂商**。
    - **采购职责（厂商选择）**：接收生管提交的"待委外工序清单" → **为每道委外工序选择厂商**（按能力 / 价格 / 历史 / 交期） → 创建委外单（WW- 单号 + 二维码）。
    - **V1.3.6 之前描述模糊**：原写"业务/生管划分、采购选厂商"，现严格区分——**生管只决定自/委，不选厂商；采购只选厂商，不决定自/委**。
  - FR-5-3-1 外协厂商：厂商档案、加工能力（表处/热处理/三次元）、价格协议、信用等级、**近 3 次平均交期（V1.3.4 新增，用于交期预估）**、**邮箱（V1.3.6 必填 → V1.3.7 仍必填，对账单附件接收）**、**电话（V1.3.6 必填 → V1.3.7 改选填，163 邮箱通知后短信不再需要）**、**通知偏好 [email_163]（V1.3.7 改固定为单一 163 邮箱渠道）**、**默认对账邮箱（V1.3.6 新增，可与厂商邮箱不同）**。
  - FR-5-3-2 委外下单：**V1.3.7 流程改版**——生管完成工单工序划分（厂内/委外）→ 系统生成「待委外工序清单」→ 推送给采购 → 采购在 PC 端"委外管理"模块选厂商 + 自动带出历史价（如有）+ **历史交期（V1.3.4）** + 录入数量/单价/交期 → 审批 → 创建委外单 → **自动发 163 邮箱通知厂商（V1.3.7，Nacos `app.outsub.notify-channel = ['email_163']`）**。
  - FR-5-3-3 委外出库/入库：参照 FR-4-2 扫码出入库流程，关联委外订单。
  - **FR-5-3-4（V1.3.4 新增 / V1.3.5 强化）整单委外 vs 工序委外**：
    - **整单委外**：整张工单全部工序由厂商负责，走 E6-S5 的 7 状态机完整流程；**V1.3.5 强调**：即使整单委外合格回厂，**也必须接续后续厂内工序**（入库 / 包装 / FA），不再"直接入成品库"。
    - **工序委外**：工单按工序切片（如 GD-001 第 3 道精加工委外），仅该工序走 7 状态机；委外完成后**自动流转到下一道工序**（无论厂内/厂外，详见 E5-S6 强化）。每道委外工序生成独立 LZ-流转码（如 LZ-GD001-P03）。
- **FR-5-4 设备与机台排产**
  - FR-5-4-1 设备台账：设备编码、名称、类型、机台号、状态（空闲/加工中/维护/故障）、维护周期。
  - FR-5-4-2 机台负荷：按日期展示机台已排工时与可用工时；超载预警。

#### Epic 6 委外加工深化（V1.3.4 大幅升级）

- **FR-6-1 委外对账与付款（V1.3.6 改版 → V1.3.7 简化描述）**
  - FR-6-1-1 委外对账：按厂商+月份汇总委外订单；核对加工数量、单价、运杂费；**生成对账单 PDF**（含厂商签字栏位 / 空白）。
  - FR-6-1-2 对账单确认：**V1.3.7 简化为"邮件发对账单 → 厂商线下核对 → 采购上传签字扫描件"**——采购在 PC 端"委外对账"模块点"发送对账单邮件" → 系统按 163 邮箱 SMTP 推送对账单 PDF 至厂商档案"默认对账邮箱"（**厂商线下流程不在 PRD 描述**） → 厂商核对后通过任意方式（微信 / 邮件 / 线下带回等）将签字盖章版交回采购 → 采购**拍照或扫描上传**签字版（PDF/JPG/PNG，≤10MB）→ 系统状态置"对账已确认"；**无供应商 APP 端，无任何线上对账入口**。
  - FR-6-1-3 一键生成付款：**V1.3.6/V1.3.7 维持触发条件**——必须同时满足"对账已确认 **+** 签字扫描件已上传 **+** 双方对账金额一致"才可生成付款申请，关联财务应付。
- **FR-6-2 工序/整单委外灵活切换**
  - FR-6-2-1 工序级委外：单工序委外，工单其他工序仍在厂内；**每道委外工序独立 LZ-（V1.3.4）**。
  - FR-6-2-2 整单委外：整个工单外协，厂内不加工。
  - FR-6-2-3 自制 ↔ 委外切换：产能变化时可一键转换；切换记录留痕。
- **FR-6-3 自动调取委外历史价**
  - FR-6-3-1 委外下单时自动带出该厂商该工序最近 3 次成交价（取中位数）。
  - FR-6-3-2 历史价为空时显示"暂无历史价"，提示询价。
- **FR-6-4 委外质检**
  - FR-6-4-1 委外入库自动触发质检任务；不合格品走不良品流程。
- **FR-6-5（V1.3.4 新增）委外订单 7 状态机**
  - FR-6-5-1 7 主状态：`PENDING_SHIP 待发货` / `SHIPPING 送货中` / `PENDING_INSPECTION 待检` / `INSPECTING 质检中` / `QUALIFIED_STORAGE 待入库` / `STORED 已入库` / `REPAIR_REQUESTED 待返修`（含 `NOTIFIED_REPAIR 已通知返修`）。
  - FR-6-5-2 触发条件与转换矩阵详见 §附录 B-2「委外订单 7 状态机速查表」。
  - FR-6-5-3 返修品复检字段 `is_rework_reinspection`（默认 false，返修后再送货时为 true）。
- **FR-6-6（V1.3.4 新增）返修闭环 + 次数预警**
  - FR-6-6-1 返修单 `rework_order` 关联原委外单 `original_outsub_order_id`；返修次数 `rework_count` 自动累加。
  - FR-6-6-2 阈值 `app.outsub.rework-alert-threshold`（默认 2，Nacos 热更新）；≥ 阈值 → APP 推送 + PC 红点 + 短信（企业微信 webhook 可选）；接收对象：高层 + 采购 + 生管。
- **FR-6-7（V1.3.4 新增）委外历史交期预估**
  - FR-6-7-1 复用 E3-S3 历史工艺库思路；`estimated_completion = order_date + median(recent_3_orders.delivery_days)`。
  - FR-6-7-2 委外订单详情页显示"预估完成时间"+"历史交期 50%/80%/100% 分位数"。

#### Epic 7 品质管控

- **FR-7-1 来料/过程/成品检验**
  - FR-7-1-1 检验方案：按物料/工序配置检验项目（尺寸/外观/硬度等）、公差、抽检比例。
  - FR-7-1-2 来料检：采购到货后扫码生成来料检单；合格入库，不合格退货/让步接收。
  - FR-7-1-3 过程检：关键工序后自动提醒品检；支持巡检录入。
  - FR-7-1-4 成品检：成品入库前终检；合格入库。
- **FR-7-2 FA 首件**
  - FR-7-2-1 FA 标识：订单/工单勾选"是否 FA"；新件首件自动触发。
  - FR-7-2-2 FA 检验单：检验项目 + 签字栏；品检 + 工程师双签确认合格后批量生产。
- **FR-7-3 三次元检测**
  - FR-7-3-1 检测任务：工单/订单可指派三次元检测；上传检测报告（PDF/Excel）。
  - FR-7-3-2 检测结果关联工单：合格/不合格/让步接收。
- **FR-7-4 不良品处理**
  - FR-7-4-1 不良品记录：数量、原因分类（材料/工艺/设备/人为）、处理方式（返工/报废/让步）。
  - FR-7-4-2 返工自动重排产；报废扣减库存并记成本。

#### Epic 8 采购管理

- **FR-8-1 询价比价**
  - FR-8-1-1 询价单：物料 + 数量 + 期望交期 → 选供应商（≥ 2 家） → 发送。
  - FR-8-1-2 报价对比：供应商报价自动对比（最低/最高/均价），支持议价记录。
  - FR-8-1-3 历史价：自动显示该物料近 3 次采购价。
- **FR-8-2 价格控制**
  - FR-8-2-1 限价规则：物料维度配置建议进价上限；高于限价需采购主管审批。
  - FR-8-2-2 审批阈值：单笔 < 1 万采购员、1-5 万采购主管、> 5 万总经理。
- **FR-8-3 到货提醒**
  - FR-8-3-1 采购订单约定交期前 3 天 + 当天提醒；逾期 1/3/7 天升级提醒。
  - FR-8-3-2 提醒列表：到货日期/产品/数量/关联单据/负责人。
- **FR-8-4 来料质检**
  - FR-8-4-1 质检方案：按物料配置（抽检/全检/免检）；批量生成质检单。
  - FR-8-4-2 检验结果：合格入库/不合格退货/让步接收。

#### Epic 9 财务与成本

- **FR-9-1 应收应付**
  - FR-9-1-1 应收账款：销售订单 → 自动生成应收；按客户 + 账期管理。
  - FR-9-1-2 应付账款：采购/委外订单 → 自动生成应付；按供应商 + 账期管理。
  - FR-9-1-3 账龄分析：30/60/90/90+ 天分级。
- **FR-9-2 成本核算**
  - FR-9-2-1 工单成本：材料 + 人工（计件/计时）+ 制造费用 + 委外费用 = 总成本。
  - FR-9-2-2 人工公式：计件工资 = 合格件 × 单价 + 工时补差（计时工时 - 标准工时）× 时薪。
  - FR-9-2-3 成本分摊：制造费用按工时比例分摊到工单。
- **FR-9-3 回款控制**
  - FR-9-3-1 自动提醒：到期前 3 天业务员；逾期 7 天升级总经理。
  - FR-9-3-2 付款审批：单笔 > 10 万总经理 + 财务总监双签；**V1.3.6 付款触发条件升级**：对账已确认 **+** 厂商签字扫描件已上传 **+** 双方对账金额一致 才可生成付款申请（详见 FR-6-1-3）。
- **FR-9-4 利润分析**
  - FR-9-4-1 订单/客户/产品维度利润；同比/环比。
  - FR-9-4-2 **【V1.3.3】利润率分级预警**：毛利率 < 10% 标黄、< 5% 标红、< 0 标深红 + 老板驾驶舱强提示；阈值在系统参数可配。
- **FR-9-5（V1.3.4 新增）料号成本聚合视图**
  - FR-9-5-1 按料号聚合所有相关订单的：材料成本（BOM × 实时单价）/ 工时成本（工时记录 × 工时费率）/ 表处成本（按面积/重量/件数）/ 外协成本（按委外单）/ 管理费分摊（按工时占比 OR 材料占比，可配置）。
  - FR-9-5-2 缓存策略：Nacos 配置 `app.cost-cache-ttl`（默认 3600 秒 = 1 小时）。
  - FR-9-5-3 实时更新触发：委外单完成 / 工时录入 / 物料价格变更 → Redis Stream 推送失效消息 → 下次查询重算。
  - FR-9-5-4 权限隔离：管理层看全量；一线员工仅看到自己工序的工时（同 G7）。

#### Epic 10 人事管理

### 2.2 非功能性需求（NFR1...NFR15+）

- **NFR-1 性能**：普通查询 P95 ≤ 2s；复杂报表（多维聚合）P95 ≤ 10s；扫码响应 P95 ≤ 1s。
- **NFR-2 并发**：≥ 100 用户同时在线；≥ 50 个 APP 端同时扫码。
- **NFR-3 可用性**：99.5%（年宕机 < 44h）。
- **NFR-4 数据存储**：≥ 5 年历史数据；图纸文件本地存储 ≥ 2TB 可扩。
- **NFR-5 安全**：密码 bcrypt 加密；敏感字段（身份证/银行卡/手机）AES 加密；HTTPS；SQL 注入/XSS 防护。
- **NFR-6 审计**：所有增删改操作留痕（操作人/时间/IP/前后值）；保留 ≥ 1 年。
- **NFR-7 兼容**：Web 端 Chrome / Edge / Firefox 最新版；Android 8.0+。
- **NFR-8 国际化**：V1.0 中文；预留 i18n 框架。
- **NFR-9 可观测**：日志（Logback）+ 指标（Micrometer）+ 链路追踪（Sleuth + Zipkin）。
- **NFR-10 备份**：每日全量备份 + 增量；RPO ≤ 1h，RTO ≤ 4h。
- **NFR-11 部署**：单台服务器 8C32G 起步；Docker Compose 编排；CI/CD 走 GitHub Actions。
- **NFR-12 代码质量**：SonarQube 扫描；Java 单元测试覆盖率 ≥ 70%；关键路径（金额/状态机）≥ 90%。
- **NFR-13 API 规范**：RESTful + OpenAPI 3.0 文档；版本前缀 `/api/v1/`。
- **NFR-14 离线能力**：APP 端离线扫码缓存 ≥ 500 条；网络恢复后 30s 内自动同步。
- **NFR-15 第三方**：文件预览 pdf.js；图表 ECharts；扫码 zxing-android-embedded。
