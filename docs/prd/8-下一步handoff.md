### 8.2 Architect Handoff

- **服务拆分**：≤ 3 服务（business/production/platform）+ `core` 公共 Module（位于 backend 仓内，与微服务同级）
- **注册中心 / 配置中心**：**Nacos 2.3+**（服务注册发现 + 动态配置 + 命名空间隔离）
- **API 规范**：OpenAPI 3.0 + 版本前缀 /api/v1/；供应商端走 `/api/v1/vendor-portal/*` 路径前缀
- **数据一致性**：业务单据 → 财务数据用**本地消息表 + XXL-JOB 补偿扫描**（替代 RabbitMQ 事务消息）
- **离线同步**：APP 端 → 服务端：HTTP 批量上传 + 冲突解决
- ~~**【V1.3.3】供应商租户隔离**~~ ——**V1.3.5 已下线**：无供应商账号，无 vendor_id 注入
- **【V1.3.3】部署演进**：阶段 1 局域网 → 阶段 2 公网（**V1.3.4 客户反馈：阶段 2 暂不考虑，详见 §8.4 Q2**）
- **【V1.3.4 / V1.3.5 调整】委外 7 状态机**：Spring State Machine 持久化到 DB，状态转换走事件驱动（Redis Stream），转换矩阵在 `outsource_state_transition` 表，预警阈值在 Nacos；**V1.3.5 触发器更新**：SHIPPING → PENDING_INSPECTION 由仓管扫 WW- 委外单码触发（E12-S2），PENDING_INSPECTION → INSPECTING 由品质领料触发（E12-S3）
- **【V1.3.4】料号成本聚合引擎**：Service 层注解 `@CostScope` 控制权限隔离，Redis Key `cost:material:{materialNo}:{from}:{to}`，TTL 默认 3600s，失效消息走 Redis Stream `stream:cost-invalidate`
- **【V1.3.4】工序委外切片接续**：`work_order_process` 表新增 `is_outsourced` / `outsource_id` / `lock_next_machine_id`，状态变更通过 Redis Stream 推接续事件
- ~~**【V1.3.4】固定区域码**：`warehouse_area` 表持久化区域码定义，APP 扫到 AREA- 前缀走专用路由 `POST /api/v1/app/area/scan`~~ ——**V1.3.5 已删除**：`warehouse_area` 表不需要了；APP 改扫 WW- 委外单码走 `POST /api/v1/app/outsource/arrival-scan`
- **【V1.3.6】厂商通知 + 线下对账**：
  - **邮件服务**：`notification-service` 集成 Spring Mail + JavaMailSender；SMTP 配置走 Nacos `app.outsub.email.*`；**失败重试** XXL-JOB 调度（1h/6h/24h 三次）
  - **短信服务**：`notification-service` 集成阿里云/腾讯云 SMS SDK；**签名 / 模板 ID** 走 Nacos `app.outsub.sms.*`；**限流**（每厂商每天 ≤10 条，防骚扰）
  - **对账单 PDF 生成**：`pdf-template` 服务（Velocity/Freemarker 模板），含签字栏位、运杂费、WW- 单明细
  - **签字扫描件存储**：`storage-service` 走 MinIO + **AES-256-GCM**（密钥由 KMS 托管，季度轮换）；路径 `reconcile-signature/{reconcileId}/{filename}`；Bucket 启用版本控制防误删
  - **数据模型新增**：
    - `vendor` 表 `contact_email` 字段从"可选"升级为"必填"（迁移脚本：旧数据标黄提示，强制补填）
    - `vendor` 表新增 `default_reconcile_email`（V1.3.6）+ `notify_channel` 数组字段（V1.3.6）
    - `outsource_reconcile` 表新增 `signature_object_key`（MinIO 路径）+ `signed_at`（对账确认日期）+ `signature_uploaded_at` + `signature_uploaded_by` + `signature_hash`（V1.3.6）
    - `notification_log` 表记录邮件 / 短信发送日志（时间 / 收件人 / 附件 hash / SMTP 响应 / SMS 流水号，保留 1 年）
  - **审计**：`signature_audit_log` 表（用户 / IP / 时间 / 操作类型 / 附件 ID，保留 5 年）
  - **下载权限**：Spring Security `@PreAuthorize("hasAnyRole('总经理','财务总监','采购员')")` + 自定义 SpEL 校验"采购员必须 = 该对账单创建人"
- **部署**：Docker Compose 单机起步，CI/CD 走 GitHub Actions
- **交付物**：架构图 + 服务划分 + 关键 API 设计 + 数据库 ER 图 + 供应商 portal 安全设计文档 + **【V1.3.4】委外 7 状态机时序图 + 料号成本聚合性能基准报告** + **【V1.3.6】对账 PDF 模板 + 邮件模板 + 短信模板 + 签字存储加密方案 + 下载审计时序图**
