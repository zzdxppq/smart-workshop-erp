## 6. IMPL 注意事项（4 条）

1. **10.1 → 10.2 启动顺序**：dev 10.2 IMPL 需在 10.1 dev 完成（generated/ 目录存在）后启动，避免 token 类型回退到 any。SM 萧何协调 PM 排期
2. **DB 隔离钩子**：每个 spec `beforeAll` 调 `POST /test/reset-db` · 若该 API 在 V1.3.7 已退役（V1.3.7.1 删）需用 seed SQL 替代
3. **二次密码弹窗**：page object `confirmHighAmountDialog` 需用 `locator('[data-testid="high-amount-confirm"]').waitFor({ state: 'visible', timeout: 5000 })` 显式等待，避免 flake
4. **多维度看板性能**：TC-10.2.D2 断言 `responseTime < 500ms`（端到端含 UI 渲染），若超 500ms 需先排查 K6（参见 docs/qa/evidence/sprint7-final-integration-test-report.md）
