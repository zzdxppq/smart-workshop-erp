## 2. 技术评审

### 2.1 数据库

- **表**：`sys_workflow_event`（V53 Sprint 8.3 已建）
- **索引**：`idx_wfe_code_role_time (workflow_code, approver_role, created_at)` · 覆盖 WHERE + GROUP BY ✅
- **风险**：🟢 低（无 schema 变更 · Sprint 8.3 已 ship）

### 2.2 性能

- **数据量**：30 天 V1.3.7 估算 1000-2000 行 · 远低于 100K 阈值
- **聚合 SQL**：复合索引覆盖，预计 30-50ms
- **前端看板调用**：D2 看板若调本端点会增加 1 次 RPC · 建议 5min 缓存（Redis）
- **P95 < 300ms SLA**：🟢 轻松达成

### 2.3 缓存策略

- 本端点不写 Redis（统计查询 · 数据实时性要求高）
- 若前端看板压力大可加 `@Cacheable("workflow-stats", ttl = 5m)` 兜底
- 🟡 dev 启动前与 PM 确认是否需要缓存（默认不加）

### 2.4 并发安全

- 读已 commit 数据 · InnoDB MVCC 默认 REPEATABLE READ 隔离
- 9.1 触发 recordEvent AFTER_COMMIT · 不影响本查询
- 🟢 零并发风险
