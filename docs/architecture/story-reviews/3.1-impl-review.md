### 2.1 ⚠️ 必查：老 PO 状态值迁移

V1.3.7 既有 PO 状态可能是 `PENDING` / `ALERT` / `ARRIVED`。DDL 改为新枚举后，老数据可能报错。

**dev 必做**：V49 迁移脚本必须包含数据迁移：

```sql
-- 老状态 → 新状态映射
UPDATE crm_purchase_order SET status = 'PENDING_SHIP'  WHERE status = 'PENDING';
UPDATE crm_purchase_order SET status = 'PARTIAL_ARRIVED' WHERE status = 'ALERT';
UPDATE crm_purchase_order SET status = 'ALL_ARRIVED'   WHERE status = 'ARRIVED';
```

**🟡 必须追加到 V49 脚本**（IMPL 阶段硬性约束）。

### 3.2 ⚠️ 3 条硬性约束

1. **对比 cron 必须每 1h 执行**，不一致率 > 0.1% 自动触发 stream:notify 告警（PM 与 architect 共识）
2. **影子表 `crm_batch_shadow` 保留到阶段 4 结束后 30 天**，便于事后追溯
3. **阶段 2 切流比例不允许 > 50%**，直至 1 周对比数据稳定
