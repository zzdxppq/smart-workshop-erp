## 5. 风险评审

| 风险 | 等级 | 缓解 |
|------|------|------|
| JAR 增大 10MB | 🟡 中 | 仅 erp-platform 模块 · runtime 内存 +10MB 可接受 |
| 启动时间 +200ms | 🟢 低 | `@PostConstruct` 异步预热 · 不阻塞 Spring 上下文就绪 |
| 字体授权 | 🟢 低 | 思源黑体 Apache 2.0 · 商用 OK · LICENSE 文件记录 |
| OS 字体差异 | 🟢 低 | classpath 资源统一 · 跨 OS 100% 一致 |
| PDFBox 兼容 | 🟡 中 | dev 启动后实测 · iText 7.2 + PDFBox 3.0.2 |

> 缓解策略展开：
> - **JAR 10MB**：使用 `maven-shade-plugin` minijar 优化 · 或拆分为 `erp-platform-fonts` 子 jar · **建议 13.2 接受，14.x 评估拆分**
> - **启动 +200ms**：`@PostConstruct(initMethod="warmup")` + `@Async` 或 `applicationReadyEvent` 异步触发
> - **PDFBox 兼容**：`BaseFont.createFont(stream, IDENTITY_H, EMBEDDED)` 三参数顺序在 iText 7.2 稳定 · dev IMPL 后跑 Sprint 12.4 32 测例回归验证

## 6. IMPL 注意事项（3 条）

1. **字体文件放 `backend/src/erp-platform/src/main/resources/fonts/`**（erp-platform 模块而非 erp-app 业务模块 · 字体属于平台基础能力）· 路径 `fonts/SourceHanSansCN-Normal.ttf`
2. **`BaseFont.createFont(stream, IDENTITY_H, EMBEDDED)`**：必须 EMBEDDED 嵌入字体到 PDF（防 PDF 阅读器替换为本地字体）
3. **`@PostConstruct` 异步预热**：使用 `ApplicationReadyEvent` + `@Async` 触发字体加载 · 避免 Spring 启动阻塞 · 加载失败时降级到 JDK SansSerif（保底）
