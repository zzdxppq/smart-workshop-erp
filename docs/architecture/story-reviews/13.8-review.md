## 2. 技术评审

### 2.1 ESC/POS 协议特点

**票据打印机主流协议（爱普生/佳博/商米）**
- **ESC/POS**（Epson Standard Code for Point of Sale）：票据/小票打印机事实标准
- 与 ZPL/TSPL（标签打印机）差异：
  - **无标签概念**：按字符/行布局（80mm 热敏纸 · 滚动走纸）
  - **命令集**：ESC 控制序列（`ESC @` 初始化 / `ESC !` 字号 / `ESC a` 对齐）+ GS 控制序列（`GS V` 切纸 / `GS ( k` 二维码）
  - **字节流直写**：不像 ZPL/TSPL 有 `^XA^XZ` 帧结构
  - **设备碎片化**：飞鹅 FP-58 Ⅱ / 佳博 GP-58MB / 爱普生 TM-T82 / 商米 V2 Pro 等型号指令子集差异

**客户已购型号**：待 PM 范蠡协同客户 IT 确认（IMPL 注意事项 #1）

### 2.3 EscPosAdapter 抽象草案

**接口定义（与 12.4 `ProtocolAdapter` 平级）**：
```kotlin
// backend/src/main/kotlin/com/btsheng/erp/print/protocol/PrintProtocolAdapter.kt (V1.3.10 backlog 草案)
interface PrintProtocolAdapter {
    fun build(payload: PrintPayload): ByteArray
}

class EscPosAdapter : PrintProtocolAdapter {
    // ESC/POS 字节流生成
    override fun build(payload: PrintPayload): ByteArray {
        val bytes = ByteArrayOutputStream()
        bytes.write(0x1B); bytes.write('@'.code)  // ESC @ 初始化
        // ... 字号 + 对齐 + 文本 + 二维码 + 切纸
        return bytes.toByteArray()
    }
}
```

**实现类清单**（4 选 1 按 `sys_printer.protocol` 注入）：
- `EscPosAdapter`（V1.3.10 backlog 新增）· 平级
- `ZplAdapter` / `TsplAdapter`（12.4 已 ship）· 复用
- `PdfBrowserAdapter`（12.4 已 ship）· 复用

**关键设计要点**：
- **字节流断言**：飞鹅 FP-58 Ⅱ / 佳博 GP-58MB 字节流单测覆盖主流型号
- **错误码复用**：50201-50203 与 ZPL/TSPL 一致（PRINTER_OFFLINE / PROTOCOL_UNSUPPORTED / 协议发送失败）
- **模式一架构复用**：`PrintService.sendEscPos()` 与 `sendZpl()` 流程对齐 · @Async 独立线程池复用 `print-zpl-` 前缀
