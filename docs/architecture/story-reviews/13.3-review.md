## 1. 契约评审

### 1.1 端点契约（3 REST）

#### 端点 1 · `GET /api/v1/drawings/{id}/links?biz_type=ORDER`

**用途**：按图纸 ID 反查关联的 5 类业务单据 ID 列表 · 单类查询（按 biz_type 过滤）

| 项 | 规格 |
|----|------|
| Path | `/api/v1/drawings/{id}/links` |
| Query | `biz_type` ∈ `ORDER \| PO \| INCOMING \| INSPECTION \| WORKORDER_PROCESS`（必填）|
| 鉴权 | `@PreAuthorize("hasAnyRole('ADMIN','ENGINEER','SALES','PURCHASER','WAREHOUSE','QC','OPERATOR')")` · SALES 仅能查 ORDER · PURCHASER 仅能查 PO · 类推 |
| 限流 | 100 req/min/user · 1000 req/min/global |
| 缓存 | **无**（图纸级查询 · 高频但非热点）|

**Response 200** · `DrawingLinkListResponse`

```typescript
{
  drawingId: number;          // long
  bizType: 'ORDER' | 'PO' | 'INCOMING' | 'INSPECTION' | 'WORKORDER_PROCESS';
  bizIds: number[];           // long[] · 该图纸关联的 biz_id 列表
  totalCount: number;         // 与 bizIds.length 同值 · 前端分页冗余字段
  querySource: 'DB_REAL' | 'CACHE';  // 13.3 真实查询上线后恒为 DB_REAL（无缓存）

```

**错误码**：
- `40401 DRAWING_NOT_FOUND` · 图纸不存在
- `40304 DRAWING_FORBIDDEN` · 角色与 biz_type 不匹配（如 SALES 查 PO）
- `40001 INVALID_BIZ_TYPE` · biz_type 不在枚举内

## 2. 技术评审

### 2.1 5 类 link JOIN SQL（material_code 关联 · 复用 V54 crm_drawing_link）

> **核心原则**：13.3 不查 5 业务表的 owner_user_id（草案 V1 思路）· 而是查 **crm_drawing_link 五元组 + 业务单据 item 表的 material_code**。V54 crm_drawing_link 已 ship，13.3 直接消费其数据。

#### JOIN 1 · ORDER（业务员 → 订单 → 图纸）

```sql
-- 端点 1：图纸 → 订单 IDs
SELECT oi.order_id AS biz_id
FROM crm_drawing_link dl
JOIN crm_order_item oi
  ON oi.material_code = (
    SELECT material_code FROM crm_drawing WHERE id = :drawingId
  )
WHERE dl.drawing_id = :drawingId
  AND dl.biz_type = 'ORDER'
  AND oi.order_id IN (
    SELECT id FROM crm_order
    WHERE owner_user_id = :userId
      AND status NOT IN ('DRAFT','CANCELLED')
  );

-- 端点 2：业务单据 → 图纸列表
SELECT DISTINCT d.id, d.code, d.name, d.version, d.thumbnail_url
FROM crm_drawing d
JOIN crm_drawing_link dl ON dl.drawing_id = d.id AND dl.biz_type = 'ORDER'
JOIN crm_order_item oi ON oi.material_code = d.material_code
WHERE dl.biz_id = :bizId;
```

#### JOIN 2 · PO（采购员 → PO → 图纸）

```sql
-- 端点 1
SELECT poi.po_id AS biz_id
FROM crm_drawing_link dl
JOIN crm_purchase_order_item poi
  ON poi.material_code = (SELECT material_code FROM crm_drawing WHERE id = :drawingId)
WHERE dl.drawing_id = :drawingId
  AND dl.biz_type = 'PO'
  AND poi.po_id IN (
    SELECT id FROM purchase_order
    WHERE created_by = :userId
      AND status NOT IN ('DRAFT','CANCELLED')
  );

-- 端点 2
SELECT DISTINCT d.id, d.code, d.name, d.version, d.thumbnail_url
FROM crm_drawing d
JOIN crm_drawing_link dl ON dl.drawing_id = d.id AND dl.biz_type = 'PO'
JOIN crm_purchase_order_item poi ON poi.material_code = d.material_code
WHERE dl.biz_id = :bizId;
```

#### JOIN 3 · INCOMING（仓管 → 入库单 → 图纸）

```sql
-- 端点 1
SELECT ii.inbound_id AS biz_id
FROM crm_drawing_link dl
JOIN crm_incoming_order_item ii
  ON ii.material_code = (SELECT material_code FROM crm_drawing WHERE id = :drawingId)
WHERE dl.drawing_id = :drawingId
  AND dl.biz_type = 'INCOMING'
  AND ii.inbound_id IN (
    SELECT id FROM wms_inbound
    WHERE operator_user_id = :userId
      AND status NOT IN ('DRAFT','CANCELLED')
  );

-- 端点 2
SELECT DISTINCT d.id, d.code, d.name, d.version, d.thumbnail_url
FROM crm_drawing d
JOIN crm_drawing_link dl ON dl.drawing_id = d.id AND dl.biz_type = 'INCOMING'
JOIN crm_incoming_order_item ii ON ii.material_code = d.material_code
WHERE dl.biz_id = :bizId;
```

#### JOIN 4 · INSPECTION（品检员 → 质检单 → 图纸）

```sql
-- 端点 1
SELECT qi.inspection_id AS biz_id
FROM crm_drawing_link dl
JOIN crm_inspection_item qi
  ON qi.material_code = (SELECT material_code FROM crm_drawing WHERE id = :drawingId)
WHERE dl.drawing_id = :drawingId
  AND dl.biz_type = 'INSPECTION'
  AND qi.inspection_id IN (
    SELECT id FROM qc_inspection
    WHERE inspector_user_id = :userId
  );

-- 端点 2
SELECT DISTINCT d.id, d.code, d.name, d.version, d.thumbnail_url
FROM crm_drawing d
JOIN crm_drawing_link dl ON dl.drawing_id = d.id AND dl.biz_type = 'INSPECTION'
JOIN crm_inspection_item qi ON qi.material_code = d.material_code
WHERE dl.biz_id = :bizId;
```

#### JOIN 5 · WORKORDER_PROCESS（操作工 → 工序 → 图纸 · 端点 3）

```sql
-- 端点 3 完整 SQL · 含缓存
SELECT DISTINCT d.id, d.code, d.name, d.version, d.thumbnail_url,
       wp.id AS process_id, wp.process_code, wp.process_name,
       wp.work_order_id, wo.code AS work_order_code,
       wp.status, wp.operator_user_id
FROM crm_workorder_process wp
JOIN crm_workorder wo ON wo.id = wp.work_order_id
JOIN crm_workorder_item woi ON woi.work_order_id = wo.id
JOIN crm_drawing d ON d.material_code = woi.material_code
WHERE wp.id = :processId
  AND wp.status = 'IN_PROGRESS';
-- 注：端点 3 鉴权层强制 operator_user_id = :userId · 否则 40304 PROCESS_FORBIDDEN
```

### 2.2 V58 Flyway 增量索引（5 条部分索引 · 关键性能优化）

> **核心原则**：V54 既有索引 `idx_drawing_link (drawing_id, biz_type, biz_id)` 是通用索引 · V58 改为 5 条 **biz_type 部分索引**（PG 风格 · MySQL 5.7+ 用函数索引或虚拟列）· 5 个小索引代替 1 个大索引 · **查询 O(N) → O(log N)**

```sql
-- V58__drawing_link_real_query.sql

-- 1. ORDER 部分索引
CREATE INDEX idx_drawing_link_order
  ON crm_drawing_link(drawing_id, biz_id)
  WHERE biz_type = 'ORDER';
-- PG: 部分索引 / MySQL 5.7+: 函数索引
--   CREATE INDEX idx_drawing_link_order ON crm_drawing_link(
--     (CASE WHEN biz_type='ORDER' THEN drawing_id END),
--     (CASE WHEN biz_type='ORDER' THEN biz_id END)
--   );

-- 2. PO 部分索引
CREATE INDEX idx_drawing_link_po
  ON crm_drawing_link(drawing_id, biz_id)
  WHERE biz_type = 'PO';

-- 3. INCOMING 部分索引
CREATE INDEX idx_drawing_link_incoming
  ON crm_drawing_link(drawing_id, biz_id)
  WHERE biz_type = 'INCOMING';

-- 4. INSPECTION 部分索引
CREATE INDEX idx_drawing_link_inspection
  ON crm_drawing_link(drawing_id, biz_id)
  WHERE biz_type = 'INSPECTION';

-- 5. WORKORDER_PROCESS 部分索引
CREATE INDEX idx_drawing_link_process
  ON crm_drawing_link(drawing_id, biz_id)
  WHERE biz_type = 'WORKORDER_PROCESS';

-- 6. 5 业务表 item 表 material_code 索引（JOIN 性能）
CREATE INDEX idx_order_item_material_code ON crm_order_item(material_code);
CREATE INDEX idx_po_item_material_code ON crm_purchase_order_item(material_code);
CREATE INDEX idx_incoming_item_material_code ON crm_incoming_order_item(material_code);
CREATE INDEX idx_inspection_item_material_code ON crm_inspection_item(material_code);
CREATE INDEX idx_workorder_item_material_code ON crm_workorder_item(material_code);

-- 7. 注释
COMMENT ON TABLE crm_drawing_link IS
  'V1.3.9 Sprint 12.1 创建 · Sprint 13.3 真实查询对接 · V58 加 5 部分索引';
```

> **部分索引收益**：5 个小索引（每类 biz_type 数据量仅占 crm_drawing_link 的 20%）vs 1 个大索引 · 索引体积小 80% · 写入开销小 80% · 查询命中率 100%（按 biz_type 过滤后必中）

### 2.3 Redis 缓存设计（5min TTL）

#### Key 设计（含 role + user_id · 防角色权限漂移）

| Key 模式 | TTL | 写入端点 | 失效触发 |
|---------|-----|----------|----------|
| `drawing:link:{drawing_id}:{role}:{user_id}` | 300s | 端点 1 + 端点 2 | crm_drawing_link 写入 · `@CacheEvict` |
| `user:current_process:{user_id}` | 300s | 端点 3 | crm_workorder_process 状态变更 · `@CacheEvict` |
| `user:access_check:{role}:{user_id}:{drawing_id}` | 300s | 12.1 既有 `DrawingAuthz.hasAccess` | 同上 |

> **Key 含 role + user_id 的原因**（IMPL 注意事项 #2）：避免 SALES 用户缓存命中 PURCHASER 角色的权限漂移 · 用户切换角色时旧缓存自动失效（user_id 相同但 role 不同 = 不同 Key）

#### 缓存一致性保障

- **写入触发**：`crm_drawing_link` 表 INSERT/UPDATE/DELETE 时 → `@CacheEvict(value="drawingLink:PERM", allEntries=true)`
- **TTL 兜底**：5 分钟 TTL 强制过期 · 即使 `@CacheEvict` 漏掉也不会长期脏读
- **降级策略**：Redis 7 失效 → fail_count 容差（3 次）· 第 4 次失败退化到 DB 真实查询 · DB QPS < 100 安全范围

## 6. IMPL 注意事项（3 条）

1. **V58 部分索引 5 条**：PostgreSQL 原生支持 `WHERE biz_type='X'` 部分索引 · **MySQL 5.7+ 需用函数索引 / 虚拟列实现**（如 `CREATE INDEX ... ON crm_drawing_link((CASE WHEN biz_type='ORDER' THEN drawing_id END), ...)`）· dev agent 启动前必须确认 backend 当前 DB 类型（PG 还是 MySQL）· 在 `application.yml` 切分 dialect
2. **Redis 缓存 Key 含 `role + user_id`**（如 `drawing:link:{drawing_id}:{role}:{user_id}`）· 避免 SALES 用户缓存命中 PURCHASER 权限 · 用户切换角色时旧 Key 自动失效（user_id 相同但 role 不同 = 不同 Key · 不命中旧 Key）· **禁止 Key 仅含 `user_id`**
3. **灰度阶段时序**：13.3 ship 后才能开 12.1 灰度阶段 2（SALES）· SM 萧何必须协调 12.1 灰度阶段 2-4 开启日期 ≥ 13.3 ship 日期 + 1 天缓冲 · 否则 SALES 用户看到空集（真实查询未对接）· **本评审 ⭐ 关键约束**
