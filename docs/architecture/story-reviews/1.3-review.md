## 风险点（5 条）

| # | 风险 | 等级 | 缓解 |
|---|------|------|------|
| 1 | **双轨阈值（DB `sys_role.amount_threshold` + Nacos `app.workflow.thresholds.*`）不一致** — 启动时 Nacos 配置缺失或 DB 数据漂移 → 路由决策可能走旧值 | 🟡 中 | 启动时 `@PostConstruct` 校验：Nacos 配置优先，缺失时 DB 回退 + 告警飞书；`ThresholdService.getThreshold()` 显式优先级注释 + 单元测试 5 边界 |
| 2 | **数据字典 6 类 30+ 种子跨模块一致性** — 销售/采购/财务模块可能引用不同的种子（如销售希望 "CNC" 工序代号为 "P-CNC"，而生产希望 "P-CNC-001"） | 🟡 中 | `DictTypeInitializer` `CommandLineRunner` 启动幂等初始化 + 跨模块契约测试（销售/采购/财务 各 3 测例 = 9 测例） |
| 3 | **`@Scheduled` 审计清理时区差异** — 系统 TZ=Asia/Shanghai vs 服务器 UTC → 清理时间漂移 8h | 🟢 低 | `@Scheduled(cron="0 0 3 * * ?", zone="Asia/Shanghai")` 显式声明时区 + 单元测试 mock `Clock` 验证 |
| 4 | **HR Feign 真实化后 `HrEmployeeClientFallback` 删除** 触发 Story 1.2 35 测例失败 | 🟡 中 | Fallback 标记 `@Deprecated` 不删，3 个 sprint 后清理；`HrEmployeeClient` 改为纯 Feign 客户端（去 `fallbackFactory`），Fallback 仅 `@Component @Deprecated` 占位 |
| 5 | **k6 性能基线本机无 docker daemon** → 推到 Story 1.3 部署阶段实测，本机只交付脚本骨架 | 🟡 中 | k6 8 脚本 + 报告模板就位，部署阶段实测；本机不阻 QA *test-design |

## 关键反馈（dev 实施时须注意的）

1. **T0.1 实体字段对齐**：`sys_dict` 字段 `id/type/code/name/sortOrder/isActive/createdBy/createdAt/updatedAt`；`sys_param` 字段 `id/key/value/group/description/updatedBy/updatedAt`；`sys_change_log` 字段 `id/entity/entityId/operation/beforeValue/afterValue/changedBy/changedAt` — 与 architect §13 DoD F.8 一致；dev 实施时显式 `CREATE TABLE` + 索引（`idx_dict_type_code` / `idx_changelog_entity_entityId`）
2. **T1.7 `ThresholdService` 双轨实现**：查询时先查 Nacos `app.workflow.thresholds.{bizType}.{roleCode}`，null 才回退 DB `sys_global_threshold` + `sys_role.amount_threshold`；写入双写（Nacos + DB + sys_change_log）；启动时 `@PostConstruct` 校验一致性 + 告警
3. **T2.1 Feign 超时配置**：`connectTimeout=2s, readTimeout=2s`（HR 调用不能阻塞审批）+ `Retryer.Default(100, 1000, 3)` 指数退避；`HrEmployeeClientFallback` 标记 `@Deprecated` 不删（保 Story 1.2 兼容）
4. **T3.4 审计清理阈值**：保留期 ≥ 1 年（V1.3.7 §8 红线 5）；归档到 `sys_audit_log_archive`（不删，5 年保留）；主表 `sys_audit_log` 仅保留 1 年内；显式 `zone="Asia/Shanghai"` 防时区漂移
5. **T3.6/T3.7/T3.8 报告归档**：`docs/qa/evidence/1.3-k6-baseline.html` / `1.3-zap-baseline.html` / `1.3-sonar.json` 三件套 + architect §11 风险缓解 ⑦
6. **Story 1.1/1.2 回归不破**：dev 实施 Story 1.3 时先 `git pull` Story 1.1/1.2 HEAD → 跑 `1.1` 35 + `1.2` 151 = 186 测例全过 → 再开始 1.3 编码；`sys_role.amount_threshold` 字段保留（双轨），不删除
7. **OpenAPI 同步**：Story 列出 11 端点，OpenAPI `E1-Dict` / `E1-Param` / `E1-Threshold` / `E1-ChangeLog` 4 tag 现有 schema 简单（无完整 DTO），dev 实施时须补全 `DictCreateRequest/DictUpdateRequest/ParamUpdateRequest/ThresholdUpdateRequest` 等 schema
8. **HR Feign 真实化时序**：dev 实施 T2.1 前必须先确认 `erp_business.hr` 模块的 `POST /api/v1/hr/employees/{id}/status` 端点已就绪（否则降级为 mock）；实施完成后删除 `HrEmployeeClientFallback.java` 中的兜底逻辑，仅保留 `@Deprecated` 占位
