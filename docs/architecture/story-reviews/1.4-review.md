## 风险点（5 条）

| # | 风险 | 等级 | 缓解 |
|---|------|------|------|
| 1 | **跨仓协作（android-impl + backend）OpenAPI 契约同步** — backend 1.1/1.2/1.3 端点签名变更 → android 1.4 Retrofit 接口需同步更新 | 🟡 中 | T0.4 Retrofit 自动同步 + MockWebServer 跨仓集成测试 + OpenAPI 100% 同步（Story 1.4 dev 实施时阻塞点） |
| 2 | **离线模式 SQLite 同步冲突** — 多设备同时离线扫码 + 网络恢复后同步 → 数据覆盖风险 | 🟡 中 | T4.3 冲突检测（服务端 ts > 本地 ts + 5s 容差）+ T4.4 弹窗 3 选项（覆盖/合并/取消）+ T4.6 审计 1 行 |
| 3 | **ML Kit Barcode 5 类码误识别** — 光线/角度/二维码质量影响识别率（特别是工单码 GD-20260610-0001 13 位字符串） | 🟡 中 | T3.1 CameraX 自动对焦 + ML Kit 多次识别 + T3.7 手动输入兜底 + 扫码性能 P95 < 500ms |

## 8 条关键反馈（dev 实施时须注意的）

1. **T0.1 依赖版本对齐**：Kotlin 1.9 + AGP 8.2 + Room 2.6 + CameraX 1.3 + ML Kit Barcode 17.2 + Biometric 1.2 + Hilt 2.50（与 android-impl 已有 `build.gradle.kts` 兼容）
2. **T1.3 加密存储**：必须 `EncryptedSharedPreferences`（不是普通 SharedPreferences）+ Android Keystore master key + AES-256-GCM（V1.3.7 红线 1）
3. **T2.5 TTL 动态读**：每次 APP 启动调 `GET /api/v1/params/app.cache-ttl` → 缓存到内存（避免每次都查 DB）；TTL 变更后下次启动生效
4. **T3.1 CameraX 权限**：必须 `Manifest.permission.CAMERA` + 运行时权限申请（Android 6.0+）+ 隐私协议
5. **T3.3 prefix 解析**：5 类码 prefix 严格按 PRD 定义（WN/WL/WR/WD/WW），大小写敏感；解析失败 → T3.7 手动输入
6. **T4.3 冲突检测**：服务端 ts 比本地 ts 新（> 5s 容差）才弹窗；避免误报
7. **跨仓集成测试**：android-impl 用 MockWebServer 模拟 backend（不依赖真实 backend 启动），保证 CI 可独立跑
8. **OpenAPI 同步**：5 端点（`/api/v1/app/{login, sync, messages, messages/{id}/read, scan/route}`）的详细 DTO schema 需 dev 在 1.4 实施时补全到 `backend/spec/openapi.yaml` 的 `E1-App` tag
