## 2. 技术评审

### 2.1 codegen 工具选型

- **openapi-typescript-codegen 0.29** · 与 V1.3.7 OpenAPI 3.0 兼容
- 备选：openapi-generator（Java 端，类型更全但体积大）/ openapi-typescript（只生成类型无 service）
- 决策：openapi-typescript-codegen 在"类型 + service stub + 体积"三点平衡最优 ✅

### 2.2 性能

- 132 端点 codegen 耗时预计 < 5s（V1.3.7 端点数）
- CI 增量：typecheck 增加 1-2s（生成文件 132 service + 80 model，类型检查 3-5s）
- 整体影响：🟢 可接受

### 2.3 与 Story 10.5 协同

- 10.1 落地"基础设施"（类型生成 + 强类型门禁）
- 10.5 落地"消费方迁移"（5 个 `.vue` any → generated）
- 顺序：10.1 必须先于 10.5 落地（dev 启动顺序）

### 2.4 与 Story 10.3 协同

- 10.3 新增 1 个端点 `GET /api/v1/workflow/events/stats`
- 10.1 IMPL 时 dev 需跑一次 `gen:api` 把该端点纳入 generated
- 🟢 已有端点的增量自动覆盖 · 无冲突

## 6. IMPL 注意事项（3 条）

1. **dry-run 优先**：dev 启动 IMPL 前先 `rm -rf src/api/generated && npm run gen:api` 试生成，确认无 openapi.yaml 字段错误（驼峰/下划线/required 漂移）后再提交
2. **`typecheck:ci` 缓存清理**：CI workflow 需在 `npm run typecheck:ci` 前加 `rm -rf node_modules/.cache`，避免增量 typecheck 误报通过
3. **与 10.3 协同**：dev 需在 10.3 dev 完成（新增 1 端点）后再跑 `gen:api`，确保 10.3 端点进入 generated。10.1 与 10.3 都在 Sprint 10，建议 10.1 dev 启动延后 0.5 天等 10.3 端点合并
