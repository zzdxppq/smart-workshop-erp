## 3. 技术评审

### 3.1 DocNoGenerator 扩展

- 模板 `BATCH{yyyyMMdd}{seq:4}` · 复用 V1.3.7 1.5 DocNoGenerator
- Redis INCR 保证并发安全
- 🟢 标准模式

### 3.2 解析正则

```regex
^WL-(.+?)(?:-BATCH-(\d{8})-(\d{4}))?$
```

- 捕获组 1：material_no（必）
- 捕获组 2/3：批次日期 + 流水（可选，兼容老格式）
- 🟢 单元测试覆盖 10+ 边界

### 3.3 旧物料码处理

- 老 `WL-A001`（无 BATCH 部分）`is_active=1`，保留历史
- 新 `WL-A001-BATCH-20260613-0001` `is_active=1`，活跃
- **🟡 必查**：1.11 老 `WL-XXXX` 是否仍需保留活跃？建议：到货后老码自动 `is_active=0`
