### 1.2 端点契约落地

```
GET /api/v1/workflow/events/stats?workflow_code=PO_APPROVAL&approver_role=PROCUREMENT_MANAGER&start_date=2026-05-14&end_date=2026-06-13
```

| 项 | 实现 | 验证 |
|----|------|------|
| workflow_code 必填 | service 第 1 道校验 `workflowCode == null \|\| isEmpty()` → `40001` | TC-10.3.1.3 + TC-10.3.2.d/e |
| start_date ≤ end_date | service `start.isAfter(end)` → `40001`（**本次新增**） | **TC-10.3.3.3** |
| 默认 30 天 | `start = now.minusDays(30)`, `end = now()` | TC-10.3.2.b |
| 复合角色值不拆分 | `approverRole` 原值 GROUP BY · 4 阈值路由 `GM+PROCUREMENT_MANAGER` 保留 | TC-10.3.2.2 |
| 权限 | `@PreAuthorize("hasAnyRole('GM', 'ADMIN')")`（Controller 元数据） | TC-10.3.4.1（注解反射）+ 后续 SecurityConfig IT |

### 1.3 4 步聚合 SQL（MyBatis `@Select` + `<script>`）

```sql
-- totalCount
SELECT COUNT(*) AS total
FROM sys_workflow_event
WHERE workflow_code = ? AND created_at >= ? AND created_at < ?
  [AND approver_role = ?]

-- byEventType GROUP BY
SELECT event_type, COUNT(*) AS cnt FROM ... GROUP BY event_type

-- byApproverRole GROUP BY
SELECT approver_role, COUNT(*) AS cnt FROM ... GROUP BY approver_role
```

复合索引 `idx_workflow_approver(workflow_code, approver_role, created_at)` 覆盖 WHERE + GROUP BY，预期 30 天 1k 行 < 50ms（与 4.3 GM 报表 P95 < 300ms 一致）。
