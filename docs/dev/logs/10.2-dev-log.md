### 4.2 DB 隔离（architect IMPL 注意事项 2）

```typescript
const TEST_RESET_ENDPOINT = '/api/v1/test/reset-db';

export async function resetDb(request: APIRequestContext): Promise<boolean> {
  const res = await request.post(TEST_RESET_ENDPOINT);
  if (!res.ok()) {
    console.warn(`[10.2 helper] resetDb failed: ${res.status()} · fallback to seed SQL`);
    return false;  // V1.3.7.1 退役兜底 · 调用方可决定 seed SQL
  }
  return true;
}
```

**每个 spec `beforeAll` 调 resetDb** · 14 测例全部覆盖。

### 4.3 二次密码弹窗（architect IMPL 注意事项 3）

```typescript
export async function confirmHighAmountDialog(page: Page, adminPassword: string): Promise<void> {
  const dialog = page.locator('[data-testid="high-amount-confirm"]');
  await dialog.waitFor({ state: 'visible', timeout: 5_000 });  // 显式等待 5s
  await page.fill('[name=adminPassword]', adminPassword);
  await page.click('button:has-text("确认")');
}
```

### 4.4 委外 7 状态机推进（architect IMPL 注意事项）

```typescript
export async function advanceOutsource(
  request: APIRequestContext,
  outsourceNo: string,
  targetStatus: 'PENDING' | 'SENT' | 'IN_PROGRESS' | 'COMPLETED' | 'INSPECTED' | 'NOTIFIED_REPAIR' | 'CLOSED',
): Promise<void> {
  const res = await request.post('/api/v1/outsource-states/advance', {
    data: { outsourceNo, targetStatus },
  });
  expect(res.ok(), `advanceOutsource ${outsourceNo} → ${targetStatus} failed: ${res.status()}`).toBeTruthy();
}
```

### 4.5 角色灵活登录（Role union 限制突破）

`e2e/fixtures/auth.ts` `Role` 类型仅覆盖 6 套预置账号。10.2 测例涉及：
- `production_manager01`（B3 转生产）
- `purchaser01`（C1 委外下单）
- `warehouse01`（C2 委外推进 + D1 入库扫码）
- `qa01`（C3 发起返修）

→ 这些角色在 V1.3.7 seed 中存在（参考 `quality-drawing-fa.spec.ts` qa01 + `sourcing-reconcile.spec.ts` purchaser01）但不在 auth.ts USERS 6 套内。

**IMPL 决策**：新增 `loginWithCredentials(page, request, username, password, displayName?, roles?)` helper，绕过 Role union 限制，**保留** loginAs(page, request, role) 用于 6 套预置角色。
