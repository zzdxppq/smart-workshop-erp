## 3. IMPL 关键决策

### 3.1 OPERATOR 权限位矩阵（view-only）

按 12.1 灰度阶段 4 角色矩阵：

| 角色 | view | print | download | upload | delete |
|------|------|-------|----------|--------|--------|
| **OPERATOR current** | ✅ | ❌ | ❌ | ❌ | ❌ |
| **OPERATOR other** | ❌ | ❌ | ❌ | ❌ | ❌ |

OPERATOR 严格限制：
- 仅 view=true
- print/download/upload/delete 全 false
- 与 web 端 `<DrawingViewer>` 仅渲染"查看"按钮，无"下载/打印"按钮对齐

### 3.2 灰度 feature flag 集成

**DrawingFeatureFlag.kt** 客户端缓存策略：
- JVM 进程内 5 分钟缓存（与 backend `@Cacheable` TTL 对齐）
- 拉取 `/api/v1/dict/list` 全量 sys_dict → 过滤 `type=DRAWING_ACL_FEATURE_FLAG`
- 解析 `value` 字段："true"/"1"/"TRUE" → true，其他 → false
- `invalidate()` 紧急清空（admin 改 sys_dict 后 5min TTL 兜底）

**feature flag 关闭行为**：
- `draw.acl.gray.OPERATOR=false` → OPERATOR 完全看不到"查看图纸"入口
- WorkorderProcessScanFragment 顶部提示"操作工图纸预览灰度未开启"
- OperatorDrawingPreviewFragment 跳回上一页 + Toast 拦截

### 3.3 跨工序拦截（40304）

**DrawPermissionInterceptor.interceptOperatorProcessAccess** 新增方法：
1. 查 `/api/v1/workorders/current-process`（5min Redis 缓存）→ 拿当前 user 的 processId
2. currentProcessId == null → "您未关联当前工序" 拒绝
3. currentProcessId != expectedProcessId → "跨工序访问被拒绝（40304）" + Toast 拦截
4. currentProcessId == expectedProcessId → 调 /permission 拿权限位 → 通过则跳预览

**40304 错误码统一规范**（12.1 arch REVIEW 附录 A 决策 1）：
- 与 backend DrawingAuthz + DrawingPreviewService 全用 40304
- 不再细分 40320/40321/40322 三种
- 详细 message 区分角色与场景

### 3.4 API 层下载/打印拦截

**ApiClient 扩展策略**：
- 仅暴露 `GET /api/v1/drawings/{id}/preview`（流式 PDF）
- **不暴露** `GET /api/v1/drawings/{id}/download` 端点（OPERATOR 系统级拦截）
- Oper. 角色即使绕过 UI 也无法调下载 API（编译期阻止）

## 8. 风险与缓解（阶段 4 专项）

| 风险 | 等级 | 缓解 |
|------|------|------|
| R1 OPERATOR 高速扫码 1-2 秒多次鉴权 | 🟡 中 | Redis 7 缓存 user→process 5min + @Cacheable 兜底 |
| R2 跨工序误访问（扫码相邻工序） | 🟡 中 | interceptOperatorProcessAccess 严格校验 + 40304 Toast |
| R3 灰度开关关闭时仍有人看到入口（缓存延迟） | 🟢 低 | 进程内 5min 缓存 + invalidate() 紧急清空 |
| R4 PDF 渲染耗时过长影响扫码节奏 | 🟢 低 | 流式渲染 + 字节数显示 · 实测 < 1s |
| R5 OPERATOR 试图绕过 UI 直接调 API | 🟢 低 | ApiClient 仅暴露 /preview · 编译期阻止 download |
