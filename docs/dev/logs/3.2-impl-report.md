## 3. 关键设计决策

### 3.1 新子包命名 `materialbarcodebatch`

**理由**：V1.3.7 1.11 老物料条码表 `crm_material_barcode`（barcode_no 格式 `BC{...}`）已有 9 个端点。V1.3.8 新表 `crm_material_barcode_batch`（barcode_no 格式 `WL-...-BATCH-...`）走新子包，**完全无侵入**。

### 3.2 URL 前缀隔离

| V1.3.7 1.11 老路径 | V1.3.8 3.2 新路径 |
|---------------------|-------------------|
| `/api/v1/materials/barcode/generate` | `/api/v1/material-barcode/generate` |
| `/api/v1/materials/barcode/{barcodeNo}` | `/api/v1/material-barcode/parse?barcode=...` |

URL 完全不冲突，可并存。

### 3.3 解析正则

```regex
^WL-(.+?)-BATCH-(\d{8})-(\d{4})$
```

- 捕获组 1：`material_no`（含连字符）
- 捕获组 2：批次日期 yyyyMMdd
- 捕获组 3：4 位流水号

老格式 `WL-A001` **不匹配**新正则 → 安全。

### 3.4 seq 简化实现

本期 IMPL 阶段用 `batchId % 10000` 作为 seq（生产应接 DocNoGenerator.nextMaterialBatchNo()）。12 测例覆盖正则即可，无需真实 DB。
