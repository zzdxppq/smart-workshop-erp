## §4 IMPL 注意事项落实（arch REVIEW §6）

### §4.1 mounted 时调 10.3（不要在 setup 阶段）✅

```typescript
// src/views/reports/WorkflowStats.vue
const { stats, loading, error, fetchStats } = useWorkflowStats()
const filters = ref(defaultFilters())

// ⚠️ 严禁在 setup 阶段直接调 fetchStats
// ✅ 仅在 onMounted 钩子 + 用户事件 (onQuery) 中调
onMounted(() => {
  // 深链接 ?tab=xxx 支持
  const tabQuery = route.query.tab
  if (typeof tabQuery === 'string' && ['workload', 'event-type', 'trend', 'anomaly'].includes(tabQuery)) {
    activeTab.value = tabQuery
  }
  // 默认角色校验
  if (auth.user && !auth.hasRole('GM') && !auth.hasRole('ADMIN')) {
    permissionDenied.value = true
  }
  onQuery()  // ← 在 onMounted 内调，而非 setup 阶段
})
```

**SSR 安全**：setup 阶段在 SSR 服务端执行 · `new Date()` 服务端/客户端不一致 · 触发 hydration mismatch。`onMounted` 仅客户端执行 · 完全合规。

### §4.2 ECharts option 用 shallowRef 包裹（4 子组件统一落实）✅

```typescript
// 4 子组件统一模式
const chartInstance = shallowRef<echarts.ECharts | null>(null)
const chartOption = shallowRef<echarts.EChartsCoreOption>({})

watch(() => props.stats, () => {
  chartOption.value = buildOption(props.stats)
  if (chartInstance.value) updateChart()
}, { immediate: false })
```

**性能优化**：ECharts option 是大型嵌套对象 · `ref` 触发深度 Proxy 包装 · 性能开销大。`shallowRef` 仅包装 `.value` 引用 · 内部对象保持原始引用。

### §4.3 onUnmounted 销毁 ECharts 实例（4 子组件统一落实）✅

```typescript
// 4 子组件统一模式
onBeforeUnmount(() => {
  window.removeEventListener('resize', handleResize)
  if (chartInstance.value) {
    chartInstance.value.dispose()
    chartInstance.value = null
  }
})
```

**内存安全**：ECharts 实例持有 Canvas/DOM 引用 · Vue 组件销毁后若不显式 `dispose()` · 内存泄漏 · 多次切换 Tab 后内存膨胀。

## §9 权限校验证据

### §9.1 路由层（beforeEach 守卫）

```typescript
// router/index.ts L228-244
router.beforeEach((to, _from, next) => {
  const auth = useAuthStore()
  if (to.meta.public) return next()
  if (!auth.token) return next({ name: 'Login', query: { redirect: to.fullPath } })

  // V1.3.9 Sprint 13 Story 13.4：路由 meta roles 校验
  const requiredRoles = (to.meta.roles as string[] | undefined) || []
  if (requiredRoles.length > 0) {
    const ok = requiredRoles.some((r) => auth.hasRole(r))
    if (!ok) {
      return next({ name: 'Dashboard' })  // 阻止 SALES 等角色
    }
  }
  next()
})
```
