### §2.2 新增 1 endpoint（paths 区块 · L3529-L3559 区间）

```yaml
/api/v1/quality/inspections:
  post:
    tags: [E7-Quality]
    summary: 创建品质检验单（V1.3.9 Sprint 13.1 · 补 InspectionDTO schema）
    operationId: createInspectionV1389
    security:
      - bearerAuth: []
    requestBody:
      required: true
      content:
        application/json:
          schema:
            $ref: '#/components/schemas/InspectionCreateRequest'
    responses:
      '201': { schema: $ref '#/components/schemas/InspectionResponse' }
      '400': { description: "字段校验失败（code: 40001 INVALID_FIELD · 字段名 details.field）", schema: $ref '#/components/schemas/Result' }
      '403': { description: "无 QC 权限（40301 QC_REQUIRED）" }
      '409': { description: "物料码已被检验（40901 INSPECTION_DUPLICATE）" }
```

> 注：description 字符串用双引号包裹，避免 YAML parser 把 `:` 解释为 mapping key（IMPL 中间阶段遇到该问题，已修正）。

### §2.3 错误码契约（与 architect REVIEW §1.7 一致）

| HTTP | code | 含义 | 触发条件 |

| 403 | 40301 QC_REQUIRED | 无 QC 权限 | 非 QC 角色调用 |
| 409 | 40901 INSPECTION_DUPLICATE | 物料码已被检验 | 同物料码已有未关闭检验单 |

> 注：openapi.yaml `ErrorCode` enum 已包含 `40001/40301/40901` · 13.1 仅新增契约映射 · 无 enum 改动。

## §4 InspectionCreate.vue Option A → generated 切换

### §4.1 改动范围（diff 摘要）

| 改动 | Option A 前 | 13.1 后 |
|------|------------|---------|
| 本地 interface | `InspectionFormPayload`（6 字段 · `type`/`materialCode`/`batchNo`/`qty`/`inspector`/`aql`/`qualityStatus?`）| ❌ **删** |
| type alias | `type InspectionType = 'IQC' \| 'IPQC' \| 'OQC'`（与 backend 实际 enum 不一致）| `type InspectionType = NonNullable<InspectionCreateRequest['inspectionType']>`（5 类 INCOMING/IN_PROCESS/OUTGOING/FA/CMM）|
| QualityStatus alias | `type QualityStatus = MaterialBarcodeParseResponse['qualityStatus']`（复用其他 schema enum）| `type QualityStatus = NonNullable<InspectionCreateRequest['qualityStatus']>`（4 类 PASS/FAIL/PENDING/REWORK）|
| form ref 类型 | `ref<InspectionFormPayload>` | `ref<InspectionCreateRequest>` |
| inspectItems | 字符串拼接 + 不规则对象 | `ref<InspectionItemDTO[]>([])` + addItem/removeItem 方法 |
| qualityStatus 字段 | optional `?` | 必填（4 值 enum） |
| 提交调用 | `qualityStore.createInspection(form.value)`（Pinia store）| `E7QualityService.createInspectionV1389(form.value)`（generated service）|
| import | `import type { MaterialBarcodeParseResponse }` | `import type { InspectionCreateRequest }` + `import type { InspectionItemDTO }` |
| 业务字段 | 仅 type/materialCode/batchNo/qty/inspector/aql | 新增 inspectionType 完整 5 类枚举选择器 + qualityStatus 4 类枚举选择器 + inspectItems 动态数组 |

### §4.2 模板改动

- 新增 `<el-select>` `inspectionType` 5 类选项（INCOMING/IN_PROCESS/OUTGOING/FA/CMM）
- 新增 `<el-select>` `qualityStatus` 4 类选项（PASS/FAIL/PENDING/REWORK）
- 新增 `inspectItems` 动态数组（itemName + standard + measuredValue + result[OK/NG/NA] + addItem/removeItem）
- 删除 batchNo/qty/inspector/aql 字段（Option A 占位字段，13.1 改用 generated schema 实际字段）
