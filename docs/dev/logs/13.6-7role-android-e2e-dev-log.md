## 3. AVD 模拟器 + 物理设备配置命令

### 3.1 AVD 创建（DevOps 张良执行）

```bash
# 1. 安装 Android SDK API 30
sdkmanager "system-images;android-30;google_apis;x86_64"
sdkmanager "platform-tools" "emulator"

# 2. 创建 4 个 AVD（worker parallel）
for i in 1 2 3 4; do
  echo "no" | avdmanager create avd \
    -n test_avd_$i \
    -k "system-images;android-30;google_apis;x86_64" \
    -d pixel_5 \
    --force
done

# 3. 配置 AVD 资源（8GB RAM · 4 cores）
for i in 1 2 3 4; do
  # 写入 ~/.android/avd/test_avd_$i.avd/config.ini
  cat >> ~/.android/avd/test_avd_$i.avd/config.ini <<EOF
hw.ramSize=8192
hw.cpu.ncore=4
hw.gpu.enabled=yes
hw.gpu.mode=swiftshader_indirect
disk.dataPartition.size=4G
EOF
done
```

### 3.2 AVD 启动（GitLab CI script）

```bash
# 启动无头 AVD（CI 沙箱友好）
nohup $ANDROID_HOME/emulator/emulator \
  -avd test_avd_$CI_NODE_INDEX \
  -no-window -no-audio -no-boot-anim \
  -gpu swiftshader_indirect \
  -no-snapshot-save &
$ANDROID_HOME/platform-tools/adb wait-for-device
$ANDROID_HOME/platform-tools/adb shell input keyevent 82   # 解锁
```

### 3.3 物理设备备份（DevOps 张良）

```bash
# USB 调试 · Android 11+ 物理设备
adb devices   # 列出可用设备
adb -s <serial> shell getprop ro.build.version.release  # 验证 API ≥ 30

# 4 worker parallel 物理设备池（client 机房）
# device 1: Pixel 6 (Android 14)
# device 2: Pixel 5 (Android 13)
# device 3: Samsung Galaxy S22 (Android 13)
# device 4: OnePlus 9 (Android 12)
```

### 3.4 connectedAndroidTest 执行（DevOps 张良）

```bash
cd android-impl
./gradlew connectedAndroidTest \
  -Pandroid.testInstrumentationRunnerArguments.class=com.btsheng.erp.e2e.RoleBasedE2ETest,com.btsheng.erp.e2e.DrawPermissionE2ETest,com.btsheng.erp.e2e.ConnectedAndroidTestConfigTest \
  -Pandroid.testInstrumentationRunnerArguments.notAnnotation=android.test.suitebuilder.annotation.LargeTest \
  -Pandroid.useOrchestrator=true \
  --max-workers=4
```
