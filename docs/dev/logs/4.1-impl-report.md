## 4. 关键设计决策

### 4.1 路由常量在 NoOrderPurchaseService 内定义

```java
public static final BigDecimal AMOUNT_PM_THRESHOLD = new BigDecimal("10000");
public static final BigDecimal AMOUNT_GM_THRESHOLD = new BigDecimal("50000");
public static final String ROUTE_PM = "PROCUREMENT_MANAGER";
public static final String ROUTE_GM_PM = "GM+PROCUREMENT_MANAGER";
```

**理由**：与 V52__procurement_manager_role.sql 中 sys_workflow_node.threshold 数值一致。**Story 4.2 应复用本常量**，避免 hardcode。

### 4.2 PurchaseReason 枚举字段冗余 color

前端下拉框配色：
- URGENT_REPLENISH → red
- CUSTOMER_ADD → orange
- STOCK_SWAP → blue
- OTHER → gray

**生产建议**：color 字段应存 sys_dict，本期硬编码简化。

### 4.3 简化实现

- poId 用 `System.currentTimeMillis()` 模拟（生产应 MyBatis insert 后取 ID）
- poNo 用 `PO-{date}-NOORDER` 模拟（生产应接 DocNoGenerator.nextOrderNo()）
- 不写 crm_purchase_order 表（由 Sprint 7 集成阶段实装持久化）
