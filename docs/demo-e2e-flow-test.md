# 全链路演示测试指南

> 下单 → 生管 → 采购 → 仓库 → 质检 · 与 PRD V1.3.8 / 演示种子数据对齐  
> 演示数据脚本：`backend/db/install/demo-flow-seed.sql`  
> 演示账号说明：`backend/db/install/README.md`

## 环境

| 项 | 地址 |
|----|------|
| Web 前端 | `https://cnc.51xiaoping.com`（本地：`http://localhost:5173`） |
| API 网关 | `https://bts.51xiaoping.com` |
| 登录 API | `POST /erp-platform/auth/login` |
| 演示密码 | **123456**（所有角色相同） |

**测试前：**

1. 浏览器清除 `localStorage` 后重新登录（Web 菜单 RBAC 生效）
2. 确认已执行 `backend/db/init.sql` 或补跑 `demo-flow-seed.sql`
3. Android APP 的 `API_BASE_URL` 指向同一网关

## 阶段 0：销售下单（Web · `sales`）

| 步骤 | 页面 | 操作 |
|------|------|------|
| 0.1 | 登录 | 用户名 `sales`，密码 `123456` |
| 0.2 | `/sales/quotes` | **销售 → 报价单**，确认 `BJ20260615-0001` 或 **新建报价** |
| 0.3 | `/sales/quotes/new` | 选客户、填明细、保存并提交 |
| 0.4 | `/sales/orders` | **销售 → 销售订单**，找到 `XS20260615-0001` |
| 0.5 | `/sales/orders/7001` | 点 **详情**，确认状态 **`CONFIRMED`（已确认）**、尚无生产工单号 |

> 报价审批通过后转销售订单，状态为 `CONFIRMED`；**生管转工单**后才会变为 `PRODUCING` 并生成 `GD-…` 工单号。

## 阶段 2：生管转工单 & 工序分配（Web · `prod_mgr`）

| 步骤 | 页面 | 操作 |
|------|------|------|
| 2.0 | `/production/pending-production` | **生产 → 待转产订单**，找到 `XS20260615-0001`，点 **转工单** |
| 2.0b | 同上或 `/sales/orders/7001` | 成功提示「已生成工单 `GD-20260615-0001`」（演示种子会关联已有扫码工单） |
| 2.1 | 登录 | `prod_mgr` / `123456` |
| 2.2 | `/production/workorders` | **生产 → 工单**，搜索 `GD-20260615-0001`，确认 **销售订单** 列显示 `XS20260615-0001` |
| 2.3 | `/production/workorder-detail/9001` | 查看工序列表 |
| 2.4 | `/production/allocation` | **生产 → 工序分配** |
| 2.5 | 同上 | 选工单 `GD-20260615-0001` → **待分配** 区勾选工序 1 → 归属 **委外** → **批量提交** |
| 2.6 | 同上 | 成功提示「已进入采购待委外清单」 |
| 2.7 | `/production/outsub-panel` | **生产 → 委外面板**，确认新委外记录 |

> 生管视图禁止选厂商；选「委外」后由采购在 `/sourcing/outsub-order` 下单。  
> **勿**用「补录工单」作为主流程——该页仅用于无销售订单的例外补录。

## 阶段 4：仓库到货 & 扫码出入库（Android · `warehouse`）

**APP Tab（PRD）：** 仓储 · 待办 · 消息 · 我的（无「扫码」Tab）

| 步骤 | 入口 | 操作 |
|------|------|------|
| 4.1 | 登录 | 快捷登录 **仓管 (warehouse)** |
| 4.2 | 仓储 Tab | 确认底部为 **仓储 · 待办 · 消息 · 我的** |
| 4.3a | 仓储 → **到货 WW-** | 点 **打开摄像头扫码**，对准 `WW20260612-0001` 二维码（扫完自动进入委外到货页并加载） |
| 4.3b | 同上（手动） | 或在输入框填 `WW20260612-0001` → **到货登记** |
| 4.4 | 委外到货页 | 确认厂商/工序/数量 → 填实收数量 → **确认到货** → `SENT → INSPECTED` |
| 4.5 | 仓储 → **入库 WL-** | **扫码** 或输入 `WL-DEMO-STEEL-001` → 选库位 → 确认入库 |
| 4.6 | 仓储 → **出库 WL-** | **扫码** 或输入物料码 → 填工单号 → 确认出库 |
| 4.7 | 仓储 → **到货 PO** | 输入 `1001` → 进入 PO 分批到货页 |

**Web 核对（`warehouse`）：**

| 步骤 | 页面 | 操作 |
|------|------|------|
| 4.8 | `/warehouse/inventory` | **仓储 → 库存** |
| 4.9 | `/warehouse/batches` | **仓储 → 批次列表** |
| 4.10 | `/sourcing/incoming` | **采购 → 到货提醒**，状态应变 **已到货** |

## 角色菜单 RBAC 自检

| 账号 | 应见核心菜单 | 不应见 |
|------|-------------|--------|
| `sales` | 销售（报价/订单/客户） | 工序分配、委外下单 |
| `prod_mgr` | 生产（工单/工序分配/委外面板） | 报价审批、月度对账 |
| `buyer` | 采购（委外下单/PO/到货提醒） | 工序分配（生管页） |
| `warehouse` | 仓储（库存/批次/库位） | 销售报价 |
| `qc` | 品质（检验/FA/CMM） | 委外下单 |

## 常见问题

| 现象 | 处理 |
|------|------|
| 各角色 Web 菜单相同 | 清 localStorage，重新登录 |
| `POST /purchase/no-order` 403 | 使用 `buyer`，不要用 `sales` |
| 到货提醒 Web 不能登记 | 设计如此，登记走 Android |
| 扫码三码 500 | 部署 erp-production Feign 改造后的版本 |
| Android 仓管仍有「扫码」Tab | 更新 APK；纯 WAREHOUSE 角色仅 4 Tab |

## 推荐测试顺序

```
阶段 0（销售确认订单 · CONFIRMED）
  → 阶段 2.0（生管 · 待转产订单 · 转工单）
  → 阶段 2（生管委外分配）
  → 阶段 3（采购委外下单）
  → 阶段 4（仓管 APP 分批到货 + 出入库）
  → 阶段 5（品检验 + 操作工三码）
  → 阶段 6（生管闭环）
```

阶段 1 仅在**新建报价**且需审批时执行。
