2. UX Goals & Principles 愿景与原则

2.1 UX 愿景（≤150 字）

"让操作工奶奶 30 秒学会扫码报工；让业务员键盘流 3 步完成报价。"

UI 不是给人"看"的，是给人在车间"用"的。一线戴手套、光线差、站着操作；办公室键盘流、表格密、效率优先。两种范式，一套灵魂：能 3 步做完的事绝不做第 4 步，能扫一下解决的绝不让用户找菜单。

2.2 设计原则（5 条）

原则 1：扫码即一切（APP 端）

一句话：任何业务都能在 3 步内通过"扫码 + 确认"完成。 反例：操作工进入 APP 后要选"业务类型 → 工序 → 工单 → 机台"四级菜单。不做。 正向案例：E5-S2 扫码开工——扫工单码 → 自动识别机台 → 按"开工"按钮（0 表单填写）。

原则 2：键盘流优先（Web 端）

一句话：高频操作必须有快捷键，Tab 序从主到次，单据保存用 Ctrl+S。 反例：业务员录完一条报价要点 3 次鼠标才能提交。不做。 正向案例：E2-S2 报价录入——F8 图号联想、Ctrl+S 保存、Alt+T 含税切换、Tab 切明细行，键盘走完全程。

原则 3：信息密度友好（Web 列表）

一句话：行高 32px、列宽可调、列可隐藏、过滤条件可见可改。 反例：每页只显示 10 条数据，要点"加载更多"才能继续。不做。 正向案例：E5-S1 工单列表默认 50 条/页，行高 32px，列可拖拽排序、可隐藏。

原则 4：离线可放心（APP 端）

一句话：扫码后断网不卡顿、不丢数；状态条明牌告诉用户"离线多少条待上传"。 反例：扫码报工后断网，结果丢失、用户不知情。不做。 正向案例：E1-S4 离线模式——本地 SQLite 缓存 500 条单据 + 扫码记录；状态条 32px 顶置，橙色横幅"离线模式 5 条待上传"。

原则 5：拒绝伪需求装饰

一句话：不做"皮肤主题切换、动画转场、骨架屏加载 800ms"等与业务无关的视觉。 反例：报价提交后弹 3 秒花瓣飘落动画。不做。 正向案例：保存草稿后立即 Toast 提示 + 列表行更新；无 Loading 动画（直接 disable 按钮 + 文字"保存中"）。

7.1 流程 1：业务员录入报价（E2-S2）—— 3 步完成

7.1.1 流程图

开始
  ↓
[1] 点击顶部菜单「销售」→「报价单」→「新建」
  ↓
[2] 填写客户（自动联想 / F2 选历史）
  ↓
[3] F8 触发图号联想 → 选图号 → 自动带出 BOM
  ↓
[4] 调整明细（数量、单价）→ Alt+T 切换含税
  ↓
[5] Ctrl+S 保存草稿 或 Ctrl+Enter 提交审批
  ↓
[6] 金额 < 5 万 → 业务员自审通过 → 结束
[6] 金额 5-20 万 → 自动路由部门经理 → 结束（待办推送给经理）
[6] 金额 > 20 万 → 自动路由总经理 → 结束
[6] 客户为黑名单 → 直接驳回 + Toast 提示
  ↓
结束

7.1.2 关键节点

节点 触发 UI 反馈
选客户 文本框 + 联想下拉 客户编码 + 名称 + 信用额度
F8 图号联想 1 字符触发远程搜索 下拉：图号 + 最新版本 + 缩略图
选图号 鼠标点击 / 回车 BOM 自动带出 + 成本自动汇总
切换含税 Alt+T 金额实时重算 + 标签切换
保存 Ctrl+S 右上角 Toast "草稿已保存" + 单据状态变"草稿"
提交 Ctrl+Enter 弹窗"已提交审批" + 跳转到列表页

7.1.3 边界 case

Case 处理
客户为黑名单 提交时弹窗拦截 + Toast "该客户为黑名单，无法报价"
金额阈值跨档 5 万 / 20 万边界走向上取整档（5.00 万 = 5-20 万档）
客户保护期内 选客户时显示红色 Tag "保护中" + 提示"该客户由 X 保护"
图号无 BOM 提示"该图号未维护 BOM"+ 引导到 BOM 维护页
网络中断 提交失败时 Toast + 草稿本地缓存，恢复后自动重试

## 13. Animation &amp; Motion 动画与动效

### 13.1 Motion Principles 动效原则

&gt; **克制 + 目的明确**：动效只用于&quot;反馈状态变化&quot;，不用于装饰。

| 原则 | 描述 |
|------|------|
| **目的性** | 每个动效必须有目的（提示状态变化、引导视线） |
| **时长 ≤ 300ms** | 95% 动效在 200-300ms 内 |
| **缓动函数** | `cubic-bezier(0.4, 0, 0.2, 1)`（Material Standard） |
| **不阻挡操作** | 不在动效期间 disable 按钮 |
| **可被关闭** | `prefers-reduced-motion` 媒体查询时禁用 |

### 13.2 Key Animations 关键动效

| 场景 | 动效 | 时长 | 缓动 |
|------|------|------|------|
| Modal 打开 | 缩放 + 淡入 | 200ms | ease-out |
| Drawer 滑入 | 从右滑入 | 300ms | ease-out |
| Toast | 从顶部滑入 + 2s 后自动淡出 | 200ms + 2s + 200ms | ease-out |
| 拖拽归位 | 弹性回弹 | 200ms | spring |
| 拖拽成功 | 绿色高亮闪烁 | 200ms × 2 | linear |
| 状态变更 | 背景色过渡 | 200ms | ease-in-out |
| Loading | 旋转（24px 圆环） | 800ms 循环 | linear |
| 扫码成功 | 屏幕闪绿 + 震动 | 200ms | - |
| 看板卡片插入 | 上滑 + 淡入 | 300ms | ease-out |
| 列表行更新 | 黄色高亮闪烁 | 200ms × 2 | linear |

### 13.3 禁用动效的场景

- **数据加载**：不显示骨架屏（直接 disable 按钮 + 文字&quot;加载中&quot;）
- **表单提交**：不显示 Loading 动画（按钮文字变&quot;提交中&quot;）
- **页面切换**：不淡入淡出（直接切换）

C.2 灵魂一致性自评

问：UX 计成的 UI 能不能让操作工奶奶学会？

答：能。

扫码页只有 1 个扫码框 + 1 个大按钮；报工只需扫一下、输三个数字、按"提交"；离线状态条让奶奶始终知道"数据有没有丢"。

业务员录报价用键盘：F8 图号联想 → Ctrl+S 保存 → Ctrl+Enter 提交，3 步完成，0 次鼠标。

生管排产用拖拽：拖卡片 → 看冲突高亮 → 确认，3 步完成，0 次菜单。

财务审批一键通过：Enter 批量通过 + Shift+Click 多选，0 次鼠标。

一码到底，一数到底——UI 是用来"用"的，不是用来"看"的。
