# 报价流程完整设计方案

> 版本：V2.1-Quote-Flow · 2026-06-21  
> 状态：**V2.1 整改已完成**（2026-06-21 · 见 [§9 整改清单](#九开发整改清单)）

## 一、核心原则

| 序号 | 原则 | 说明 |
|------|------|------|
| 1 | **业务员发起报价** | 业务员上传图纸、填基础数据、填数量，**不填工艺、不填单价** |
| 2 | **工程师定义工艺** | 工程师收到待办后，**填写工艺路线和工时** |
| 3 | **报价范本驱动成本** | 系统套用报价范本（成本项基础数据）自动计算各项成本 |
| 4 | **报价需要审批** | 涉及定价，必须走审批流程 |
| 5 | **报价单可导出** | 审批通过后可按客户范本导出报价单 PDF |

## 四、完整报价流程（支持多图号）

### 4.1 流程总览

```
业务员收到客户询价（可能多张 PDF）
    ↓
① 打开「新建报价」
    ↓
② 选择客户
    ↓
③ 添加明细行（支持多个图号）
    └── 每行：搜索/选择/上传图号
    ↓
④ 每明细行操作：
    ├── 输入【客户图号】搜索
    │   ├── 找到 → 选择，带出材质/尺寸/重量
    │   └── 未找到 → 上传 PDF
    │       ├── 文件名 → 客户图号（可改）
    │       ├── 系统生成 DWG-YYYYMMDD-XXXX
    │       └── 填材质/尺寸/重量
    └── 填写数量
    ↓
⑤ 填写客户需求描述（整体）；**报价阶段无交货日期**
    ↓
⑥ 【保存草稿】或【提交给工程师】
    ↓
工程师待办 → 按图号分别定义工艺 + 表处面积
    ↓
⑧ 系统套用报价范本自动计算每行单价/小计
    ↓
⑨ 业务员确认 → 【提交审批】
    ↓
⑩ 多级审批 → APPROVED
    ↓
⑪ 导出 PDF（显示客户图号）
```

### 4.2 业务员页面要点

- 明细列：**客户图号**、产品名称、数量；**单价待计算**（只读）
- 去掉：交货日期、业务员手填单价
- 按钮：`保存草稿` | `提交给工程师 →`（非「提交审批」）
- 工艺预览区：只读，工程师完成后展示

### 4.3 工程师页面要点

- 图号 Tab/下拉切换（多明细行）
- 只读：材质、尺寸、重量
- 从工艺库添加工序 → 排序 → 填工时
- 表处面积：阳极氧化 / 固溶 / 整形
- 按钮：`保存` | `计算报价` | `上一个/下一个图号`

## 五、图号处理机制

### 5.1 两套图号体系

| 图号类型 | 字段 | 格式 | 生成时机 | 用途 |
|----------|------|------|----------|------|
| **客户图号** | `crm_drawing.customer_drawing_no` | PDF 文件名如 `615-03953-0009` | 上传时读取，可改 | 搜索、对客户展示 |
| **系统图号** | `crm_drawing.drawing_no` | `DWG-YYYYMMDD-XXXX` | 上传时自动生成 | 系统内部唯一标识 |

### 5.2 搜索逻辑

业务员输入 `615-03953` → 在 `customer_drawing_no`（及 title）模糊匹配 → 列表选择或提示上传。

### 5.3 上传逻辑

```
选择 PDF → 读取文件名作为 customer_drawing_no 预览
         → 生成 drawing_no (DWG-…)
         → MinIO 存储
         → 填写材质/规格/单件重量
         → 入库
```

**提示文案**：请将 PDF 文件名改为与客户图号一致，便于搜索复用。

## 六、状态机

```
DRAFT（业务员草稿）
    ↓ POST /quotes/{id}/submit-to-engineer
PENDING_ENG（待工程师定义工艺）
    ↓ 工程师全部明细 calculate 完成
PENDING_APPROVAL（待审批，兼容旧值 SUBMITTED）
    ↓ 多级审批通过
APPROVED（已审批）
    ↓ 转单
CONVERTED

REJECTED ← 审批驳回（可退回 DRAFT 修改）
```

| 状态 | 业务员可操作 | 工程师可操作 |
|------|-------------|-------------|
| DRAFT | 编辑、提交工程师 | — |
| PENDING_ENG | 只读查看 | 定义工艺、计算 |
| PENDING_APPROVAL | 只读、提交审批前确认 | — |
| APPROVED | 导出、转订单 | — |

## 八、API 设计（V2.1 报价流）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/drawings?keyword=` | 扩展：keyword 同时搜 `customer_drawing_no` |
| POST | `/drawings/upload` | 扩展：`customerDrawingNo`、`unitWeight` |
| POST | `/quotes` | 创建（DRAFT，items 无单价） |
| POST | `/quotes/{id}/submit-to-engineer` | DRAFT → PENDING_ENG |
| POST | `/quotes/items/{itemId}/process` | 工程师保存工艺 |
| POST | `/quotes/items/{itemId}/calculate` | 套用成本项计算单价 |
| POST | `/quotes/{id}/submit` | PENDING_APPROVAL 提交审批（或工程师完成后业务员确认） |
| GET | `/quotes/{id}/export?format=pdf` | 导出 PDF（含客户图号） |
| POST | `/quotes/{id}/send-email` | 审批通过后发送 PDF 至客户邮箱 |
| PUT | `/customers/{id}` | 更新客户联系人/邮箱 |
