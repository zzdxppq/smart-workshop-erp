## 1. UX 愿景（V1.3.7 升级版）

**"让操作工奶奶 30 秒学会扫码报工，让采购 30 秒生成对账单 + 发邮件，让总经理 30 秒看完料号成本。"**

UI 不是给人"看"的，是给人在车间"用"的、在办公室"跑"的。**新增 3 类典型场景的灵魂自检**：

| 角色 | 30 秒目标 | 灵魂自检 |
|------|----------|----------|
| 操作工 | 30 秒学会扫码报工 | 扫一次码 + 录数字 + 提交，无菜单 |
| 采购 | 30 秒生成对账单 + 发邮件 | 选月份+厂商 → 点【生成对账单】 → 点【发送邮件】，3 步搞定 |
| 总经理 | 30 秒看完料号成本 | Ctrl+K 搜料号 → 5 Tab 切换，每 Tab ≤ 6 行核心数据 |

**V1.3.7 新增 3 条灵魂自检**：
- **生管 vs 采购职责灵魂自检**：生管只看到"工序归属"勾选框，看不到厂商下拉；采购只看到"厂商选择"下拉，看不到工序归属（V1.3.7 客户原话）
- **对账不含"线下"灵魂自检**：对账模块无"采购带纸去厂商处"等系统外动作按钮（V1.3.7）
- **单一 163 邮箱灵魂自检**：消息中心无"短信发送"按钮；通知偏好只有"163 邮箱"单选项（V1.3.7）

## 2. 设计原则（5 条 + 3 条 V1.3.7 新增）

### 原则 1：扫码即一切（APP 端）
**一句话**：任何业务都能在 3 步内通过"扫码 + 确认"完成。
**反例**：操作工进入 APP 后要选"业务类型 → 工序 → 工单 → 机台"四级菜单。**不做**。

### 原则 2：键盘流优先（Web 端）
**一句话**：高频操作必须有快捷键，Tab 序从主到次，单据保存用 `Ctrl+S`。
**V1.3.7 新增快捷键**：
- `Ctrl+Shift+M` = "生成月度对账单"
- `Ctrl+Shift+E` = "163 邮箱测试发送"
- `Ctrl+K` 搜索：物料/工单/客户/供应商/料号

### 原则 3：信息密度友好（Web 列表）
**一句话**：行高 32px、列宽可调、列可隐藏、过滤条件可见可改。
**V1.3.7 新增**：委外面板（E11-S4）支持按 7 状态分组 + 返修≥2 高亮 + 即将逾期红色。

### 原则 4：离线可放心（APP 端）
**一句话**：扫码后断网不卡顿、不丢数；状态条明牌告诉用户"离线多少条待上传"。
**V1.3.7 新增**：APP"到货扫码"入口（**V1.3.5 新增**）扫 WW- 委外单码，离线缓存 500 条。

### 原则 5：拒绝伪需求装饰
**一句话**：不做"皮肤主题切换、动画转场、骨架屏加载 800ms"等与业务无关的视觉。

### 原则 6（V1.3.7 新增）：**职责越权 = 红线**
**一句话**：生管界面看不到厂商选择下拉，采购界面看不到工序归属勾选框。
**反例**：给生管"待委外工序清单"页加一个"代选厂商"按钮 —— **不做**（V1.3.7 客户原话"生管决定自/委，采购选厂商"）。

### 原则 7（V1.3.7 新增）：**PRD 边界外动作 = 不画按钮**
**一句话**：对账模块不画"采购带纸去厂商处"、"采购线下跟厂商核对"等按钮。
**反例**：对账单详情页加"标记已线下核对"按钮 —— **不做**（V1.3.7 客户原话"线下流程不去涉及"）。

### 原则 8（V1.3.7 新增）：**单渠道通知 = 单一收件人字段**
**一句话**：厂商资料只显示"邮箱"必填 + "电话"选填；通知偏好下拉只有"163 邮箱"一个选项。
**反例**：厂商资料"通知偏好"下拉有"短信/邮件/微信/163" 4 选 —— **不做**（V1.3.7 客户原话"通知厂商改为 163 邮箱"）。

## 5. Web 端 - 关键 Story 五段式规格

### 5.1 E5-S4 工序分配（V1.3.7 新增 · 关键 Story）

> **生管操作：勾选"自制/委外" → 保存 → 推送给采购**

| 维度 | 规格 |
|------|------|
| **角色** | 生管 |
| **入口** | 生产菜单 → 工序分配 |
| **数据** | 工单 + 工艺路线（来自 E5-S1 + E3-S4） |
| **操作** | 每道工序单选"自制/委外" + 数量（默认继承工单数量，可改） |
| **保存** | 写入 `outsub_allocation` 表（V1.3.7 新增），生管 ID 留痕 |
| **推送** | 勾"委外"的工序自动推送给采购"待委外清单"页 |
| **红线** | 页面**无厂商下拉框**（V1.3.7 红线） |
| **响应** | P95 ≤ 1s；批量保存支持（一次勾多道工序） |
| **离线** | 不支持（生管 PC 端操作） |

### 5.2 E5-S4 委外下单（V1.3.7 改版 · 关键 Story）

> **采购操作：从待委外清单取 → 选厂商 → 创建 WW- 单 → 自动 163 邮箱通知**

| 维度 | 规格 |
|------|------|
| **角色** | 采购 |
| **入口** | 采购菜单 → 委外下单（V1.3.7 改版） |
| **数据** | 待委外工序清单（来自生管推送） |
| **操作** | 为每道工序选厂商（自动带历史价 + 历史交期）→ 创建 WW- 单 |
| **创建** | 写入 `outsub_order`（V1.3.7） + `outsub_allocation_vendor` |
| **通知** | **自动发 163 邮箱**（V1.3.7 单一渠道）含 PDF 委外订单详情 |
| **红线** | 页面**无"自制/委外"切换**（V1.3.7 红线） |
| **响应** | P95 ≤ 2s；创建 + 邮件发送 ≤ 5s |
| **离线** | 不支持 |

### 5.3 E6-S1 月度对账（V1.3.6 改版 · 关键 Story）

> **采购操作：4 步完成对账 · 全系统内动作 · 不含"线下"**

| 维度 | 规格 |
|------|------|
| **角色** | 采购 + 财务 |
| **入口** | 采购菜单 → 月度对账 |
| **数据** | 月份 + 厂商 → 系统汇总已 STORED 的 WW- 单 |
| **4 步** | ① 生成对账单 PDF ② 发送 163 邮件 ③ 上传签字扫描件 ④ 对账已确认 |
| **红线** | 页面**无"采购带纸去厂商处"按钮**（V1.3.7 红线）|
| **签字件** | AES-256-GCM 加密 · 5 年保留 · 下载限总经理/财务总监/采购 · 全审计 |
| **付款触发** | 三条件：对账已确认 + 签字附件上传 + 双方金额一致 |
| **响应** | PDF 生成 ≤ 10s；邮件发送 ≤ 5s；上传 ≤ 5s |
| **离线** | 不支持 |

### 5.4 E9-S5 + E11-S5 料号成本聚合（V1.3.4 新增 · 关键 Story）

> **Ctrl+K 搜料号 → 5 Tab 切换 → 权限隔离 → 导出**

| 维度 | 规格 |
|------|------|
| **角色** | 全员（按权限隔离） |
| **入口** | Ctrl+K 全局搜索 / 物料菜单 → 料号成本聚合 |
| **检索** | 料号（必填）+ 时间范围（可选）+ 客户（可选） |
| **5 Tab** | 价格 / 材料成本 / 工时成本 / 外协成本 / 总成本 |
| **权限** | 管理层看全量；业务员只看自己客户；一线只看自己工序工时 |
| **缓存** | Redis TTL 3600s + Stream cost-invalidate 失效驱动 |
| **失效触发** | 委外单完成 / 工时录入 / 物料价格变更 |
| **导出** | Excel / PDF |
| **响应** | 聚合 P95 ≤ 3s |

### 5.5 E6 委外 7 状态机（V1.3.4 升级 · 关键 Story）

> **从下单到对账 7 主状态 + 1 衍生态 · 返修闭环 + 次数预警**

| 状态 | 触发 | 通知 |
|------|------|------|
| PENDING_SHIP | 采购创建 WW- 单 | 厂商：163 邮箱 |
| SHIPPING | 厂商发货（系统外）+ 仓管代录 | 厂商：163 邮箱 |
| PENDING_INSPECTION | 仓管扫 WW- 到货 | 生管 + 品质（APP+PC） |
| INSPECTING | 品质去仓库领料 | 生管 CC |
| QUALIFIED_STORAGE | 品检录入"合格" | 仓管 |
| STORED | 仓管扫 WL- + 录库位 | 生管 + 财务 |
| REPAIR_REQUESTED | 品检录入"不合格" | 采购 + **厂商：163 邮箱** |
| NOTIFIED_REPAIR | 采购生成返修单 | **厂商：163 邮箱** + 生管 |

**返修闭环**：返修次数 ≥ 2（阈值 Nacos 可配）→ 自动推高层 + 采购主管 + 生管主管（APP + PC 红点 + 企业微信 webhook）
