## 1. 项目概述

### 1.1 项目背景

本系统面向CNC机加工行业，针对离散型制造企业的核心业务痛点，打造一套集客户管理、报价订单、生产排产、车间执行、品质管控、仓储物流、委外加工、财务人事于一体的综合ERP管理平台。

CNC机加工行业具有显著的行业特征：非标定制件多、工序流转复杂、委外加工频繁、图纸管理核心、编程管理依赖历史积累、材料管理需追踪损耗、产能依赖设备状态、品质检测要求严格。传统通用型ERP难以满足这些细分需求，亟需一套贴合行业特点的专业化解决方案。

### 1.2 项目目标

- **提升报价效率**：建立标准化的报价流程，实现报价到订单的无缝转化
- **优化生产排产**：支持自加工与外协混合模式，提升产能利用率
- **强化图纸管理**：贯穿客户图纸→工程转化→厂内图纸→打印随单的全生命周期
- **精准品质管控**：三次元检测、FA首件确认、质检报告留痕
- **实时数据看板**：产值、交付、逾期、库存等关键指标实时呈现
- **移动端协同**：扫码出入库、报工打卡、扫码过站、消息推送

### 1.3 行业特点分析

| 序号 | 行业特点 | 系统应对策略 |
|------|---------|-------------|
| 1 | 离散型制造，非标定制件多 | 图纸+图号管理，BOM灵活配置 |
| 2 | 工序流转复杂（车床→CNC→后加工→表面处理→质检→入库） | 工序路线管理，工单流转追踪 |
| 3 | 委外加工频繁（氧化/电镀/喷涂/热处理/三次元检测） | 委外加工管理模块，外协厂商协同 |
| 4 | 图纸管理核心 | 图纸版本管理，PDF附件跟随订单流转 |
| 5 | 编程管理（新件必编程，历史件可复用） | 编程任务管理，历史工艺库复用 |
| 6 | 材料管理按料号+尺寸，下料损耗追踪 | 物料编码规则，损耗系数管理 |
| 7 | 产能依赖设备（机台号、设备状态、班别） | 设备台账，机台排产，班别管理 |
| 8 | 品质检测（三次元、FA首件） | 质检方案配置，检测报告上传 |
| 9 | 产值计算（自制+外协） | 工时统计，产值汇总报表 |

## 3. 功能模块详述

|---------|---------|-------|---------|---------|

| 联系人管理 | 多联系人维护、职位、电话、邮箱、主要联系人标记 | P0 | 业务员 | 甲方需求 |
| 客户保护 | 领用保护期、申请/指派/收回机制，防止撞单 | P0 | 业务员、管理员 | 甲方需求 |
| 客户共享 | 跨部门共享、协作跟进 | P1 | 业务员 | 甲方需求 |
| 洽谈进展 | 跟进记录、下次跟进提醒、进展阶段 | P0 | 业务员 | 甲方需求 |
| 客户积分 | 累计交易额、积分规则、积分兑换 | P2 | 管理员 | 甲方需求 |
| 竞争对手 | 竞品信息、价格策略、竞争分析 | P2 | 业务员 | 甲方需求 |
| 客户智能分配 | 导入/领用/指派/共享/保护/收回/转移，智能防撞单抢单 | P1 | 业务员、管理员 | 行业ERP参考 |
| 客户高效跟进 | 自动化动态监控待办通知/跟进过程/业绩成果，通过日程/邮件/提醒加速跟进 | P1 | 业务员 | 行业ERP参考 |
| 客户管理策略 | 分类/地区/资质/审核/跟进策略 | P1 | 业务员 | 行业ERP参考 |
| 客户实时统计 | 新增客户对比/跟进领用对比/分布/走势/联系人/绩效统计 | P1 | 业务员、管理员 | 行业ERP参考 |

#### 关键业务流程

1. **客户录入**：业务员录入客户基本信息，上传营业执照等附件
2. **客户领用**：申请领用或管理员指派，进入保护期（如30天）
3. **跟进维护**：记录洽谈进展，设置下次跟进提醒
4. **客户转化**：客户进入报价/下单环节，累计交易记录

#### 数据实体/字段说明

| 实体名称 | 核心字段 |
|---------|---------|
| 客户档案 | 客户编码、客户名称、行业分类、信用额度、开票信息、地址、保护状态、负责人、创建时间 |
| 联系人 | 联系人姓名、职位、电话、邮箱、是否主要联系人、所属客户 |
| 洽谈记录 | 跟进时间、跟进内容、进展阶段、下次跟进时间、跟进人 |

#### 与其他模块关联

- **报价管理**：客户信息关联报价单
- **订单管理**：客户信息关联销售订单
- **财务管理**：客户应收账款、对账管理
- **报表中心**：客户交易统计、客户价值分析

### 3.2 报价管理

#### 模块概述
报价管理模块承接客户需求，支持上传客户图纸、录入图号、BOM报价预估、多级审批，实现报价单导出和报价转订单的无缝衔接。

#### 核心功能列表

| 功能名称 | 功能描述 | 优先级 | 使用角色 | 参考来源 |
|---------|---------|-------|---------|---------|
| 报价单录入 | 客户选择、交期要求、产品信息、图号录入 | P0 | 业务员 | 甲方需求 |
| 图纸上传 | PDF格式客户图纸上传、版本管理、在线预览 | P0 | 业务员 | 甲方需求 |
| BOM报价预估 | 物料清单、材料成本、加工成本、表处成本估算 | P0 | 业务员、工程师 | 甲方需求 |
| 材料成本核算 | 材料型号、尺寸、计量单位、单价、用量计算 | P0 | 工程师 | 甲方需求 |
| 工艺预估 | 初步工艺路线、工时估算、参考历史工艺 | P0 | 工程师 | 甲方需求 |
| 表处工艺 | 表面处理要求（氧化/电镀/喷涂等）录入 | P1 | 工程师 | 甲方需求 |
| 销售费用 | 预估销售费用、运杂费、包装费等 | P1 | 业务员 | 甲方需求 |
| 整体报价 | 成本汇总、利润率计算、报价审批 | P0 | 业务员、管理员 | 甲方需求 |
| 报价审批 | 多级审批流程、审批意见、审批历史 | P0 | 管理员 | 甲方需求 |
| 报价单导出 | Excel/PDF格式导出，自定义模板 | P0 | 业务员 | 甲方需求 |
| 报价转订单 | 审批通过后转销售订单，附件自动带出 | P0 | 业务员 | 甲方需求 |
| 是否字段扩展 | 是否FA、是否新件、是否需要编程等扩展字段 | P1 | 工程师 | 甲方需求 |
| 报价价格控制 | 支持零售/批发/促销/折扣多种价格，自动限价/一键询价/库存查询/历史价格对比 | P1 | 业务员、工程师 | 行业ERP参考 |
| 报价跟进管理 | 支持报价复制/指派/点评/修改/导入/导出/打印/统计 | P1 | 业务员 | 行业ERP参考 |

#### 关键业务流程

1. **报价录入**：业务员接到客户询价，创建报价单，上传客户图纸PDF，录入图号
2. **工程评估**：工程师评估工艺路线、工时、材料尺寸、材质、表处工艺
3. **成本核算**：系统自动计算材料成本、加工成本、表处成本
4. **报价审批**：提交审批，按金额分级审批
5. **报价输出**：审批通过后导出报价单发送客户
6. **报价转订单**：客户确认后，报价单转销售订单，图纸附件自动带出

#### 数据实体/字段说明

| 实体名称 | 核心字段 |
|---------|---------|
| 报价单 | 报价单号、客户ID、报价日期、交期要求、总金额、审批状态、是否转订单、创建人 |
| 报价明细 | 明细ID、图号、产品名称、材质、尺寸、数量、单价、金额、是否FA |
| 报价图纸 | 图纸ID、报价单号、图号、文件名、文件路径、上传时间、版本号 |
| 报价工艺 | 工艺ID、报价明细ID、工序名称、工时、单价、金额 |
| 报价材料 | 材料ID、报价明细ID、材料编码、尺寸、数量、单价、金额 |
| 报价表处 | 表处ID、报价明细ID、表处类型、单价、金额 |

#### 与其他模块关联

- **客户管理**：客户信息关联
- **图纸管理**：图纸上传与版本管理
- **订单管理**：报价转订单
- **BOM管理**：报价BOM估算
- **财务管理**：成本核算

| 工程转化 | 客户图纸→厂内图纸转化，格式转换 | P0 | 工程师 | 甲方需求 |
| 厂内图纸设计 | CAD/CAM文件管理，设计模块集成 | P1 | 工程师 | 甲方需求 |
| 图纸版本管理 | 版本号、变更记录、历史版本追溯 | P0 | 工程师 | 甲方需求 |
| 图纸打印 | 随工单打印、批量打印、打印记录 | P0 | 工程师、生管员 | 甲方需求 |
| 图纸权限 | 按角色/项目控制查看/下载权限 | P1 | 管理员 | 甲方需求 |

#### 关键业务流程

1. **图纸上传**：报价或订单环节上传客户图纸PDF，录入图号
2. **工程转化**：工程师将客户图纸转化为厂内图纸，标注加工要求
3. **版本更新**：图纸变更时创建新版本，记录变更内容
4. **图纸下发**：图纸关联工单，车间现场查看或打印
5. **图纸归档**：订单完成后图纸归档，便于历史查询

| 图纸版本 | 版本ID、图纸ID、版本号、变更说明、创建时间 |
| 图纸关联 | 关联ID、图纸ID、业务类型（报价/订单/工单）、业务单号 |

#### 与其他模块关联

### 3.8.1 扫码功能详细设计

#### 使用场景

**扫码开工**：操作工开始加工前，扫描工单上的二维码，系统自动记录开工时间、操作人工号、设备机台号，并将工单状态更新为"加工中"。适用于需要精确记录每个工序开始时间的场景。

**扫码报工**：工序加工完成后，操作工扫描工单二维码，录入完工数量、合格数量、报废数量，系统自动校验并报工。适用于需要实时报工、统计产量的场景。

**扫码过站**：工序间物料流转时，操作工扫描流转码（不是工单码），记录物料去向、交接数量、交接时间，实现工序间的物料追溯。适用于多工序协作、需要追踪物料流转路径的场景。

#### 功能设计

**扫码开工**：
- 扫描对象：工单二维码（一维码/二维码均可）
- 记录内容：开工时间、操作人工号、设备机台号、工单状态更新
- 校验规则：工单状态必须为"待加工"或"已排产"，否则不允许开工
- 权限控制：只能扫自己负责的工单，不能扫他人的工单

**扫码报工**：
- 扫描对象：工单二维码
- 记录内容：完工数量、合格数量、报废数量、加工人员、机台号、班别、报工时间
- 校验规则：报工数量不能大于派工数量；合格+报废数量必须等于完工数量
- 触发动作：报工完成后，自动通知品检员进行工序检验

**扫码过站**：
- 扫描对象：流转码（每批物料一个流转码，区别于工单码）
- 记录内容：物料去向（下工序）、交接数量、交接时间、交接人
- 校验规则：流转码必须与上工序的报工记录关联，确保物料来源可追溯
- 触发动作：下工序接收物料后，扫描流转码自动开始本工序的扫码开工

#### 扫码开工、扫码报工、扫码过站的关系

这三个功能构成了车间生产执行的完整闭环：

1. **扫码开工** → 记录工序开始时间，工单状态变为"加工中"
2. **加工完成** → 操作工完成本工序加工
3. **扫码报工** → 录入完工数量、合格/报废数量，通知品检员检验
4. **检验合格** → 品检员确认合格，允许流入下工序
5. **扫码过站** → 记录物料流向哪个下工序，生成流转码
6. **下工序接收** → 下工序操作工扫描流转码，自动触发本工序的扫码开工

**关键区别**：
- 扫码开工和扫码报工扫描的是**工单码**（记录加工人和时间）
- 扫码过站扫描的是**流转码**（记录物料去向和数量）

#### 码的贴法建议

为了确保扫码功能落地，建议采用以下码的贴法：

**1. 工单码**（用于扫码开工、扫码报工）：
- 生成时机：工单下发车间时生成
- 贴码位置：每个工单打印一张工单卡，工单卡上附工单二维码
- 码的内容：工单号、产品编码、工序号、计划数量
- 使用场景：操作工扫码开工、扫码报工时扫描

**2. 流转码**（用于扫码过站）：
- 生成时机：上工序报工完成、检验合格后生成
- 贴码位置：随物料一起流转，可以贴在物料外包装或流转单上
- 码的内容：流转ID、工单号、工序号、合格数量、下工序
- 使用场景：工序间物料交接时扫描，记录物料去向

**3. 设备码**（可选，用于快速选择机台）：
- 生成时机：设备台账创建时生成
- 贴码位置：贴在设备显眼位置
- 码的内容：设备ID、设备名称、机台号
- 使用场景：扫码开工时，快速扫描设备码录入机台号，避免手动选择

**实施建议**：
- 先试点：选1-2个工单试点，验证码的贴法和扫码流程
- 培训操作工：确保操作工理解三种码的区别和使用场景
- 逐步推广：试点成功后，全面推广到所有工单
